## 1. Edge Cases (엣지 케이스)

### ✅ 해결됨

**[EC-001] 직원 선택 시 자동입력 데이터 덮어쓰기 정책**
- **위치**: 2.2.2 직원정보입력 > 직원 선택
- **답변**: 수동으로 입력한 데이터가 있어도 확인 없이 덮어씌웁니다
- **반영 완료**:
  - 정책서 2.2.2에 설명 추가
  - 코드는 기존대로 유지 (이미 구현됨)

**[EC-002] 임시저장 복원 시 확인 절차**
- **위치**: 2.5.2 임시저장 불러오기
- **답변**: 현재 작성 중인 데이터가 있을 때 confirm 추가
  - 메시지: "임시저장 데이터를 불러올 경우, 현재 입력한 데이터는 사라집니다. 불러오시겠습니까?"
- **반영 완료**:
  - 정책서 2.5.2에 확인 절차 추가
  - InsuranceAcquisition.tsx의 handleLoadTempData 함수에 confirm 로직 구현

**[EC-003] 보험 체크 해제 시 입력 데이터 보존 정책**
- **위치**: 2.3.1 보험 종류 선택
- **답변**: 보험 체크 해제 시 해당 보험의 입력 데이터는 모두 초기화됩니다
- **반영 완료**:
  - 정책서 2.3.1에 데이터 초기화 설명 추가
  - InsuranceAcquisition.tsx의 handleEmployeeChange 함수에 초기화 로직 구현

---

### ℹ️ Info

**[EC-004] 피부양자 0명 허용 여부**
- **위치**: 2.3.3 건강보험 > 피부양자 정보
- **내용**: "기본 1명, 추가/삭제 가능, 최소 1명 유지"
- **문제**: 건강보험 가입자가 피부양자가 없을 수도 있는데 최소 1명 강제?
- **구현 영향**: 비즈니스 로직 검증

**[EC-005] 임시저장 없을 때 "임시저장 불러오기" 버튼 동작**
- **위치**: 2.5.1 임시저장 불러오기
- **문제**: 저장된 데이터가 없을 때 버튼 클릭 시 어떻게 표시?
- **구현 영향**: 빈 상태 UI

## 2. 모호한 표현 (Ambiguous Expressions)

### 🔴 Critical

**[AM-001] 공단 FAX 번호 매핑 규칙**
- **위치**: 2.2.1 사업장 정보 > 공단 FAX
- **내용**: "사업장 주소를 기준으로 관할 공단의 FAX 번호가 자동으로 입력됩니다"
- **문제**:
  - 주소 → 공단 매핑 테이블이 있나요?
  - 시/구 단위? 동 단위?
  - 매핑 데이터는 어디서 관리?
- **구현 영향**: 핵심 비즈니스 로직

---

### ⚠️ Warning

**[AM-002] PDF 미리보기 버튼 활성화 조건**
- **위치**: 2.7 PDF 파일 미리보기
- **내용**: "신고서 작성이 완료되면 활성화됩니다"
- **문제**:
  - "작성 완료"의 정확한 조건은?
  - 모든 필수 항목 입력 완료?
  - 유효성 검증 통과?
  - 어떤 필드가 필수인지 명확히 정의 필요
- **구현 영향**: 버튼 활성화 조건

**[AM-003] 글자수 제한의 기준**
- **위치**: 여러 곳 (성명 30자, 부호 6자 등)
- **내용**: "최대 30자까지 입력 가능"
- **문제**:
  - 바이트 기준? 글자 기준?
  - 특수문자 포함?
  - 공백 포함?
- **구현 영향**: 입력 검증 로직

---

### ℹ️ Info

**[AM-004] "관할 공단" 용어 정의**
- **위치**: 여러 곳
- **내용**: "관할 공단", "지사"
- **문제**: 공단의 정식 명칭과 조직 구조 확인 필요 (4대보험 각각 다름)
- **구현 영향**: 용어 일관성

## 3. 기획자 질문 (Planner Questions)

### 🔴 Critical

**[PQ-001] 직원 자동입력 정책**
- **질문**: 직원 선택 시 자동입력되는 필드 중 수정 가능한 필드는?
  - 모든 필드 수정 가능?
  - 특정 필드만 읽기 전용?
- **영향**: 폼 필드 권한 설계

**[PQ-002] 공단 FAX 번호 데이터 소스**
- **질문**:
  - 주소 → 공단 FAX 매핑 데이터를 어디서 가져오나요?
  - LocalStorage? 외부 API? 하드코딩?
  - 데이터 업데이트 주기는?
- **영향**: 데이터 아키텍처

**[PQ-003] PDF 미리보기 필수 항목 정의**
- **질문**: PDF 미리보기 버튼 활성화를 위한 필수 입력 항목 목록은?
  - 사업장 정보 전체?
  - 직원 1명 이상?
  - 보험 1개 이상 체크?
- **영향**: 유효성 검증 로직

---

### ⚠️ Warning

**[PQ-004] 사업장 정보 동기화 정책**
- **질문**: "사업장 정보는 직원 정보와 동기화됩니다"
  - 언제 동기화? (실시간? 저장 시?)
  - 어느 방향? (직원 → 사업장? 양방향?)
- **영향**: 상태 관리

**[PQ-005] 신고내역 조회 데이터 저장 위치**
- **위치**: 2.9 신고내역 조회
- **질문**:
  - 신고 완료된 데이터는 어디에 저장?
  - LocalStorage 키는?
  - 데이터 구조는?
- **영향**: 데이터 모델링
