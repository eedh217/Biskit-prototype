description: B2B 관리자 페이지의 리스트 화면/추가 팝업/수정 팝업 공통 정책을 표준으로 적용한다. 화면 구현 시 본 정책을 기본값으로 반영하고, 화면별 추가 요구사항이 있으면 공통 정책에 덧붙여 함께 구현한다.
---

## 목표
- 모든 신규 화면(리스트/추가 팝업/수정 팝업)은 본 공통 정책을 "기본값"으로 사용한다.
- 화면별 정책이 공통 정책과 충돌할 경우, 화면별 정책을 우선 적용하되 충돌 지점을 명시한다.
- 공통 정책에 없는 추가 기능 요구사항이 제공되면, 공통 정책 + 추가 기능을 함께 반영한다.
- 추측하지 않는다. 문구/항목명/검색항목/컬럼 등 화면별 정의가 필요한 값은 입력 요구사항으로 분리한다.

---

## 적용 범위
- 리스트 화면
- 추가 팝업
- 수정 팝업
- 금액 입력/표시 포맷

---

## 공통 용어 규칙
- "항목명"은 화면별 도메인 단어로 치환한다.
  - 예: 임직원 추가/수정, 급여 추가/수정, 가족 추가/수정 등
- "리스트"는 해당 화면의 레코드 단위(행)이다.
- "최근 등록 순"은 기본 정렬이며, 화면별 정렬 요구가 있으면 그 요구를 우선 적용한다.

---

## 화면 생성 시 필수로 받아야 하는 화면별 정의(입력)
아래 값이 없으면 "확인 필요"로 표기하고 질문 목록을 생성한다.

### 리스트 화면
- 검색항목(예: 사번/성명/기간 등)
- 테이블 컬럼 정의(컬럼명/표시형식/정렬 가능 여부)
- 행 클릭 시 열리는 수정 팝업의 대상/키(예: id, 사번 등)
- 삭제의 의미(물리삭제/논리삭제/데이터만 삭제 등) 및 삭제 API 정책(부분 성공/전체 실패)
- **삭제 실패 케이스 여부** (선택삭제 시 실패 가능성이 있는지 확인 필수)

### 추가/수정 팝업
- 입력 필드 목록(필수/선택/타입/placeholder)
- 저장 버튼 라벨(필요 시)
- 유효성 규칙(길이/포맷/범위/중복 등)
- 저장 API 정책(성공/실패 시 처리)
- 삭제 가능 여부 조건(수정 팝업 삭제 기능용)
- 삭제 후 이동할 리스트 화면(리스트 화면이 없는 경우 명시 필요)

### 금액
- 통화 단위(기본: 원 KRW)
- 소수점 허용 여부(기본: 불가)
- 음수 허용 여부(기본: 불가)

# 1. 리스트 화면 공통 정책

## 1.0 뒤로가기(화면 이동)
- 하위 화면에서 상위 화면으로 돌아가는 뒤로가기 버튼이 있는 경우, **화살표 아이콘과 화면명(타이틀)을 하나의 클릭 영역으로 묶는다.**
- 화살표 아이콘만이 아닌, 화면명 텍스트를 클릭해도 이전 화면으로 이동한다.

## 1.1 검색영역
- 구성: 인풋박스 + 검색 버튼
- 검색항목은 각 화면별로 정의한다.
- (권장) Enter 입력 시 검색 실행. (화면별 요구가 있으면 우선)
- placeholder 문구 규칙
  - 검색항목이 1개일 경우: `XXX를 입력해주세요.`
  - 검색항목이 2개 이상일 경우: `XXX 또는 XXX, XXX를 입력해주세요.`
- 검색 인풋박스가 비어있는 상태(빈 값)에서 검색 시 아무 동작 하지 않음
- 공백만 입력 시 trim 처리 후 빈 값으로 간주하여 동일하게 아무 동작 하지 않음
- 검색 인풋박스 최대 입력 길이: 제한 없음
- 검색 실행 시 페이지네이션을 1페이지로 초기화한다.
- 검색 실행 시 체크박스 선택 상태를 모두 초기화한다.
- 페이지 이동 시 체크박스 선택 상태를 모두 초기화한다.

## 1.2 리스트 개수(카운트)
- 텍스트로 제공한다.
- 기본: 등록된 총 리스트 개수 제공
- 검색 상태: 검색 결과 개수 제공

## 1.3 리스트(테이블)
- 테이블 컴포넌트 사용
- 기본 정렬: 최근 등록 순
- 행 클릭 시 수정 팝업 노출
  - 선택 영역: 행 전체
- 가장 좌측에 "행 선택" 체크박스 제공(멀티 선택)
- 헤더에 전체 선택/해제 체크박스를 제공한다. (현재 페이지의 전체 항목을 선택/해제)

## 1.4 리스트 개수 선택(페이지 사이즈)
- 셀렉박스 제공
- **위치: 페이지네이션 영역 내부 (테이블 하단)**
- 옵션: 15개 / 30개 / 50개 / 100개
- 기본 선택: 30개
- 페이지 사이즈 변경 시 현재 보고 있던 데이터 위치를 유지한다.
  - 구체적으로, 변경 전 현재 페이지의 첫 번째 행이 변경 후에도 화면에 보이도록 페이지 번호를 조정한다.

## 1.5 리스트 다운 받기(엑셀 다운로드)
- 버튼 제공
- 다운로드 범위 규칙
  1) 검색하지 않았을 경우: 전체 리스트 엑셀 다운로드
  2) 검색했을 경우: 검색 결과 리스트 엑셀 다운로드
  3) 리스트 선택 후 버튼 클릭 시: 선택한 리스트만 엑셀 다운로드
- 엑셀 정렬(sorting)은 화면과 동일
- 데이터 0건일 경우 엑셀 다운로드 버튼 클릭 시 "다운로드할 데이터가 없습니다" 안내 메시지를 표시한다.
- 엑셀 파일 내 금액 컬럼은 숫자 형식(순수 숫자)으로 저장한다. (화면 표시와 달리 콤마, "원" 없이 숫자만)

## 1.6 선택 삭제
- 버튼 제공
- 좌측 체크박스 선택 시 활성화
- 데이터 0건일 경우 삭제 버튼을 비활성화한다. (선택 삭제 및 전체 삭제 모두 해당)
- 버튼 클릭 시 confirm 노출
  - 문구: "총 N개의 리스트를 삭제하시겠습니까?"
- 삭제 성공 시 토스트 1.5초 노출
  - 문구: "삭제가 완료되었습니다."
- 삭제 후 리스트/카운트 UI는 최신 상태로 갱신한다.

**※ 정책 작성 시 주의사항**
- **화면별 정책 작성 시 반드시 삭제 실패 케이스가 있는지 확인해야 합니다.**
- **삭제 실패 케이스가 없는 경우:** "해당 화면은 삭제 실패 케이스가 없습니다"라고 명시하고, 아래 삭제 실패 시 처리 정책은 작성하지 않습니다.
- **삭제 실패 케이스가 있는 경우:** 아래 삭제 실패 시 처리 정책을 적용합니다.

**삭제 실패 시 처리 정책** (삭제 실패 케이스가 있는 경우에만 적용)
  - 건별 트랜잭션 적용, 부분 성공을 허용한다.
  - 삭제 실패 리스트 팝업을 제공한다.
  - 삭제 실패한 리스트 개수를 제공한다.
  - 실패한 리스트 정보와 실패 사유를 제공한다.

# 3. 추가 팝업 공통 정책

## 3.1 닫기/이탈 방지(Confirm)
- 팝업을 벗어나려는 모든 행동(팝업 외 영역 클릭, ESC 키 입력, 닫기 버튼 클릭, 브라우저 뒤로가기 등) 발생 시 confirm 노출
- 단, 사용자가 아무런 입력도 하지 않은 초기 상태에서는 confirm 없이 팝업을 바로 닫는다.
- confirm 문구
  - "항목명 추가를 취소하시겠습니까?"
  - 여기서 "항목명"은 화면별 도메인 단어로 치환한다.

## 3.2 추가 완료 토스트
- 추가 성공 시 토스트 1.5초 노출
- 문구
  - "항목명 추가를 완료했습니다."
  - "항목명"은 화면별 도메인 단어로 치환
- 추가 완료 후 팝업을 자동으로 닫고 부모 화면(리스트)을 갱신한다.

## 3.3 유효성 검사(Validation)
- 검증 타이밍: blur / submit
- 에러 표시:
  - 컴포넌트 하단에 에러 문구 노출
  - 재입력 후 검증 통과 시 에러 문구 제거
  - 포커스 아웃 시 정상 입력으로 판단되면 에러 문구 제거(화면별 검증 규칙에 따름)

# 4. 수정 팝업 공통 정책

## 4.1 기본 동작
- 기능/검증/토스트/confirm 정책은 "추가 팝업"과 동일

## 4.2 수정 팝업 호출 키
- 수정 팝업 호출 시 전달하는 키값은 별도 DB PK(id)를 사용한다.

## 4.3 닫기/이탈 방지(Confirm)
- 팝업을 벗어나려는 모든 행동(팝업 외 영역 클릭, ESC 키 입력, 닫기 버튼 클릭, 브라우저 뒤로가기 등) 발생 시 confirm 노출
- 단, 기존 입력값에서 변경된 내용이 없는 경우에는 confirm 없이 팝업을 바로 닫는다.
- confirm 문구
  - "항목명 수정을 취소하시겠습니까?"
  - "항목명"은 화면별 도메인 단어로 치환한다.

## 4.4 수정 완료 토스트
- 수정 성공 시 토스트 1.5초 노출
- 문구
  - "항목명 수정을 완료했습니다."
  - "항목명"은 화면별 도메인 단어로 치환

## 4.5 삭제
- 수정 팝업 내 삭제 버튼을 제공한다.
- 삭제 버튼 클릭 시 confirm 팝업 노출
  - 데이터 수정 상태(변경사항 존재)와 무관하게, 수정 취소 confirm 없이 바로 삭제 confirm을 표시한다.
  - 문구: "항목명을 삭제하시겠습니까? 삭제한 정보는 복구할 수 없습니다."
  - "항목명"은 화면별 도메인 단어로 치환한다.
- confirm 확인 버튼 클릭 시
  1. 삭제 가능 여부를 확인한다. (삭제 가능 여부 조건은 각 화면별로 정의한다.)
  2. 삭제 가능한 경우: 삭제 처리 후 토스트 1.5초 노출 → 리스트 화면으로 이동 (초기 상태로 리셋: 검색 조건, 필터 등 초기화)
     - 토스트 문구: "삭제 완료되었습니다."
     - 리스트 화면이 없는 경우, 화면 정책 작성 시 이동 대상을 질문한다.
  3. 삭제 불가능한 경우: alert 팝업 노출 → 수정 팝업 유지
     - alert 내용: 삭제 실패 사유를 제공한다. (실패 사유는 서버 응답 기반)

# 5. 금액 공통 정책(입력/표시)

## 5.1 천 단위 콤마
- 금액은 1,000 단위마다 콤마(,)를 표시한다.
- 적용 범위
  - 사용자가 입력 중인 상태(typing)에도 콤마 적용
  - 값이 표시되는 모든 영역(테이블 셀, 팝업 입력값, 요약 영역 등)에도 콤마 적용

## 5.2 금액 정렬
- 테이블/리스트에서 금액 컬럼은 우측 정렬(text-align: right)을 적용한다.

## 5.3 금액 단위 표기
- 금액을 표시할 때는 항상 뒤에 '원'을 붙인다.
  - 예: 1,000,000원, 0원

## 5.4 금액 입력 허용 값(기본값)
- 기본: 0 이상의 정수만 허용
- 소수점 불가
- 음수 불가
- (화면별 예외가 있으면 해당 화면의 정책을 우선한다)

# 6. 서버 에러 공통 정책

## 6.1 서버 에러 발생 시
- alert 팝업 노출
- 문구: "서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."

## 6.2 네트워크 에러 발생 시
- 네트워크 오류(타임아웃, 연결 끊김 등) 발생 시 alert 팝업 노출
- 문구: "네트워크 연결을 확인해 주세요."

---

# 7. URL 파라미터 에러 공통 정책

## 7.1 잘못된 접근 시 에러 화면
- URL 파라미터 없이 직접 접근하거나 비정상 값이 전달된 경우 에러 화면을 표시한다.
- 에러 화면 구성:
  - 화면 중앙에 에러 아이콘 표시
  - 메시지: **"죄송합니다. 잘못된 접근이거나 요청하신 페이지를 찾을 수 없습니다."**
  - 화면별로 돌아갈 화면이 명확한 경우: "XXX으로 이동" 버튼 제공 (클릭 시 해당 화면으로 이동)

# 8. 셀렉박스 연도 옵션 공통 정책

## 8.1 연도 옵션 생성 기준
- 연도 셀렉박스의 옵션은 **화면/팝업 진입 시점 기준**으로 생성된다.
- 진입 후에는 자정을 넘어가더라도 연도 옵션이 동적으로 변경되지 않는다.
- 예: 2026년 12월 31일 23:59에 팝업 진입 시, 연도 옵션이 생성되고 00:00이 지나 2027년이 되어도 옵션은 변경되지 않는다.
