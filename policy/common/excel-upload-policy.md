# 🎯 목적

이 스킬은 모든 관리자 화면에서 사용하는 **공통 엑셀 업로드 정책 및 동작 규칙**을 정의한다.
각 화면에서 "엑셀 업로드 버튼이 존재한다"라고 정의하면 본 정책을 반드시 따른다.

---

# 🧩 UI 정책

## 1. 엑셀 업로드 버튼

버튼 클릭 시 **팝업**을 통해 2가지 옵션을 제공한다:

- 파일 올리기
- 엑셀 양식 받기

> 엑셀 업로드 버튼을 누르면 바로 파일 선택이 열리는 것이 아니라, **팝업이 먼저 노출**되고 팝업 내에서 "파일 올리기" 또는 "엑셀 양식 받기" 중 하나를 선택한다.

---

## 1-1. 파일 올리기

- 시스템 파일 선택 팝업 노출
- 엑셀 파일만 업로드 가능
  - 허용 확장자: `.xlsx`, `.xls`
- 파일은 1개만 업로드 가능
- 다중 선택 불가

---

## 1-2. 엑셀 양식 받기

- 해당 화면에서 정의한 양식 파일 즉시 다운로드
- 양식 파일은 각 화면에서 개별 관리한다

---

# 📊 업로드 처리 정책

## 2. 기본 검증

- 엑셀 컬럼 구조는 양식과 **완전히 일치해야 한다**
- 컬럼 불일치 시 → 전체 업로드 실패

---

## 2-1. 전체 실패 조건

- 컬럼 구조 불일치
- 양식 외 입력 데이터가 없음

### 에러 메시지

- "양식에 맞지 않는 엑셀파일입니다."
- "데이터가 없습니다. 데이터 입력 후 파일을 업로드해주세요."

---

# 📌 Row 단위 처리 정책

엑셀 구조가 정상일 경우:

- 각 row별 유효성 검증 수행
- 실패 row만 실패 처리
- 성공 row는 정상 저장

---

# 📊 업로드 결과 화면 정책

업로드 완료 후 반드시 다음을 표시한다:

- 총 N건
- 성공 건수
- 실패 건수
- 실패 row 목록

---

## 실패 row 표시 규칙

각 실패 row는 다음을 포함해야 한다:

- 대표 식별값 1개 (화면에서 정의)
- 실패 사유

대표 식별값이 정의되지 않으면 반드시 질문한다.

---

# 📥 실패 row 엑셀 다운로드 기능 (필수)

부분 실패가 발생한 경우 반드시 다음 기능을 제공한다:

## 1. 실패 데이터 다운로드 버튼 노출

- 버튼명: "실패 데이터 다운로드"
- 실패 row만 포함된 엑셀 파일 생성
- 파일명 예시 : 급여업로드_실패목록_20260204
- 업로드 일시  포함

## 2. 다운로드 파일 구성

- 기존 업로드 컬럼 유지
- 마지막 컬럼에 "실패사유" 컬럼 추가
- 각 실패 row에 실패 사유 입력
- 금액 컬럼은 숫자 형식(순수 숫자)으로 저장한다. (콤마, "원" 없이 숫자만)

## 3. 목적

- 사용자가 수정 후 재업로드 가능하도록 지원
- 재작업 최소화

---

# 💾 데이터 저장 정책

업로드 데이터는 각 화면에서 정의된 데이터와 연결된다.

각 화면은 반드시 아래 항목을 정의해야 한다:

1. 저장 모드 (신규 / 업데이트)
2. 업데이트 기준 컬럼 (업데이트 모드일 경우)
3. row 실패 사유 정의
4. 실패 리스트에 표시할 대표 식별값

정의되지 않으면 반드시 질문한다.

---

## 3-1. 신규 등록 모드

- 성공 row 전체 신규 저장

---

## 3-2. 업데이트 모드

- 특정 기준 컬럼으로 기존 데이터 검색
- 기준 컬럼 외 나머지 컬럼 업데이트

기준 컬럼 미정의 시 반드시 질문한다.

---

# 🔔 업로드 완료 처리

## 전체 성공 시

- 토스트 팝업 1.5초 노출
- 문구:
  "엑셀 업로드를 완료했습니다."
- 리스트 화면 새로고침
- 데이터 반영

---

## 부분 실패 시

- 토스트 미노출
- 성공/실패 집계 표시
- 실패 목록 표시
- 실패 데이터 다운로드 버튼 노출

---

# 🛑 제한 정책

- 여러 파일 업로드 불가
- 자동 컬럼 매핑 금지
- 컬럼 순서 무시 로직 금지
- 임의 컬럼 허용 금지
