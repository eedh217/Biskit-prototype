# 보수월액 변경신고 화면(HRIS_INS_04) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_INS_04
- **화면명**: 보수월액 변경신고
- **목적**: 4대보험(국민연금, 건강보험, 고용보험, 산재보험) 보수월액 변경 신고 및 관리
- **주요 기능**: 사업장 정보 입력, 직원 추가/삭제, 보험별 보수월액 변경 정보 입력, 임시 저장, 신고내역 조회, 신고하기
- **진입 경로**: 인사정보 모듈 > 4대보험 관리 > 보수월액 변경신고 메뉴 선택 시 진입
- **연관 화면**: 없음 (단일 화면에서 모든 작업 완료)

---

## 2. 요청한 화면 기능

### 2.1 사업장 정보

#### 2.1.1 입력 항목

**필수 입력 항목**:
- 사업장관리번호 *
- 명칭 *
- 우편번호 *
- 주소 *
- 상세주소 *
- 전화번호 *

**선택 입력 항목**:
- 단위사업장 명칭
- 영업소 명칭
- FAX번호

#### 2.1.2 입력 검증 규칙

**전화번호**:
- 허용 문자: 숫자(0-9), +, (, ), -
- 정규식: `/^[0-9+()-]*$/`
- 입력 예시: `02-1234-5678`, `+82-2-1234-5678`, `02)1234-5678`
- 허용되지 않은 문자 입력 시: 입력되지 않음 (실시간 검증)

**FAX번호**:
- 허용 문자: 숫자(0-9), -
- 정규식: `/^[0-9-]*$/`
- 입력 예시: `02-1234-5678`
- 허용되지 않은 문자 입력 시: 입력되지 않음 (실시간 검증)

#### 2.1.3 주소 검색 기능

- 우편번호/주소 필드는 직접 입력 불가 (readOnly)
- "주소 검색" 버튼 클릭 시 주소 검색 다이얼로그 오픈
- Daum 우편번호 서비스 API 사용
- 주소 선택 완료 시 우편번호와 주소 자동 입력
- 상세주소만 사용자가 직접 입력

#### 2.1.4 사업장 정보 저장 및 동기화

**저장 방식 (병합/Merge)**:
- 신고 완료 시 사업장 정보를 LocalStorage에 저장
- **기존 정보 유지**: 이미 저장된 사업장 정보가 있으면 기존 값과 병합(merge)
- **덮어쓰기 규칙**: 새로 입력한 필드만 업데이트, 입력하지 않은 필드는 기존 값 유지
- 예시:
  - 자격취득신고에서 모든 필드 입력 후 저장 (사업장관리번호, 명칭, 우편번호, 주소, 전화번호 등)
  - 보수월액 변경신고에서 일부 필드만 입력 후 저장 (사업장관리번호, 명칭, 전화번호만)
  - 결과: 우편번호, 주소 등 기존 정보는 유지되고, 입력한 3개 필드만 업데이트됨

**메뉴 간 동기화**:
- 다음 신고 시 자동으로 불러오기 (편의성 제공)
- **전체 연동**: 이 사업장 정보는 4대보험관리 메뉴 전체에서 공유됩니다
  - 자격 취득신고
  - 자격 상실신고
  - 피부양자 관리
  - 보수월액 변경신고
  - 기타 4대보험 관련 메뉴
- 한 곳에서 사업장 정보를 입력하면 다른 메뉴에서도 동일한 정보가 자동으로 표시됩니다

### 2.2 직원 목록

#### 2.2.1 직원 추가/삭제

**초기 상태**:
- 화면 진입 시 기본 직원 1명이 자동 추가됨

**추가 기능**:
- 버튼명: "추가하기"
- 위치: 카드 헤더 우측
- 클릭 시 직원 목록 최하단에 빈 직원 카드 1개 추가

**삭제 기능 (다중)**:
- 각 직원 카드 좌측에 체크박스 제공
- 1명 이상 선택 시 "삭제" 버튼 활성화
- 삭제 버튼 클릭 시 선택된 직원 카드 삭제
- 삭제 시 확인 메시지: "선택한 직원 N명을 삭제하시겠습니까?"
- 최소 1명은 유지 (전체 삭제 불가)

**삭제 기능 (단일)**:
- 각 직원 카드 우측 상단에 휴지통 아이콘 버튼 제공
- 클릭 시 해당 직원 카드 즉시 삭제 (confirm 없음)
- 최소 1명은 유지 (전체 삭제 불가)

### 2.3 직원 정보 입력

#### 2.3.1 기본 정보

**필수 입력 항목**:
- 성명 * (EmployeeCombobox 사용)
  - 직원 관리에 등록된 직원 목록에서 선택
  - 선택 시 주민등록번호/외국인등록번호, 변경년월 자동 입력
  - **중복 방지**: 이미 추가된 직원은 선택 목록에서 제외
  - **데이터 덮어쓰기**: 직원 선택 시 기존에 수동으로 입력한 데이터가 있어도 확인 없이 덮어씌웁니다
- 주민등록번호/외국인등록번호 * (자동 입력, 수정 가능)
  - 형식: 000000-0000000 (숫자만 입력, 하이픈 자동 추가)
  - 13자리 제한
- 변경년월 * (자동 입력, 수정 가능)
  - 형식: YYYY-MM
  - 직원 선택 시 현재 년월로 자동 입력
  - 월 입력 검증 적용
- 변경사유 *
  - 자유 입력 (텍스트 필드)
  - placeholder: "보수인상, 보수인하, 착오정정 등 입력"
  - 한글 입력 처리 (composing) 적용
  - 입력 예시:
    - 보수인상
    - 보수인하
    - 착오정정
    - 정기결정
    - 승급
    - 강임
    - 채용
    - 고용
    - 업무변경
    - 기타
    - 보수체계 변경
    - 휴직
    - 정년단축
    - 퇴직
    - 사업장 이전
    - 경영상태 변동
    - 고정급 외 임금 3개월 평균 누적액이 기준소득월액 대비 2배 이상 증감

#### 2.3.2 주민등록번호/외국인등록번호 포맷팅

- 숫자만 입력 가능
- 13자리 제한
- 6자리 이후 하이픈 자동 추가
- 형식: 000000-0000000

#### 2.3.3 변경년월 검증

- 형식: YYYY-MM
- 7자리 제한
- 연도 4자리 제한
- 월 01~12 범위 검증은 별도로 하지 않음 (자유 입력)

### 2.4.4 고용보험·산재보험 (선택 시 표시)

#### 조건부 표시 규칙

- 고용보험과 산재보험 중 **하나라도 체크**되어 있으면 섹션 전체 표시
- 고용보험 체크 해제 시: 고용보험 관련 필드 숨김
- 산재보험 체크 해제 시: 산재보험 관련 필드 숨김
- 둘 다 체크 해제 시: 섹션 전체 숨김

#### 필드 구성 - 체크박스 미체크 시 (기본)

**조건**:
- 고용보험과 산재보험이 **둘 다 체크**되어 있고
- "다른가요?" 체크박스가 **미체크** (기본 상태)

**표시 필드**:
- 1열: 변경 후 월평균보수 * (단일 필드)
  - 고용보험과 산재보험에 동일한 값 적용
  - 천 단위 콤마 자동 추가
  - 숫자만 입력 가능
  - placeholder: "0"
  - 입력 시 employmentChangedSalary와 workersCompChangedSalary에 동일한 값 자동 설정
- 2열: 체크박스
  - 라벨: "고용보험과 산재보험의\n변경 후 월평균보수가 다른가요?"
  - 기본값: 미체크 (false)
  - 정렬: 하단 정렬 (items-end), 중앙 정렬 (items-center)

#### 필드 구성 - 체크박스 체크 시

**조건**:
- 고용보험과 산재보험이 **둘 다 체크**되어 있고
- "다른가요?" 체크박스가 **체크** 상태

**표시 필드**:
- 1열: 고용보험 변경 후 월평균보수 * (고용보험 체크 시만)
  - 천 단위 콤마 자동 추가
  - 숫자만 입력 가능
  - placeholder: "0"
- 2열: 산재보험 변경 후 월평균보수 * (산재보험 체크 시만)
  - 천 단위 콤마 자동 추가
  - 숫자만 입력 가능
  - placeholder: "0"
- 3열: 체크박스
  - 라벨: "고용보험과 산재보험의\n변경 후 월평균보수가 다른가요?"
  - 체크 상태 유지
  - 정렬: 하단 정렬 (items-end), 중앙 정렬 (items-center)

#### 필드 구성 - 고용보험만 체크

**조건**:
- 고용보험만 체크, 산재보험 미체크

**표시 필드**:
- 변경 후 월평균보수 * (단일 필드)
  - employmentChangedSalary 값 사용
  - 체크박스 미표시

#### 필드 구성 - 산재보험만 체크

**조건**:
- 산재보험만 체크, 고용보험 미체크

**표시 필드**:
- 변경 후 월평균보수 * (단일 필드)
  - workersCompChangedSalary 값 사용
  - 체크박스 미표시

#### 검증 로직

**둘 다 체크 + 체크박스 미체크 (동일한 값)**:
- employmentChangedSalary만 검증
- 0보다 큰 값이어야 함

**둘 다 체크 + 체크박스 체크 (다른 값)**:
- employmentChangedSalary와 workersCompChangedSalary 개별 검증
- 각각 0보다 큰 값이어야 함

**고용보험만 체크**:
- employmentChangedSalary만 검증
- 0보다 큰 값이어야 함

**산재보험만 체크**:
- workersCompChangedSalary만 검증
- 0보다 큰 값이어야 함

### 2.6 임시 저장

#### 2.6.1 저장 기능

- 버튼명: "임시 저장"
- 위치: 하단 고정 영역 우측
- 클릭 시 현재 입력된 모든 데이터를 LocalStorage에 저장
- Storage Key: `biskit_insurance_salary_change_temp`
- 저장 성공 시 토스트 1.5초 노출 — 문구: "임시 저장되었습니다."
- 저장 시각 함께 저장 (ISO 형식)

#### 2.6.2 불러오기 기능

- 버튼명: "임시저장 불러오기"
- 위치: 하단 고정 영역 좌측
- 임시 저장 데이터가 있을 때만 표시
- **확인 절차**: 현재 작성 중인 데이터가 있을 경우 확인 메시지 표시
  - 메시지: "임시저장 데이터를 불러올 경우, 현재 입력한 데이터는 사라집니다. 불러오시겠습니까?"
  - 취소 클릭 시: 아무 동작 없음
  - 확인 클릭 시: 데이터 복원 진행
- 클릭 시 저장된 데이터를 폼에 복원
- 복원 성공 시 토스트 1.5초 노출 — 문구: "임시 저장 데이터를 불러왔습니다."

#### 2.6.3 삭제 기능

- 버튼명: "임시저장 삭제"
- 위치: 하단 고정 영역 좌측
- 임시 저장 데이터가 있을 때만 표시
- 클릭 시 임시 저장 데이터 삭제
- 삭제 성공 시 토스트 1.5초 노출 — 문구: "임시 저장 데이터를 삭제했습니다."
- 삭제 후 UI 변경:
  - "임시저장 불러오기" 버튼 미노출
  - "임시저장 삭제" 버튼 미노출
  - 저장 시각 표시 미노출

#### 2.6.4 저장 시각 표시

- 표시 위치: 하단 고정 영역 좌측
- 표시 형식: "마지막 저장: YYYY-MM-DD HH:mm:ss"
- 임시 저장 데이터가 있을 때만 표시

### 2.8 신고하기

#### 2.8.1 신고 확인 다이얼로그

**오픈 조건**:
- 필수값이 모두 입력된 경우에만 버튼 활성화
- 버튼 클릭 시 확인 다이얼로그 오픈

**다이얼로그 내용**:
- 제목: "보수월액 변경 신고 확인"
- 신고 대상: N명 (이름(사번) 형식으로 리스트 표시)
  - 스크롤 가능
- 사업장 정보:
  - "사업장: [주소 표시 규칙]"
    - 서울: 구까지 표시 (예: 서울 강남구)
    - 광역시: 구까지 표시 (예: 부산 해운대구, 인천 남동구)
    - 도 지역: 시까지 표시 (예: 경기도 성남시, 충청남도 천안시)
  - 공단 FAX 번호 입력:
    - "FAX번호 찾기" 버튼 제공 - 클릭 시 새탭에서 https://www.4insure.or.kr/pbiz/ntcn/selectInstBrofSrchView.do 열림
    - FAX번호 입력 필드 (숫자, -, (, ) 만 입력 가능)
    - 안내문구: "사업장 주소에 맞춰 공단 FAX 번호를 입력해주세요. 1개의 공단 FAX 번호만 입력해주시면 됩니다. (신고하려는 보험에 속하는 공단이어야 함)"
- 버튼:
  - 취소: 다이얼로그 닫기
  - 확인: 신고 처리

#### 2.8.2 신고 처리

**신고 완료 시**:
1. 신고내역 저장되고 신고내역 조회 다이얼로그에 내역 추가
   - 저장 데이터: 신고일시, 사업장 정보, 직원 목록
   - Storage Key: `biskit_insurance_salary_change_history`
2. 사업장 정보 저장 (다음 신고 시 사용)
3. 임시 저장 데이터 삭제
4. 토스트 1.5초 노출 — 문구: "보수월액 변경 신고가 완료되었습니다."
5. 폼 초기화 (기본 직원 1명 추가된 상태로)

### 2.9 신고내역 조회

#### 2.9.1 다이얼로그 오픈

- 버튼명: "신고내역 조회"
- 클릭 시 신고내역 조회 다이얼로그 오픈

#### 2.9.2 기간 검색

**기본 설정**:
- 다이얼로그 오픈 시 기본 기간: 최근 1개월

**날짜 선택**:
- 시작일 / 종료일 직접 선택 (date input)
- 날짜 변경 시 기간 버튼 선택 해제

**기간 버튼**:
- 당일 / 일주일 / 1개월 / 3개월 / 6개월 / 1년
- 클릭 시 해당 기간으로 날짜 자동 설정 (종료일은 당일로 지정)

#### 2.9.3 검색어 검색

- 검색 항목: 사번 또는 성명
- placeholder: "사번 또는 성명"
- Enter 입력하거나 검색 버튼 클릭 시 검색 실행
- 검색 시 첫 페이지로 이동

#### 2.9.4 신고내역 리스트

**컬럼 구성**:
- 신고일자 (YYYY-MM-DD HH:mm:ss)
- 사업장명
- 직원 (N명(이름1, 이름2, ...))
- 미리보기 버튼

**리스트 개수**:
- 표시 위치: 리스트 상단
- 표시 형식: "총 N건"

**페이지네이션**:
- 페이지당 10개 항목
- 스마트 페이지네이션 적용
  - 7페이지 이하: 모든 페이지 번호 표시
  - 7페이지 초과: 1 ... 5 6 7 ... 20 형식
  - 현재 페이지 앞뒤 1페이지씩 표시
  - 첫 페이지와 마지막 페이지 항상 표시

**미리보기 버튼**:
- 각 행에 "미리보기" 버튼 제공
- 클릭 시 새탭 열려서 해당 신고 내역 상세 제공

**데이터 없음**:
- 신고내역이 없을 경우: "신고내역이 없습니다." 문구 중앙 표시

## 4. 주요 특징

### 4.1 고용보험·산재보험 체크박스 로직

- **목적**: 고용보험과 산재보험의 변경 후 월평균보수가 동일한 경우와 다른 경우를 구분하여 입력 편의성 제공
- **기본 동작**: 체크박스 미체크 시 단일 필드로 두 보험에 동일한 값 적용
- **확장 동작**: 체크박스 체크 시 개별 필드로 분리하여 각각 다른 값 입력 가능
- **체크박스 위치**: 미체크 시 2열, 체크 시 3열로 이동하여 항상 표시 유지

### 4.2 변경년월 자동 입력

- 직원 선택 시 현재 년월(YYYY-MM)로 자동 입력
- 수동 수정 가능

### 4.3 국민연금 안내 문구

- 보건복지부장관이 고시하는 비율 이상 변동된 자만 신고하도록 안내
- 잘못된 신고 방지
