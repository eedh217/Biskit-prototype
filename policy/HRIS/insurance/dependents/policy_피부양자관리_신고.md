# 피부양자 관리 신고 화면(HRIS_INS_DEP_02) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_INS_DEP_02
- **화면명**: 피부양자 관리 신고
- **목적**: 건강보험 피부양자 취득 및 상실 신고 입력 및 제출
- **주요 기능**: 사업장 정보 수정, 직원 정보 입력, 피부양자 추가/삭제/선택, 피부양자 상세 정보 입력, 임시 저장, 신고하기
- **진입 경로**:
  - HRIS_INS_DEP_01 '신고하기' 버튼 클릭 → 신규 작성 모드
  - HRIS_INS_DEP_01 작성중 상태 행 클릭 → 편집 모드
- **연관 화면**:
  - HRIS_INS_DEP_01 (피부양자 관리 목록) — 뒤로가기 클릭 시 이동 / 신고 완료 시 이동
  - HRIS_INS_DEP_03 (피부양자 관리 상세) — 신고서 확인 버튼 클릭 시 새 탭으로 열림

---

## 2. 레이아웃 구조

- **상단**: PageHeader (뒤로가기 + 화면명 + 사업장 정보 수정 버튼)
- **직원 정보 영역**: PageHeader 아래 가로 배치 영역 (성명 / 주민등록번호 / 전화번호)
- **중앙**: 좌측 패널 + 우측 패널 (좌우 분할 레이아웃)
  - **좌측 패널**: 피부양자 목록 (이름/구분 리스트)
  - **우측 패널**: 선택된 피부양자의 상세 입력 폼
- **하단**: 고정 버튼 영역 (임시 저장 / 미리보기 / 신고하기)

---

## 3. 초기 데이터 로드

### 3.1 신규 작성 모드

- LocalStorage에 저장된 사업장 정보가 있으면 자동으로 불러와 표시
- 피부양자 1명이 빈 상태로 기본 추가됨

### 3.2 편집 모드

- 임시 저장된 신고 데이터를 불러와 사업장 정보, 직원 정보, 피부양자 목록 복원
- 복원 성공 시 토스트 노출 — 문구: "작성중인 신고 데이터를 불러왔습니다."
- 복원 실패 시 토스트 노출 — 문구: "데이터를 불러오는데 실패했습니다."

## 4. 사업장 정보

### 4.1 위치 및 노출 방식

- 사업장 정보는 메인 화면에 직접 표시하지 않음
- PageHeader 우측에 **"사업장 정보 수정"** 버튼 배치
- 버튼 클릭 시 사업장 정보 수정 다이얼로그 오픈

### 4.2 사업장 정보 수정 다이얼로그

#### 4.2.1 입력 항목

**필수 입력 항목**:
- 사업장관리번호 *
- 명칭 *
- 우편번호 * (주소 검색으로만 입력, 직접 입력 불가)
- 주소 * (주소 검색으로만 입력, 직접 입력 불가)
- 상세주소 *
- 전화번호 *

> 피부양자 관리 사업장 정보에는 단위사업장 명칭, 영업소 명칭 항목이 없음 (자격취득신고와 다름)

#### 4.2.2 입력 검증 규칙

**전화번호**:
- 허용 문자: 숫자(0-9), +, (, ), -
- 정규식: `/^[0-9+()-]*$/`
- 허용되지 않은 문자 입력 시 입력되지 않음 (실시간 검증)

#### 4.2.3 주소 검색

- 우편번호·주소 필드는 직접 입력 불가 (readOnly)
- "주소 검색" 버튼 클릭 시 주소 검색 다이얼로그 오픈 (Daum 우편번호 서비스)
- 주소 선택 완료 시 우편번호·주소 자동 입력
- 상세주소는 사용자가 직접 입력

#### 4.2.4 저장 버튼 활성화

- 필수 입력 항목이 모두 채워진 경우에만 "저장" 버튼 활성화
- 저장 클릭 시:
  1. 다이얼로그 닫히고 메인 화면의 사업장 정보 즉시 반영
  2. 사업장 정보 LocalStorage에 즉시 저장
  3. 토스트 노출 — 문구: "사업장 정보를 저장하였습니다."

#### 4.2.5 사업장 정보 동기화 (메뉴 간 공유)

- 사업장 정보 수정 다이얼로그에서 저장 시 LocalStorage에 즉시 저장
- 저장 방식: 병합(merge) — 새로 입력한 필드만 업데이트하고 기존 값은 유지
- 4대보험 관리 메뉴 전체에서 동일한 사업장 정보 공유 (자격취득신고, 자격상실신고, 피부양자관리, 보수월액변경 등)

## 5. 직원 정보 영역

직원 정보 영역은 PageHeader 아래, 좌우 패널 위에 가로로 배치된 독립 영역.

### 5.1 입력 항목

**필수 입력 항목**:
- **성명 \*** — EmployeeCombobox 사용
  - 직원 관리에 등록된 **재직 중인** 직원 목록에서 선택
  - 재직자 필터링: 퇴사일이 없거나 퇴사일이 오늘 이후인 직원만 표시
  - 선택 시 주민등록번호, 전화번호 자동 입력 (기존 입력값 덮어쓰기)
- **주민등록번호/외국인등록번호 \***
  - 자동 입력 후 직접 수정 가능
  - 숫자만 자동 추출, 13자리 제한, 6자리 입력 후 하이픈 자동 추가
  - 형식: 000000-0000000
- **전화번호(휴대폰번호) \***
  - 자동 입력 후 직접 수정 가능
  - 허용 문자: 숫자(0-9), 하이픈(-)
  - 최대 13자

## 6. 좌측 패널 — 피부양자 목록

### 6.1 헤더 영역

- 현재 추가된 피부양자 수 표시 (N명)
- "추가하기" 버튼 — 클릭 시 목록 하단에 빈 피부양자 1명 추가 및 해당 피부양자 선택 상태로 이동

### 6.2 피부양자 리스트

- 추가된 피부양자를 목록으로 표시
- 각 항목: 성명 + 구분(취득/상실) 표시 (이름 미입력 시 "(미입력)" 표시)
- 클릭 시 해당 피부양자가 선택 상태가 되어 우측 패널에 해당 피부양자의 입력 폼 표시
- 필수 입력값이 누락된 피부양자는 이름 왼쪽에 **빨간 점(●) 표시** (신고하기 버튼 클릭 후 showErrors 상태일 때만 표시)

### 6.3 삭제

- 각 항목 우측에 삭제 아이콘 버튼 제공
- **첫 번째 피부양자(피부양자 1)는 삭제 버튼 미표시** (최소 1명 유지)
- 삭제 시 confirm 없이 즉시 삭제
- **삭제 후 선택 상태 처리**:
  - 삭제된 피부양자 다음 피부양자가 선택 상태가 됨
  - 마지막 피부양자를 삭제한 경우 새로운 마지막 피부양자가 선택 상태가 됨

## 7. 우측 패널 — 피부양자 입력 폼

선택된 피부양자의 상세 정보를 입력하는 영역.

### 7.1 기본 정보

**필수 입력 항목**:

- **구분 \*** — 라디오 버튼
  - 취득 (기본값)
  - 상실
  - 구분 변경 시 취득부호/상실부호 목록 필터링

- **취득일자 / 상실일자 \*** (구분에 따라 라벨 변경)
  - 날짜 선택 UI (date picker)
  - 입력 형식: YYYY-MM-DD
  - 최대값: 2100-12-31

- **취득부호 / 상실부호 \*** (구분에 따라 라벨 및 목록 변경)
  - 구분이 "취득"인 경우:
    - 03 - (취득사유) 출생
    - 04-취득 - (취득사유) 외국국적을 상실하고 취득자로서 재외
    - 05 - (취득사유) 직장가입자 변경
    - 06 - (취득사유) 피부양자 상실
    - 07 - (취득사유) 직장가입자에서 변경
  - 구분이 "상실"인 경우:
    - 02 - (상실사유) 사망
    - 04-상실 - (상실사유) 외국국적을 상실하고자를 취득
    - 10 - (상실사유) 유공자를 긴급보장 배제신청
    - 14 - (상실사유) 지주불동등
    - 17 - (상실사유) 국적 상실
    - 18 - (상실사유) 의국민(지역국민)으로서 출국
    - 19 - (상실사유) 이민출국
    - 21 - (상실사유) 행방불명
    - 13 - (상실사유) 기타

- **피부양자 관계 \*** (구분이 "취득"일 때만 표시 및 필수)
  - 구분이 "취득"인 경우: 필드 표시 + 필수값 검증
  - 구분이 "상실"인 경우: 필드 미노출 + 필수값 검증 제외
  - 선택 항목:
    - 02 - 배우자, 03 - 부, 04 - 모, 05 - 자녀, 06 - 며느리, 08 - 사위
    - 10 - 시부, 11 - 시모, 12 - 장인, 13 - 장모, 18 - 양부, 19 - 양모
    - 1A - 외증손, 20 - 형, 22 - 제, 30 - 누이, 32 - 매, 3A - 언니
    - 40 - 손, 43 - 증손, 48 - 오빠, 49 - 배우자의 자녀, 50 - 조부, 51 - 조모
    - 6A - 누나, 76 - 외손, 78 - 외조부, 79 - 외조모
    - A7 - 기타, A8 - 처조부, A9 - 처조모, B6 - 처(사실혼), B7 - 남편(사실혼)

- **성명 \*** — 최대 30자

- **주민등록번호/외국인등록번호 \***
  - 형식: 000000-0000000 (숫자만 입력, 하이픈 자동 추가)
  - 13자리 제한

### 7.2 장애인·국가유공자 (체크박스)

- 체크 시 추가 필수 입력 항목 표시:
  - **종별부호 \***
    - 1 - 지체장애인, 2 - 뇌병변장애인, 3 - 시각장애인, 4 - 청각장애인, 5 - 언어장애인
    - 6 - 지적장애인, 7 - 자폐성장애인, 8 - 정신장애인, 9 - 신장장애인, 10 - 심장장애인
    - 11 - 호흡기장애인, 12 - 간 장애인, 13 - 안면장애인, 14 - 장루ㆍ요루장애인
    - 15 - 뇌전증장애인, 19 - 국가유공자 등 또는 보훈보상대상자
  - **등록일 \***
    - 날짜 선택 UI (date picker)
    - 입력 형식: YYYY-MM-DD
    - 최대값: 2100-12-31

### 7.3 외국인 (체크박스)

- 체크 시 추가 필수 입력 항목 표시:
  - **국적 \*** — ISO 3166-1 alpha-2 국가 코드 사용
  - **체류자격 \***
    - 41 - 기여동포(F-4-1), A1 - 외교, A2 - 공무, A3 - 협정
    - B1 - 사증면제, B2 - 관광통과, C0 - 재외국민
    - C1-1 - 일시취재, C1-2 - 유학(재외동포), C2 - 단기상용, C3 - 단기종합, C4 - 단기취업
    - C7 - 난민등, C9 - 유학(재외국인), D1 - 문화예술, D2 - 유학(외국인), D3 - 산업연수생
    - D4 - 일반연수, D5 - 취재, D6 - 종교, D7 - 주재, D8 - 기업투자, D9 - 무역경영, D10 - 구직
    - E1 - 교수, E2 - 회화지도, E3 - 연구, E4 - 기술지도, E5 - 전문직원, E6 - 예술흥행
    - E7 - 특정활동, E8 - 연수취업, E9 - 비전문취업, E10 - 선원취업
    - F1 - 방문동거, F2 - 거주, F3 - 동반, F4 - 외국국적동포, F5 - 영주, F6 - 결혼이민
    - G1 - 기타(체류자격), H1 - 관광취업, H2 - 방문취업, T1 - 관광상륙, ZZ - 기타
  - **체류기간 \*** — 최대 30자

## 8. 필수값 검증

### 8.1 필수값 표시

- 모든 필수 입력 항목 라벨 옆에 `*` 표시
- 조건부 필수 항목도 조건 충족 시 `*` 표시

### 8.2 에러 표시 방식

- **신고하기 버튼은 항상 활성화 상태**
- 필수값 미입력 상태에서 신고하기 버튼 클릭 시:
  - 신고 확인 다이얼로그 열리지 않음
  - 현재 우측 패널의 미입력 필수 항목 **테두리 빨간색으로 표시**
  - 좌측 피부양자 목록에서 미입력 필수 항목이 있는 피부양자 이름 왼쪽에 **빨간 점(●) 표시**
    - 해당 피부양자의 필수 항목이 모두 입력되면 빨간 점 자동 제거

### 8.3 필수값 검증 범위

**사업장 정보**:
- 사업장관리번호, 명칭, 우편번호, 주소, 상세주소, 전화번호

**직원 정보**:
- 성명 (employeeId), 주민등록번호/외국인등록번호, 전화번호(휴대폰번호)

**피부양자** (1명 이상 존재해야 함, 각 피부양자별):
- 구분, 취득일자 또는 상실일자, 취득부호 또는 상실부호
- 피부양자 관계 (구분이 "취득"인 경우만 필수)
- 성명, 주민등록번호/외국인등록번호
- 장애인·국가유공자 체크 시: 종별부호, 등록일
- 외국인 체크 시: 국적, 체류자격, 체류기간

## 9. 하단 고정 버튼 영역

화면 하단에 고정되어 항상 표시.

### 9.1 임시 저장

- 버튼명: "임시 저장"
- 클릭 시 현재까지 입력한 내용을 '작성중' 상태로 저장
  - 처음 임시 저장하는 경우: 신고 건이 새로 생성되고 URL이 편집 모드로 변경됨
  - 이전에 임시 저장한 적 있는 경우: 기존 저장 내용을 현재 입력 내용으로 덮어씀
- 직원이 선택되지 않은 경우 저장 불가 — 토스트: "직원을 선택해주세요."
- 저장 성공 시 토스트 — 문구: "임시 저장되었습니다."
- 저장 실패 시 토스트 — 문구: "임시 저장에 실패했습니다."

### 9.2 미리보기

- 버튼명: "미리보기"
- 버튼 클릭 시 신고서 pdf를 새 탭으로 열기 (현재 미구현)

### 9.3 신고하기

- 버튼명: "신고하기"
- 항상 활성화 상태
- 클릭 시 필수값 검증 수행 (8.2 참고)
- 모든 필수값 입력 완료 시 신고 확인 다이얼로그 오픈

## 10. 신고 확인 다이얼로그

### 10.1 다이얼로그 구성

- **제목**: "피부양자 관리 신고 확인"
- **사업장 정보 요약**:
  - 사업장관리번호
  - 명칭
  - 주소
  - 전화번호
- **신고 대상**:
  - 직원: 성명(사번)
  - 피부양자: N명 (피부양자 성명 - 취득/상실, ... 형식 리스트)
- **사업장 FAX 번호** 입력 필드
  - 허용 문자: 숫자, -, (, )
- **공단 FAX 번호** 입력 필드
  - 허용 문자: 숫자, -, (, )
  - "FAX번호 찾기" 버튼 — 클릭 시 새 탭으로 `https://www.4insure.or.kr/pbiz/ntcn/selectInstBrofSrchView.do` 열림
  - 안내 문구: "사업장 주소에 맞춰 공단 FAX 번호를 입력해주세요. 1개의 공단 FAX 번호만 입력해주시면 됩니다."

### 10.2 FAX 번호 자동 저장 및 불러오기

- 신고 완료 시 입력한 FAX 번호 LocalStorage에 저장
- 다이얼로그 열릴 때 이전에 입력한 FAX 번호가 있으면 자동으로 불러옴
- **메뉴 간 공유**: 4대보험 관리 메뉴 전체에서 동일한 FAX 번호 공유

### 10.3 확인 버튼 활성화

- 사업장 FAX 번호와 공단 FAX 번호 모두 입력된 경우에만 "확인" 버튼 활성화

### 10.4 신고 처리

확인 버튼 클릭 시:
1. 신고 데이터를 완료(completed) 상태로 저장 (FAX 번호 포함하여 formData에 저장)
2. 사업장 정보 LocalStorage에 저장 (다음 신고 시 자동 불러오기)
3. FAX 번호 LocalStorage에 저장
4. 토스트 노출 — 문구: "피부양자 관리 신고가 완료되었습니다."
5. HRIS_INS_DEP_01 피부양자 관리 목록 화면으로 이동

## 11. 페이지 이탈 방지

신고 화면에서 데이터 입력 후 실수로 페이지를 이탈하는 것을 방지하는 기능.

### 11.1 이탈 방지 활성화 조건

- **직원·피부양자 정보가 변경된 경우**에만 이탈 방지 활성화
- 사업장 정보만 수정한 경우에는 이탈 방지 미적용 (confirm 없이 즉시 이탈)
- 화면 진입 후 아무 입력도 하지 않은 상태에서는 이탈 방지 미적용 (confirm 없이 즉시 이탈)

### 11.2 이탈 시나리오별 동작

| 이탈 유형 | 동작 |
|----------|------|
| 새로고침 / 탭 닫기 | 브라우저 기본 confirm 노출 |
| 브라우저 뒤로가기 / 앞으로가기 | confirm 노출 — 문구: "피부양자 신고를 취소하시겠습니까?" |
| 사이드바 / 메뉴 클릭 | confirm 노출 — 문구: "피부양자 신고를 취소하시겠습니까?" |
| PageHeader 뒤로가기 버튼 클릭 | confirm 노출 — 문구: "피부양자 신고를 취소하시겠습니까?" |

### 11.3 confirm 결과 처리

- **확인**: 이탈 진행 (해당 경로로 이동)
- **취소**: 이탈 취소, 현재 화면 유지

### 11.4 이탈 방지 해제

- 신고하기(신고 완료) 시: 이탈 방지 자동 해제 후 목록 화면으로 이동
- 임시 저장 시: 이탈 방지 유지 (저장 후에도 계속 입력 가능)
