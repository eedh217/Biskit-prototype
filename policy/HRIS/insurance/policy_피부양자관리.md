# 피부양자 관리 화면(HRIS_INS_03) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_INS_03
- **화면명**: 피부양자 관리
- **목적**: 건강보험 피부양자 취득 및 상실 신고 관리
- **주요 기능**: 사업장 정보 입력, 직원 정보 입력, 피부양자 취득/상실 관리, 임시 저장, 신고내역 조회, 신고하기
- **진입 경로**: 인사정보 모듈 > 4대보험 > 피부양자 관리 메뉴 선택 시 진입
- **연관 화면**: 없음 (단일 화면에서 모든 작업 완료)

---

## 2. 요청한 화면 기능

### 2.1 사업장 정보

#### 2.1.1 입력 항목

**필수 입력 항목**:
- 사업장관리번호 *
- 명칭 *
- 전화번호 *

#### 2.1.2 입력 검증 규칙

**전화번호**:
- 허용 문자: 숫자(0-9), +, (, ), -
- 정규식: `/^[0-9+()-]*$/`
- 입력 예시: `02-1234-5678`, `+82-2-1234-5678`, `02)1234-5678`
- 허용되지 않은 문자 입력 시: 입력되지 않음 (실시간 검증)

#### 2.1.3 사업장 정보 저장 및 동기화

**저장 방식 (병합/Merge)**:
- 신고 완료 시 사업장 정보를 LocalStorage에 저장
- **기존 정보 유지**: 이미 저장된 사업장 정보가 있으면 기존 값과 병합(merge)
- **덮어쓰기 규칙**: 새로 입력한 필드만 업데이트, 입력하지 않은 필드는 기존 값 유지
- 예시:
  - 자격취득신고에서 모든 필드 입력 후 저장 (사업장관리번호, 명칭, 우편번호, 주소, 전화번호 등)
  - 피부양자관리에서 일부 필드만 입력 후 저장 (사업장관리번호, 명칭, 전화번호만)
  - 결과: 우편번호, 주소 등 기존 정보는 유지되고, 입력한 3개 필드만 업데이트됨

**메뉴 간 동기화**:
- 다음 신고 시 자동으로 불러오기 (편의성 제공)
- **전체 연동**: 이 사업장 정보는 4대보험관리 메뉴 전체에서 공유됩니다
  - 자격 취득신고
  - 자격 상실신고
  - 피부양자 관리
  - 보수월액 변경
  - 기타 4대보험 관련 메뉴
- 한 곳에서 사업장 정보를 입력하면 다른 메뉴에서도 동일한 정보가 자동으로 표시됩니다

### 2.2 직원 정보

#### 2.2.1 입력 항목

**필수 입력 항목**:
- 성명 * (EmployeeCombobox 사용)
  - 직원 관리에 등록된 **재직 중인** 직원 목록에서 선택
  - **재직자 필터링**: 퇴사일이 없거나 퇴사일이 오늘보다 이후인 직원만 표시
  - 퇴사자는 목록에 표시되지 않음
  - 선택 시 주민등록번호, 전화번호 자동 입력
  - **데이터 덮어쓰기**: 직원 선택 시 기존에 수동으로 입력한 데이터가 있어도 확인 없이 덮어씌웁니다
- 주민등록번호/외국인등록번호 *
  - **자동 입력**: 성명에서 직원 선택 시 자동으로 입력됨
  - **직접 입력 가능**: 자동 입력 후에도 직접 수정 가능
  - **입력 방식** (자격취득신고와 동일):
    - 숫자만 자동 추출 (하이픈 등 다른 문자 제거)
    - 13자리로 자동 제한
    - 6자리 입력 후 자동으로 하이픈 추가
    - 형식: 000000-0000000
    - 입력 예시: `9001011234567` → 자동 변환 → `900101-1234567`
- 전화번호(휴대폰번호) *
  - **자동 입력**: 성명에서 직원 선택 시 자동으로 입력됨
  - **직접 입력 가능**: 자동 입력 후에도 직접 수정 가능
  - **입력 규칙**:
    - 허용 문자: 숫자(0-9), 하이픈(-)
    - 최대 13자
    - 형식 예시: 010-1234-5678

### 2.3 피부양자 목록

#### 2.3.1 피부양자 추가/삭제

**초기 상태**:
- 화면 진입 시 기본 피부양자 1명이 자동 추가됨

**추가 기능**:
- 버튼명: "피부양자 추가"
- 클릭 시 피부양자 목록 최하단에 빈 피부양자 카드 1개 추가

**삭제 기능**:
- 각 피부양자 카드 우측 상단에 휴지통 아이콘 버튼 제공
- 클릭 시 해당 피부양자 카드 즉시 삭제 (confirm 없음)
- 최소 1명은 유지 (전체 삭제 불가)
- 첫 번째 피부양자(피부양자1)의 삭제 버튼은 표시하지 않음

#### 2.3.2 피부양자 정보 입력

1. **구분 * (라디오 버튼)**

### 2.4 필수값 검증 및 버튼 활성화

#### 2.4.1 필수값 표시

- 모든 필수 입력 항목 라벨 옆에 `*` 표시
- 조건부 필수 항목도 조건 충족 시 `*` 표시

#### 2.4.2 필수값 검증 규칙

**사업장 정보**:
- 사업장관리번호
- 명칭
- 전화번호

**직원 정보**:
- 성명 (employeeId)
- 주민등록번호/외국인등록번호
- 전화번호(휴대폰번호)

**피부양자**:
- 피부양자가 1명 이상 존재해야 함
- 각 피부양자별 필수 항목:
  - 구분에 따라:
    - 취득인 경우: 피부양자 관계 필수
    - 상실인 경우: 피부양자 관계 미노출 (필수 아님)
  - 성명
  - 주민등록번호/외국인등록번호
  - 취득일자/상실일자
  - 취득부호/상실부호
  - 장애인·국가유공자 체크 시:
    - 종별부호
    - 등록일
  - 외국인 체크 시:
    - 국적
    - 체류자격
    - 체류기간

#### 2.4.3 신고하기 버튼 활성화

- 필수값이 모두 입력되지 않으면: 버튼 **비활성화**
- 필수값이 모두 입력되면: 버튼 **활성화**

### 2.5 임시 저장

#### 2.5.1 저장 기능

- 버튼명: "임시 저장"
- 위치: 하단 고정 영역 우측
- 클릭 시 현재 입력된 모든 데이터를 LocalStorage에 저장
- 저장 성공 시 토스트 1.5초 노출 — 문구: "임시 저장되었습니다."
- 저장 시각 함께 저장 (ISO 형식)

#### 2.5.2 불러오기 기능

- 버튼명: "임시저장 불러오기"
- 위치: 하단 고정 영역 좌측
- 임시 저장 데이터가 있을 때만 표시
- **확인 절차**: 현재 작성 중인 데이터가 있을 경우 확인 메시지 표시
  - 메시지: "임시저장 데이터를 불러올 경우, 현재 입력한 데이터는 사라집니다. 불러오시겠습니까?"
  - 취소 클릭 시: 아무 동작 없음
  - 확인 클릭 시: 데이터 복원 진행
- 클릭 시 저장된 데이터를 폼에 복원
- 복원 성공 시 토스트 1.5초 노출 — 문구: "임시 저장 데이터를 불러왔습니다."

#### 2.5.3 삭제 기능

- 버튼명: "임시저장 삭제"
- 위치: 하단 고정 영역 좌측
- 임시 저장 데이터가 있을 때만 표시
- 클릭 시 임시 저장 데이터 삭제
- 삭제 성공 시 토스트 1.5초 노출 — 문구: "임시 저장 데이터를 삭제했습니다."
- 삭제 후 UI 변경:
  - "임시저장 불러오기" 버튼 미노출
  - "임시저장 삭제" 버튼 미노출
  - 저장 시각 표시 미노출

#### 2.5.4 저장 시각 표시

- 표시 위치: 하단 고정 영역 좌측
- 표시 형식: "마지막 저장: YYYY-MM-DD HH:mm:ss"
- 임시 저장 데이터가 있을 때만 표시

- 신고 대상:
  - 직원: 성명(사번)
  - 피부양자: N명 (피부양자1 성명 - 취득/상실, 피부양자2 성명 - 취득/상실, ...)
    - 예시: "피부양자: 2명 (홍아들 - 취득, 김아내 - 상실)"
- 사업장 정보:
  - "사업장: [주소 표시 규칙]"
    - 서울: 구까지 표시 (예: 서울 강남구)
    - 광역시: 구까지 표시 (예: 부산 해운대구, 인천 남동구)
    - 도 지역: 시까지 표시 (예: 경기도 성남시, 충청남도 천안시)
    - 주소 정보가 없는 경우: 명칭 표시
  - 공단 FAX 번호 입력:
    - "FAX번호 찾기" 버튼 제공 - 클릭 시 새탭에서 https://www.4insure.or.kr/pbiz/ntcn/selectInstBrofSrchView.do 열림
    - FAX번호 입력 필드 (숫자, -, (, ) 만 입력 가능)
    - 안내문구: "사업장 주소에 맞춰 공단 FAX 번호를 입력해주세요. 1개의 공단 FAX 번호만 입력해주시면 됩니다. (신고하려는 보험에 속하는 공단이어야 함)"
- 버튼:
  - 취소: 다이얼로그 닫기
  - 확인: 신고 처리

#### 2.7.2 신고 처리

**신고 완료 시**:
1. 신고내역 저장되고 신고내역 조회 다이얼로그에 내역 추가
   - 저장 데이터: 신고일시, 사업장 정보, 직원 정보, 피부양자 목록
2. 임시 저장 데이터 삭제
3. 토스트 1.5초 노출 — 문구: "피부양자 관리 신고가 완료되었습니다."
4. 폼 초기화 (기본 피부양자 1명 추가된 상태로)

### 2.8 신고내역 조회

#### 2.8.1 다이얼로그 오픈

- 버튼명: "신고내역 조회"
- 위치: PageHeader 우측
- 클릭 시 신고내역 조회 다이얼로그 오픈

#### 2.8.2 기간 검색

**기본 설정**:
- 다이얼로그 오픈 시 기본 기간: 최근 1개월

**날짜 선택**:
- 시작일 / 종료일 직접 선택 (date input)
- 날짜 변경 시 기간 버튼 선택 해제

**기간 버튼**:
- 당일 / 일주일 / 1개월 / 3개월 / 6개월 / 1년
- 클릭 시 해당 기간으로 날짜 자동 설정 (종료일은 당일로 지정)

#### 2.8.3 검색어 검색

- 검색 항목: 사번 또는 성명
- placeholder: "사번 또는 성명"
- Enter 입력하거나 검색 버튼 클릭 시 검색 실행
- 검색 시 첫 페이지로 이동

#### 2.8.4 신고내역 리스트

**컬럼 구성**:
- 신고일자 (YYYY-MM-DD HH:mm:ss)
- 사업장명
- 직원 (이름만 표시, 1명이므로)
- 피부양자 (N명(이름1, 이름2, ...))
  - 예시: "2명(김아들, 김길동)"
- 미리보기 버튼

**리스트 개수**:
- 표시 위치: 리스트 상단
- 표시 형식: "총 N건"

**페이지네이션**:
- 페이지당 10개 항목
- 스마트 페이지네이션 적용
  - 7페이지 이하: 모든 페이지 번호 표시
  - 7페이지 초과: 1 ... 5 6 7 ... 20 형식
  - 현재 페이지 앞뒤 1페이지씩 표시
  - 첫 페이지와 마지막 페이지 항상 표시

**미리보기 버튼**:
- 각 행에 "미리보기" 버튼 제공
- 클릭 시 새탭 열려서 해당 신고 내역 상세 제공 (현재 비활성 상태)

**데이터 없음**:
- 신고내역이 없을 경우: "신고내역이 없습니다." 문구 중앙 표시
