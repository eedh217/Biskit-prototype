# 보수총액신고 화면(HRIS_INS_05) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_INS_05
- **화면명**: 보수총액신고
- **목적**: 4대보험(국민연금, 건강보험, 고용보험, 산재보험) 연간 보수총액 신고 및 관리
- **주요 기능**: 사업장 정보 입력, 직원 추가/삭제, 보험별 보수총액 정보 입력, 임시 저장, 신고내역 조회, 신고하기
- **진입 경로**: 인사정보 모듈 > 4대보험 관리 > 보수총액신고 메뉴 선택 시 진입
- **연관 화면**: 없음 (단일 화면에서 모든 작업 완료)

## 2. 요청한 화면 기능

### 2.1 사업장 정보

#### 2.1.1 입력 항목

**필수 입력 항목**:
- 사업장관리번호 *
- 명칭 *
- 우편번호 *
- 주소 *
- 상세주소 *
- 전화번호 *

**선택 입력 항목**:
- FAX번호

#### 2.1.2 입력 검증 규칙

**전화번호**:
- 허용 문자: 숫자(0-9), +, (, ), -
- 정규식: `/^[0-9+()-]*$/`
- 입력 예시: `02-1234-5678`, `+82-2-1234-5678`, `02)1234-5678`
- 허용되지 않은 문자 입력 시: 입력되지 않음 (실시간 검증)

**FAX번호**:
- 허용 문자: 숫자(0-9), -
- 정규식: `/^[0-9-]*$/`
- 입력 예시: `02-1234-5678`
- 허용되지 않은 문자 입력 시: 입력되지 않음 (실시간 검증)

#### 2.1.3 주소 검색 기능

- 우편번호/주소 필드는 직접 입력 불가 (readOnly)
- "주소 검색" 버튼 클릭 시 주소 검색 다이얼로그 오픈
- Daum 우편번호 서비스 API 사용
- 주소 선택 완료 시 우편번호와 주소 자동 입력
- 상세주소만 사용자가 직접 입력

#### 2.1.4 사업장 정보 저장 및 동기화

**저장 방식 (병합/Merge)**:
- 신고 완료 시 사업장 정보를 LocalStorage에 저장
- **기존 정보 유지**: 이미 저장된 사업장 정보가 있으면 기존 값과 병합(merge)
- **덮어쓰기 규칙**: 새로 입력한 필드만 업데이트, 입력하지 않은 필드는 기존 값 유지

**메뉴 간 동기화**:
- 다음 신고 시 자동으로 불러오기 (편의성 제공)
- **전체 연동**: 이 사업장 정보는 4대보험관리 메뉴 전체에서 공유됩니다
  - 자격 취득신고
  - 자격 상실신고
  - 피부양자 관리
  - 보수월액 변경신고
  - 보수총액신고
- 한 곳에서 사업장 정보를 입력하면 다른 메뉴에서도 동일한 정보가 자동으로 표시됩니다

### 2.2 직원 목록

#### 2.2.1 직원 추가/삭제

**초기 상태**:
- 화면 진입 시 기본 직원 1명이 자동 추가됨

**추가 기능**:
- 버튼명: "추가하기"
- 위치: 카드 헤더 우측
- 클릭 시 직원 목록 최하단에 빈 직원 카드 1개 추가

**삭제 기능**:
- 각 직원 카드 우측 상단에 휴지통 아이콘 버튼 제공
- 클릭 시 해당 직원 카드 즉시 삭제 (confirm 없음)
- 최소 1명은 유지 (전체 삭제 불가)
- 첫 번째 직원(직원1)의 삭제 버튼은 표시하지 않음

#### 2.2.2 직원 정보 입력

**기본 정보**:

**필수 입력 항목**:
- 성명 * (EmployeeCombobox 사용)
  - 직원 관리에 등록된 직원 목록에서 선택
  - 선택 시 주민등록번호, 신고년도, 자격취득일 자동 입력
- 주민등록번호/외국인등록번호 *
  - 형식: 000000-0000000 (숫자만 입력, 하이픈 자동 추가)
  - 13자리 제한
- 신고년도 * (text 입력, 4자리 연도)
  - 기본값: 전년도 (예: 2026년이면 2025)
  - 직원 선택 시 자동 설정
- 자격취득일 * (date 입력)
  - 직원 선택 시 입사일(joinDate)을 YYYY-MM-DD 형식으로 자동 설정
  - YYYYMMDD → YYYY-MM-DD 변환 적용

**자동 입력 규칙**:
- 성명 선택 시:
  - 주민등록번호: 직원의 `residentRegistrationNumber` 또는 `foreignerRegistrationNumber`
  - 신고년도: 현재 연도 - 1 (전년도)
  - 자격취득일: 직원의 `joinDate`를 YYYY-MM-DD 형식으로 변환
    - 예: joinDate가 "20240315"이면 "2024-03-15"로 설정

### 2.4 국민연금

#### 2.4.1 입력 항목 (2행 레이아웃)

**첫 번째 행 (4열)**:
- 신고유형 (선택박스)
  - 옵션: 정기신고, 수정신고, 소급신고
  - 기본값 없음
- YYYY년 근무시작일
  - 형식: MM-DD (월-일만 입력)
  - 예시: 01-15, 06-30
  - YYYY는 기본정보의 신고년도와 연동
  - placeholder: "MM-DD"
  - maxLength: 5
- YYYY년 보수총액 *
  - 숫자만 입력, 천 단위 콤마 자동 표시
  - YYYY는 기본정보의 신고년도와 연동

**두 번째 행 (4열)**:
- 1열: 체크박스 "신고년도 기간 중 휴직하셨나요?"
  - 체크 시 2열에 "실제 휴직일수" 입력 필드 표시
- 2열: 실제 휴직일수 (조건부 표시)
  - 휴직 체크박스 체크 시에만 표시
  - 숫자만 입력 가능
  - 최대값: 365
  - blur 시 검증: 365 초과 시 에러 메시지 "신고년도 기간 중 휴직일수를 입력해주세요."

#### 2.4.2 날짜 검증

**근무시작일 (MM-DD) 검증**:
- 형식: MM-DD (5자리, 하이픈 포함)
- 월(MM): 01~12
- 일(DD): 해당 월의 유효한 일자 (1~31, 월별 다름)
  - 1,3,5,7,8,10,12월: 1~31일
  - 4,6,9,11월: 1~30일
  - 2월: 1~29일 (윤년 고려)
- blur 시 검증 실패 시 인풋박스 하단에 에러 메시지 표시: "유효한 날짜가 아닙니다."

## 6. 참고 사항

### 6.1 자격취득신고와의 차이점

- **자격취득신고**: 신규 입사자의 보험 가입
- **보수총액신고**: 기존 가입자의 연간 보수총액 신고

### 6.2 신고년도 연동

- 모든 "YYYY년" 표시는 기본정보의 "신고년도" 필드 값과 연동
- 기본값: 전년도 (현재 연도 - 1)
- 예시: 2026년이면 기본 신고년도는 2025

### 6.3 동적 Label

- Label의 "YYYY년"은 직원별 신고년도에 따라 동적으로 변경
- 예: 신고년도가 "2025"이면 "2025년 보수총액"으로 표시
- 신고년도가 없으면 "YYYY년 보수총액"으로 표시
