# 보수월액 변경신고 화면(HRIS_INS_CHG_02) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_INS_CHG_02
- **화면명**: 보수월액 변경신고
- **목적**: 4대보험(국민연금, 건강보험, 고용보험, 산재보험) 보수월액 변경 신고 입력 및 제출
- **주요 기능**: 사업장 정보 수정, 직장가입자 추가/삭제/선택, 보험별 상세 정보 입력, 첨부서류 업로드, 임시 저장, 신고하기
- **진입 경로**:
  - HRIS_INS_CHG_01 '신고하기' 버튼 클릭 → 신규 작성 모드
  - HRIS_INS_CHG_01 작성중 상태 행 클릭 → 편집 모드
- **연관 화면**:
  - HRIS_INS_CHG_01 (보수월액 변경신고 목록) — 뒤로가기 클릭 시 이동
  - HRIS_INS_CHG_03 (보수월액 변경신고 상세) — 신고 완료 시 이동

---

## 2. 레이아웃 구조

- **상단**: PageHeader (뒤로가기 + 화면명 + 사업장 정보 수정 버튼)
- **중앙**: 좌측 패널 + 우측 패널 (좌우 분할 레이아웃)
  - **좌측 패널**: 직장가입자 목록 (이름/사번 리스트)
  - **우측 패널**: 선택된 직장가입자의 상세 입력 폼
- **하단**: 고정 버튼 영역 (임시 저장 / 미리보기 / 신고하기)

---

## 3. 초기 데이터 로드

### 3.1 신규 작성 모드

- LocalStorage에 저장된 사업장 정보가 있으면 자동으로 불러와 표시
- 직장가입자 1명이 빈 상태로 기본 추가됨

### 3.2 편집 모드

- 임시 저장된 신고 데이터를 불러와 사업장 정보 및 직장가입자 목록 복원
- 복원 성공 시 토스트 노출 — 문구: "작성중인 신고 데이터를 불러왔습니다."
- 복원 실패 시 토스트 노출 — 문구: "데이터를 불러오는데 실패했습니다."

## 4. 사업장 정보

### 4.1 위치 및 노출 방식

- 사업장 정보는 메인 화면에 직접 표시하지 않음
- PageHeader 우측에 **"사업장 정보 수정"** 버튼 배치
- 버튼 클릭 시 사업장 정보 수정 다이얼로그 오픈

### 4.2 사업장 정보 수정 다이얼로그

#### 4.2.1 입력 항목

**필수 입력 항목**:
- 사업장관리번호 *
- 명칭 *
- 우편번호 * (주소 검색으로만 입력, 직접 입력 불가)
- 주소 * (주소 검색으로만 입력, 직접 입력 불가)
- 상세주소 *
- 전화번호 *

**선택 입력 항목**:
- 전자우편주소
- 휴대폰번호

#### 4.2.2 입력 검증 규칙

**전화번호 / 휴대폰번호**:
- 허용 문자: 숫자(0-9), +, (, ), -
- 정규식: `/^[0-9+()-]*$/`
- 허용되지 않은 문자 입력 시 입력되지 않음 (실시간 검증)

#### 4.2.3 주소 검색

- 우편번호·주소 필드는 직접 입력 불가 (readOnly)
- "주소 검색" 버튼 클릭 시 주소 검색 다이얼로그 오픈 (Daum 우편번호 서비스)
- 주소 선택 완료 시 우편번호·주소 자동 입력
- 상세주소는 사용자가 직접 입력

#### 4.2.4 저장 버튼 활성화

- 필수 입력 항목이 모두 채워진 경우에만 "저장" 버튼 활성화
- 저장 클릭 시:
  1. 다이얼로그 닫히고 메인 화면의 사업장 정보 즉시 반영
  2. 사업장 정보 LocalStorage에 즉시 저장
  3. 토스트 노출 — 문구: "사업장 정보를 저장하였습니다."

#### 4.2.5 사업장 정보 동기화 (메뉴 간 공유)

- 사업장 정보 수정 다이얼로그에서 저장 시 LocalStorage에 즉시 저장
- 저장 방식: 병합(merge) — 새로 입력한 필드만 업데이트하고 기존 값은 유지
- 4대보험 관리 메뉴 전체에서 동일한 사업장 정보 공유 (자격취득신고, 자격상실신고, 피부양자관리, 보수월액변경 등)

## 5. 좌측 패널 — 직장가입자 목록

### 5.1 헤더 영역

- 현재 추가된 직장가입자 수 표시 (N명)
- "추가하기" 버튼 — 클릭 시 리스트 하단에 빈 직장가입자 1명 추가 및 해당 직원 선택 상태로 이동

### 5.2 직장가입자 리스트

- 추가된 직장가입자를 목록으로 표시
- 각 항목: 성명 + 사번 표시 (이름 미입력 시 "(미입력)" 표시)
- 클릭 시 해당 직장가입자가 선택 상태가 되어 우측 패널에 해당 직원의 입력 폼 표시

### 5.3 삭제

- 각 항목 우측에 삭제 아이콘 버튼 제공
- **첫 번째 직장가입자(직원 1)는 삭제 버튼 미표시** (최소 1명 유지)
- 삭제 시 confirm 없이 즉시 삭제
- **삭제 후 선택 상태 처리**:
  - 삭제된 직원 다음 직원이 선택 상태가 됨
  - 마지막 직원을 삭제한 경우 새로운 마지막 직원이 선택 상태가 됨

---

## 6. 우측 패널 — 직장가입자 입력 폼

선택된 직장가입자의 상세 정보를 입력하는 영역.

### 6.1 기본 정보

**필수 입력 항목**:

- **성명 \*** — EmployeeCombobox 사용
  - 직원 관리에 등록된 직원 목록에서 선택
  - 선택 시 주민등록번호, 보수 변경 월(당월), 현재 기준소득월액 자동 입력 (기존 입력값 덮어쓰기)
    - 보수 변경 월: 선택 시점의 당월(YYYY-MM) 자동 입력
    - 현재 기준소득월액: 직원의 과세 급여 합산액 자동 입력 (연봉 직원 기준)
  - 중복 방지: 다른 직장가입자에서 이미 선택된 직원은 목록에서 제외
- **주민등록번호/외국인등록번호 \*** — 자동 입력, 수정 가능
  - 형식: 000000-0000000 (숫자만 입력, 하이픈 자동 추가)
  - 13자리 제한
- **보수 변경 월 \*** — 자동 입력, 수정 가능
  - 형식: YYYY-MM (월 선택 UI)
  - 최대값: 2100-12
- **변경사유 \***
  - 텍스트 입력
  - placeholder: "보수인상, 보수인하, 착오정정 등"

### 6.2 보험 신고

- 국민연금 / 건강보험 / 고용보험 / 산재보험 — 체크박스로 선택/해제
- **초기 상태**: 4개 보험 모두 체크됨
- **데이터 초기화**: 보험 체크 해제 시 해당 보험의 입력 데이터 모두 초기화
  - 고용보험·산재보험: 두 항목 모두 체크 해제될 때만 관련 데이터 초기화

### 6.3 국민연금 (국민연금 체크 시 표시)

안내 문구: "기준소득월액 대비 실제 소득이 보건복지부장관이 고시하는 비율 이상으로 변동된 자만 신고해주세요."

**필수 입력 항목**:
- 현재 기준소득월액 \*
  - 양수만 입력 가능, 천 단위 콤마 자동 표시
  - 직원 선택 시 과세 급여 합산액 자동 입력
- 변경 기준소득월액 \*
  - 양수만 입력 가능, 천 단위 콤마 자동 표시

### 6.4 건강보험 (건강보험 체크 시 표시)

**필수 입력 항목**:
- 변경 후 보수월액 \*
  - 양수만 입력 가능, 천 단위 콤마 자동 표시

### 6.5 고용보험·산재보험 (고용보험 또는 산재보험 체크 시 표시)

**필수 입력 항목**:

- **변경 후 월평균보수 \***
  - 양수만 입력 가능, 천 단위 콤마 자동 표시
  - 고용보험과 산재보험을 둘 다 체크하고 월평균보수가 동일한 경우: 필드 1개 (고용·산재 공통 입력, 동일한 값 자동 적용)
  - 고용보험과 산재보험의 월평균보수가 다른 경우: 필드 2개 (고용보험 / 산재보험 각각 입력)

**선택 입력 항목**:
- **고용보험과 산재보험의 변경 후 월평균보수가 다른가요?** (체크박스)
  - 표시 조건: 고용보험과 산재보험을 **둘 다** 체크한 경우에만 표시
  - 체크 해제 시: 변경 후 월평균보수 필드 1개 (고용·산재 공통)
  - 체크 시: 고용보험 변경 후 월평균보수 / 산재보험 변경 후 월평균보수 필드 각각 표시

### 6.6 첨부서류

우측 패널 입력 폼의 가장 하단에 위치하는 파일 첨부 영역.

#### 6.6.1 파일 첨부 규격

- **허용 확장자**: PDF, JPG, JPEG, PNG
- **파일당 최대 크기**: 5MB
- **직장가입자 1인당 최대 첨부 파일 수**: 5개

#### 6.6.2 UI 구성

- 영역 상단에 "첨부서류" 제목과 "파일 추가" 버튼 나란히 배치
- 안내 문구: "PDF, JPG, JPEG, PNG · 파일당 최대 5MB · 최대 5개"
- 첨부된 파일이 없는 경우: 점선 테두리 영역에 "첨부된 파일이 없습니다." 텍스트 표시
- 첨부된 파일이 있는 경우: 파일 목록 표시
  - 각 항목: 파일명 + 파일 크기(KB 단위) + 삭제 버튼
- 최대 파일 수(5개) 도달 시 "파일 추가" 버튼 비활성화

#### 6.6.3 파일 추가

- "파일 추가" 버튼 클릭 시 파일 선택 다이얼로그 오픈
- 다중 파일 선택 가능
- 파일 유효성 검사:
  - 허용되지 않는 확장자 → 토스트: "{파일명}: 지원하지 않는 파일 형식입니다. (PDF, JPG, JPEG, PNG만 가능)"
  - 5MB 초과 → 토스트: "{파일명}: 파일 크기가 5MB를 초과합니다."
  - 최대 5개 초과 → 토스트: "최대 5개까지 첨부 가능합니다."

#### 6.6.4 파일 삭제

- 각 파일 항목 우측 삭제 버튼 클릭 시 즉시 삭제 (confirm 없음)

#### 6.6.5 직원 추가/삭제 시 동기화

- 직장가입자 추가 시: 해당 직원의 첨부 파일 배열 빈 배열로 초기화
- 직장가입자 삭제 시: 해당 직원의 첨부 파일 배열도 함께 삭제

## 7. 필수값 검증

### 7.1 필수값 표시

- 모든 필수 입력 항목 라벨 옆에 `*` 표시

### 7.2 에러 표시 방식

- **신고하기 버튼은 항상 활성화 상태**
- 필수값 미입력 상태에서 신고하기 버튼 클릭 시:
  - 신고 확인 다이얼로그 열리지 않음
  - 현재 우측 패널에 표시된 직장가입자의 미입력 필수 항목 **테두리 빨간색으로 표시**
  - 다른 직장가입자 선택 시 해당 직장가입자의 미입력 항목도 동일하게 표시
  - 좌측 직장가입자 목록에서 미입력 필수 항목이 있는 직장가입자의 이름 왼쪽에 **빨간 점(●) 표시**
    - 해당 직장가입자의 필수 항목이 모두 입력되면 빨간 점 자동 제거

### 7.3 필수값 검증 범위

**사업장 정보**:
- 사업장관리번호, 명칭, 우편번호, 주소, 상세주소, 전화번호

**직장가입자** (1명 이상 존재해야 함, 각 직장가입자별):
- 성명(employeeId), 주민등록번호/외국인등록번호, 보수 변경 월, 변경사유
- 국민연금 체크 시: 현재 기준소득월액, 변경 기준소득월액
- 건강보험 체크 시: 변경 후 보수월액
- 고용보험 또는 산재보험 체크 시:
  - 고용·산재 공통인 경우: 변경 후 월평균보수
  - 고용·산재 보수가 다른 경우: 고용보험 변경 후 월평균보수, 산재보험 변경 후 월평균보수

## 8. 하단 고정 버튼 영역

화면 하단에 고정되어 항상 표시.

### 8.1 임시 저장

- 버튼명: "임시 저장"
- 클릭 시 현재까지 입력한 내용을 '작성중' 상태로 저장
  - 처음 임시 저장하는 경우: 신고 건이 새로 생성되며, URL이 편집 모드로 변경됨
  - 이전에 임시 저장한 적 있는 경우: 기존 저장 내용을 현재 입력 내용으로 덮어씀
- 직원이 한 명도 선택되지 않은 경우 저장 불가 — 토스트: "직원을 선택해주세요."
- 저장 성공 시 토스트 — 문구: "임시 저장되었습니다."
- 저장 실패 시 토스트 — 문구: "임시 저장에 실패했습니다."

### 8.2 미리보기

- 버튼명: "미리보기"
- 버튼 클릭 시 신고서 pdf를 새 탭으로 열기 (현재 미구현)

### 8.3 신고하기

- 버튼명: "신고하기"
- 항상 활성화 상태
- 클릭 시 필수값 검증 수행 (7.2 참고)
- 모든 필수값 입력 완료 시 신고 확인 다이얼로그 오픈

## 9. 신고 확인 다이얼로그

### 9.1 다이얼로그 구성

- **제목**: "보수월액 변경 신고 확인"
- **사업장 정보 요약**:
  - 사업장관리번호
  - 명칭
  - 주소
  - 전화번호
  - 전자우편주소 — **값이 입력된 경우에만 표시**
  - 휴대폰번호 — **값이 입력된 경우에만 표시**
- **신고 대상**: N명 (이름(사번) 형식 리스트, 스크롤 가능)
- **사업장 FAX 번호** 입력 필드
  - 허용 문자: 숫자, -, (, )
- **공단 FAX 번호** 입력 필드
  - 허용 문자: 숫자, -, (, )
  - "FAX번호 찾기" 버튼 — 클릭 시 새 탭으로 `https://www.4insure.or.kr/pbiz/ntcn/selectInstBrofSrchView.do` 열림
  - 안내 문구: "사업장 주소에 맞춰 공단 FAX 번호를 입력해주세요. 1개의 공단 FAX 번호만 입력해주시면 됩니다. (신고하려는 보험에 속하는 공단이어야 함)"

### 9.2 FAX 번호 자동 저장 및 불러오기

- 신고 완료 시 입력한 FAX 번호 LocalStorage에 저장
- 다이얼로그 열릴 때 이전에 입력한 FAX 번호가 있으면 자동으로 불러옴
- **메뉴 간 공유**: 저장된 FAX 번호는 자격취득신고 / 자격상실신고 / 보수월액변경신고 메뉴에서 공유됨
  - 어느 메뉴에서 입력·저장하든 세 메뉴에서 동일한 FAX 번호가 자동으로 불러와짐
  - 피부양자관리 메뉴는 FAX 번호를 별도로 관리하며 공유되지 않음

### 9.3 확인 버튼 활성화

- 사업장 FAX 번호와 공단 FAX 번호 모두 입력된 경우에만 "확인" 버튼 활성화

### 9.4 신고 처리

확인 버튼 클릭 시:
1. 신고 데이터를 완료(completed) 상태로 저장 (FAX 번호 포함하여 formData에 저장)
2. 사업장 정보 LocalStorage에 저장 (다음 신고 시 자동 불러오기)
3. FAX 번호 LocalStorage에 저장
4. 토스트 노출 — 문구: "보수월액 변경 신고가 완료되었습니다."
5. HRIS_INS_CHG_03 보수월액 변경신고 상세 화면으로 이동

## 10. 페이지 이탈 방지

신고 화면에서 데이터 입력 후 실수로 페이지를 이탈하는 것을 방지하는 기능.

### 10.1 이탈 방지 활성화 조건

- **직장가입자 정보가 변경된 경우**에만 이탈 방지 활성화
- 사업장 정보만 수정한 경우에는 이탈 방지 미적용 (confirm 없이 즉시 이탈)
- 화면 진입 후 아무 입력도 하지 않은 상태에서는 이탈 방지 미적용 (confirm 없이 즉시 이탈)

### 10.2 이탈 시나리오별 동작

| 이탈 유형 | 동작 |
|----------|------|
| 새로고침 / 탭 닫기 | 브라우저 기본 confirm 노출 |
| 브라우저 뒤로가기 / 앞으로가기 | confirm 노출 — 문구: "보수월액 변경신고를 취소하시겠습니까?" |
| 사이드바 / 메뉴 클릭 | confirm 노출 — 문구: "보수월액 변경신고를 취소하시겠습니까?" |
| PageHeader 뒤로가기 버튼 클릭 | confirm 노출 — 문구: "보수월액 변경신고를 취소하시겠습니까?" |

### 10.3 confirm 결과 처리

- **확인**: 이탈 진행 (해당 경로로 이동)
- **취소**: 이탈 취소, 현재 화면 유지

### 10.4 이탈 방지 해제

- 신고하기(신고 완료) 시: 이탈 방지 자동 해제 후 상세 화면으로 이동
- 임시 저장 시: 이탈 방지 유지 (저장 후에도 계속 입력 가능)
