# 보수월액 변경신고 목록 화면(HRIS_INS_CHG_01) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_INS_CHG_01
- **화면명**: 보수월액 변경신고 목록
- **목적**: 작성된 보수월액 변경신고 내역을 조회하고 신규 신고를 시작하는 화면
- **주요 기능**: 이름/사번 검색, 신고 리스트 조회, 신고 삭제(작성중), 신고서 확인(신고완료), 팩스 재발송(발송실패), 신규 신고 진입, 수신여부 확인
- **진입 경로**: 인사정보 모듈 > 4대보험 관리 > 보수월액 변경신고 메뉴 선택 시 진입
- **연관 화면**:
  - HRIS_INS_CHG_02 (보수월액 변경신고) — '신고하기' 버튼 클릭 시 이동 / 작성중 행 클릭 시 이동
  - HRIS_INS_CHG_03 (보수월액 변경신고 상세) — 신고완료 행 클릭 시 이동
---

## 2. 화면 기능

### 2.1 검색 영역

- 구성: 인풋박스 + 검색 버튼
- 검색 항목: **직원 이름 또는 사번**
- placeholder 문구: `이름 또는 사번을 입력해주세요.`
- Enter 입력 시 검색 실행
- 검색 방식: **포함 검색** — 입력된 검색어가 신고에 포함된 직원의 이름 또는 사번에 포함되는 경우 조회
  - 신고 1건에 여러 직원이 포함될 수 있으며, 그 중 한 명이라도 검색어와 일치하면 해당 신고가 노출됨
- 검색 인풋박스 빈 값/공백 처리:
  - 빈 값 또는 공백만 입력 시 전체 조회
  - 앞뒤 공백은 자동 trim 처리
- 검색 결과가 없을 경우 테이블 영역에 "검색 결과가 없습니다." 문구를 중앙에 표시
- 검색 시 하단 리스트 및 리스트 개수를 검색 결과 기준으로 갱신
- **검색 실행 후 입력 필드 포커스 유지**

### 2.2 리스트 개수(카운트)

- 기본: 등록된 총 신고 건수 표시
- 검색 상태: 검색 결과 건수 표시
- **표시 형식**: "N개" (예: "5개")
- 위치: 리스트 상단 좌측

### 2.3 리스트(테이블)

- 기본 정렬: **상태 우선 → 동일 상태 내 최신순**
  - 1순위: 작성중이 신고완료보다 상단에 노출
  - 2순위: 동일 상태 내에서는 신고일자 기준 최신순 정렬
- 행 클릭 시 상태에 따라 다른 화면으로 이동 (2.7 행 클릭 참고)
- 데이터가 없을 경우 테이블 영역에 "데이터가 없습니다." 문구를 중앙에 표시
- 컬럼 정의:

| 컬럼명 | 표시 형식 | 정렬 | 비고 |
|--------|----------|------|------|
| 신고일자 | 날짜+시간 (yyyy-MM-dd HH:mm:ss) | 좌측 | 작성중 상태는 임시저장 시각 |
| 직원 | N명(이름, 이름, ...) | 좌측 | 이름 나열, 30자 초과 시 말줄임(...) 처리, 전체 이름 호버 툴팁 표시 |
| 상태 | 배지 | 좌측 | 작성중: 노란색 배지 / 신고완료: 초록색 배지 |
| 팩스발송 | 배지 (+버튼) | 좌측 | 작성중 상태: "-" 표시 / 신고완료: 발송성공(파란색) · 발송실패(빨간색 + 재발송 버튼) · 발송중(회색) |
| 액션 | 버튼 | 우측 | 상태에 따라 다른 버튼 표시 (2.4, 2.5 참고) |

### 2.4 삭제 버튼 (작성중 상태)

- **노출 조건**: 상태가 '작성중'인 행에만 삭제 버튼 노출
- 버튼 클릭 시 confirm 노출
  - 문구: "작성중인 신고를 삭제하시겠습니까?"
- **삭제 방식**: 물리삭제 (LocalStorage에서 완전 삭제)
- 삭제 성공 시 토스트 노출 — 문구: "삭제 완료되었습니다."
- 삭제 후 리스트/카운트를 최신 상태로 갱신

### 2.5 신고서 확인 버튼 (신고완료 상태)

- **노출 조건**: 상태가 '신고완료'인 행에만 '신고서 확인' 버튼 노출
- 버튼 클릭 시 신고서 pdf를 **새 탭**으로 열기

### 2.6 재발송 버튼 (발송실패 상태)

- **노출 조건**: 팩스발송 상태가 '발송실패'인 행에만 발송실패 배지 옆에 재발송 버튼 노출
- 버튼 클릭 시 confirm 노출
  - 문구: "신고서를 재발송하시겠습니까?\n내용을 수정하실 경우, 신고하기 버튼을 눌러 재작성 후 신고해주세요."
- 확인 클릭 시 재발송 처리 후 토스트 노출 — 문구: "재발송되었습니다."

### 2.7 행 클릭

- **작성중** 상태 행 클릭 시 → HRIS_INS_CHG_02 '보수월액 변경신고' 화면으로 이동 (편집 모드)
- **신고완료** 상태 행 클릭 시 → HRIS_INS_CHG_03 '보수월액 변경신고 상세' 화면으로 이동

### 2.8 신고하기 버튼

- **위치**: 리스트 상단 우측
- 버튼명: "신고하기"
- 버튼 클릭 시 HRIS_INS_CHG_02 '보수월액 변경신고' 화면으로 이동 (신규 작성 모드)

### 2.9 수신여부 확인 버튼

- **위치**: 신고하기 버튼 좌측
- 버튼명: "수신여부 확인"
- 버튼 클릭 시 팝업(다이얼로그) 노출
- **팝업 구성**:
  - 제목: "수신여부 확인"
  - 안내 문구: "팩스를 보낸 공단을 클릭하여 수신여부를 확인해주세요."
  - 공단별 외부 링크 (클릭 시 새 탭으로 열림):

| 공단명 | URL |
|--------|-----|
