# 휴가 승인 화면(HRIS_VAC_02) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_VAC_02
- **화면명**: 휴가 승인
- **목적**: 직원들이 신청한 휴가를 승인하거나 반려하는 화면
- **주요 기능**: 대기중인 휴가 신청 목록 조회, 승인, 반려 (사유 입력)
- **진입 경로**: 인사정보 모듈 > 연차/휴가관리 메뉴 > 휴가 승인 탭 선택 시 진입
- **연관 화면**:
  - HRIS_VAC_02_DLG (반려 사유 입력 Dialog) — 반려 버튼 클릭 시 팝업

---

## 2. 요청한 화면 기능

### 2.1 헤더 영역

- **제목**: "대기중인 휴가 신청"
- **대기 건수 Badge**:
  - 표시 형식: "N건" (예: "5건")
- **일괄승인 버튼**:
  - 버튼명: "일괄승인"
  - **노출 조건**: 대기중인 휴가 신청이 1건 이상 있을 경우에만 노출 (0건이면 버튼 미노출)
  - 클릭 시: Popover 메뉴 노출
    - **전체 신청내역(N)**: 현재 목록의 모든 신청 승인
    - **선택 신청내역(N)**: 체크박스로 선택한 신청만 승인
  - 선택 신청내역 클릭 시:
    - 선택된 항목이 없으면 alert 노출: "선택된 신청내역이 없습니다. 목록 내 좌측 신청내역을 선택해주세요."
    - 선택된 항목이 있으면 선택된 신청들을 일괄 승인
  - 승인 완료 시:
    - 토스트 노출: "총 N개의 휴가 승인을 완료했습니다."
    - 리스트 즉시 새로고침

### 2.2 리스트(테이블)

- **조회 대상**: 승인 대기중인 휴가 신청만 (상태='대기중')
- **정렬**: 신청일시 기준 (먼저 신청한 순)
- **체크박스**: 제공함
  - 헤더: 전체 선택/해제 체크박스
  - 각 행: 개별 선택 체크박스
  - 선택된 항목은 일괄승인 기능에서 활용
- **행 클릭**: 제공하지 않음
- 컬럼 정의:

| 컬럼명 | 표시형식 | 비고 |
|--------|---------|------|
| 체크박스 | Checkbox | 일괄 승인용 선택 |
| 신청일 | YYYY. M. D. | |
| 신청자 | 텍스트 | 직원명 표시|
| 유형 | 텍스트 | 휴가 유형명 (연차, 경조휴가, 병가 등) |
| 기간 | YYYY-MM-DD (단일) 또는<br>YYYY-MM-DD ~ YYYY-MM-DD (기간) | 시작일과 종료일이 같으면 단일 날짜만 표시,<br>다르면 날짜 범위 표시 |
| 일수 | 사용 단위별 표시 | • 일 단위: "N일" (예: "2일")<br>• 오전반차: "오전반차(0.5일)"<br>• 오후반차: "오후반차(0.5일)"<br>• 시간 단위: "N시간(실제일수)" (예: "3시간(0.375일)") |
| 사유 | 텍스트 | hover 시 전체 내용 표시 (title 속성) |
| 액션 | 버튼 2개 | 승인 버튼 + 반려 버튼 |

### 2.3 액션 버튼

#### 승인 버튼
- **버튼명**: "승인"
- **클릭 시**:
  1. confirm 노출
     - 문구: "휴가 신청을 승인하시겠습니까?"
  2. 확인 선택 시:
     - 해당 신청의 상태를 '승인'으로 변경
     - 신청자의 연차 잔액에서 일수 자동 차감
     - 토스트 노출: "휴가 신청이 승인되었습니다."
     - 리스트 즉시 새로고침 (해당 신청은 목록에서 제거됨)
  3. 취소 선택 시: 아무 동작 없음

#### 반려 버튼
- **버튼명**: "반려"
- **클릭 시**: 반려 사유 입력 Dialog 열림 (HRIS_VAC_02_DLG)

### 2.4 반려 사유 입력 Dialog (HRIS_VAC_02_DLG)

#### Dialog 헤더
- **제목**: "휴가 신청 반려"

#### Dialog 내용
- **반려 사유 입력**:
  - 입력 컴포넌트: Textarea (4줄)
  - placeholder: "반려 사유를 입력하세요"
  - **필수 입력**: 빈 값 또는 공백만 입력 시 제출 불가

#### Dialog 액션
1. **취소 버튼**
   - 버튼명: "취소"
   - 클릭 시: Dialog 닫기 (입력 내용 초기화)

2. **반려 버튼**
   - 버튼명: "반려"
   - **활성화 조건**: 반려 사유가 입력된 경우에만 활성화 (빈 값 또는 공백만 있으면 비활성화)
   - 클릭 시:
     - **검증**: 반려 사유가 빈 값이거나 공백만 있는 경우
       - 토스트 노출: "입력 오류 - 반려 사유를 입력해주세요."
       - Dialog 유지
     - **정상 처리**: 반려 사유가 입력된 경우
       - 해당 신청의 상태를 '반려'로 변경
       - 반려 사유 저장
       - 연차 잔액 변동 없음
       - 토스트 노출: "휴가 신청이 반려되었습니다."
       - Dialog 닫기
       - 리스트 즉시 새로고침 (해당 신청은 목록에서 제거됨)

### 2.5 빈 화면 처리

#### 로딩 중
- 표시 문구: "로딩 중..."
- 스타일: 기본 텍스트

#### 데이터 0건
- 표시 문구: "대기중인 휴가 신청이 없습니다."

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
|--------------|--------------|------------|------|
| 1.1 검색 | 검색 기능 | **미적용** | 대기중인 신청만 표시, 검색 불필요 |
| 1.2 리스트 개수 | 카운트 표시 | **부분 적용** | Badge로 건수 표시 (N건) |
| 1.3 리스트(테이블) | 체크박스, 행 클릭 | **부분 적용** | 체크박스 제공 (일괄 승인용), 행 클릭 없음 |
| 1.4 페이지 사이즈 | 페이지네이션 | **미적용** | 전체 신청 한 화면에 표시 (페이징 없음) |
| 1.5 엑셀 다운로드 | 다운로드 기능 | **미적용** | 엑셀 다운로드 기능 없음 |
| 1.6 선택 삭제 | 삭제 기능 | **미적용** | 삭제 기능 없음 (일괄 승인 기능 제공) |

> 본 화면은 승인 업무에 특화된 화면으로, 대기중인 신청만 조회하며 승인/반려 액션에 집중한다. 일괄 승인을 위한 체크박스를 제공하며, 검색, 페이징, 엑셀 다운로드 등의 부가 기능은 제공하지 않는다.
