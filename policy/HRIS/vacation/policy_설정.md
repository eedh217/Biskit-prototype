# 설정 화면(HRIS_VAC_03) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_VAC_03
- **화면명**: 설정
- **목적**: 연차/휴가 시간 단위 사용 및 연차 발생 기준, 계산 방법을 설정하는 화면
- **주요 기능**: 시간 단위 사용 설정, 연차 발생 기준 설정 (입사일 기준 / 연초 기준), 비례부여 소수점 처리 설정, 전체 직원 연차 재계산
- **진입 경로**: 인사정보 모듈 > 연차/휴가관리 메뉴 > 설정 탭 선택 시 진입

---

## 2. 요청한 화면 기능

### 2.1 연차/휴가 시간 단위 사용 Card

#### 시간 단위 사용 설정

**Switch 컴포넌트**
- **제목**: "연차/휴가 시간 단위 사용"
- **설명**: "임직원이 연차/휴가를 시간 단위로 사용할 수 있는지 설정합니다."
- **기본값**: on (활성화)
- **표시 텍스트**:
  - 제목: "시간 단위 사용"
  - 설명: "활성화 시, 임직원은 1시간 단위로 연차/휴가를 사용할 수 있습니다. (1시간 = 0.125일)"

#### 주의 안내 영역

- **표시 조건**: 시간 단위 사용 설정을 변경했을 때만 표시 (변경 사항이 없으면 숨김)
- **제목**: "주의"
- **내용**:
  - **on으로 변경 시**:
    ```
    on으로 변경 시, 임직원은 연차/휴가를 시간 단위로 사용할 수 있으며, 1시간 사용 시 0.125일이 차감됩니다.

    또한, 시간 단위 사용이 설정된 휴가 종류도 시간 단위 사용이 허용됩니다.
    영향 휴가 종류: N개
    (연차, XXX, XXX)
    ```
  - **off로 변경 시**:
    ```
    off로 변경 시, 임직원이 보유한 시간 단위 잔여 연차/휴가(예: 0.125일, 0.25일 등)는 더 이상 사용할 수 없습니다.
    변경 전에 잔여 연차/휴가 사용 여부를 확인해주세요.

    또한, 시간 단위 사용이 설정된 휴가 종류에서도 시간 단위 사용이 제한됩니다.
    영향 휴가 종류: N개
    (연차, XXX, XXX)

    ※ 이미 대기중인 시간 단위 신청은 유지되며, 정상적으로 승인/반려 처리할 수 있습니다.
    ```

#### 액션 버튼 영역

**1. 저장 버튼**
- **활성화 조건**: 변경 사항이 있을 때만 활성화
- **클릭 시**:
  1. confirm 노출
     - **on으로 변경 시 문구**: "연차/휴가 시간 단위 사용을 허용하시겠습니까?"
     - **off로 변경 시 문구**: "연차/휴가 시간 단위 사용을 제한하시겠습니까?"
  2. 확인 선택 시:
     - 설정 업데이트 (시간 단위 사용 여부)
     - **시간 단위만 설정된 휴가 종류** 활성/비활성 처리
       - on으로 변경: 시간 단위만 설정된 휴가 → 활성화
       - off로 변경: 시간 단위만 설정된 휴가 → 비활성화
     - 토스트 노출: "시간 단위 연차/휴가 사용이 활성화/비활성화되었습니다."
     - 설정 다시 로드 (변경 사항 초기화)
  3. 취소 선택 시: 아무 동작 없음

**2. 취소 버튼**
- **표시 조건**: 변경 사항이 있을 때만 표시
- **클릭 시**: 설정 다시 로드 (변경 사항 초기화)

#### 연관 기능

**1. 휴가 종류 관리 탭**
- **시간 단위만 설정된 휴가 활성화 제한**:
  - 시간 단위 설정이 off 상태에서 시간 단위만 설정된 휴가는 활성화 불가
  - 활성화 시도 시 alert 노출: "연차/휴가 시간 단위 설정이 off 상태일 경우, 시간 단위만 사용 가능한 휴가는 활성화 시킬 수 없습니다."
- **복수 단위 설정된 휴가**: 활성 상태 유지 (시간 단위 옵션만 숨김)

**2. 연차/휴가 신청 팝업 (직원 상세 > 연차/휴가 탭)**
- **시간 단위 옵션 표시 제어**:
  - 시간 단위 설정이 on: 1~8시간 선택 옵션 표시
  - 시간 단위 설정이 off: 1~8시간 선택 옵션 숨김 (일, 반일 옵션은 정상 표시)
- **적용 대상**: '사용 단위'에 '시간 단위'가 포함된 모든 휴가 종류

**3. 대기중인 시간 단위 신청 처리**
- **정책**: 시간 단위 설정을 off로 변경해도 이미 대기중인 시간 단위 신청은 유지
- **처리 방식**:
  - 대기중인 시간 단위 신청은 자동으로 반려되지 않음
  - 정상적으로 승인/반려 처리 가능
  - 설정 변경은 "새로운 신청"에만 적용 (소급 적용 없음)

### 2.2 연차 발생 기준 Card

#### 연차 발생 기준 선택

**옵션 1: 입사일 기준**
- **제목**: "입사일 기준"
- **설명**: "입사일부터 1년 단위로 연차를 계산합니다."
- **상세 계산 방법**:
  - • 1년 미만: 월 1일 발생 (예: 3월 입사 → 10개월 = 10일)
  - • 1년 이상: 15일 + 근속가산 (2년마다 1일, 최대 25일)
  - • 예: 입사 3주년 → 16일 (15 + 1)

**옵션 2: 연초 기준 (1월 1일 일괄 부여)**
- **제목**: "연초 기준 (1월 1일 일괄 부여)"
- **설명**: "매년 1월 1일에 해당 연도의 연차를 일괄적으로 부여합니다."
- **상세 계산 방법**:
  - • 입사년도: 입사월 다음 달부터 12월까지 월차 1개씩
  - • 다음 해 (1년 미만): 비례 부여 (15 × 전년도 근무개월 / 12)
  - • 1년 이상: 15일 + 근속가산 (2년마다 1일, 최대 25일)
  - • 예: 2025년 5월 10일 입사 → 2025년 7개, 2026년 1월 1일 10개

##### 비례부여 소수점 처리 (Select)
- **활성화 조건**: 연초 기준 선택 시에만 활성화 (입사일 기준 선택 시 비활성화)
- **옵션**:
  - **버림**: "버림 (예: 8.75개 → 8개)" (값: "floor")
  - **올림**: "올림 (예: 8.75개 → 9개)" (값: "ceil")
- **설명**: "중도입사자의 연차를 비례 계산할 때 소수점을 어떻게 처리할지 선택합니다."`

#### 주의 안내 영역

- **표시 조건**: 연차 발생 기준 또는 비례부여 소수점 처리를 변경했을 때만 표시 (변경 사항이 없으면 숨김)
- **위치**: 액션 버튼 영역 바로 위 (저장 버튼 위)
- **제목**: "주의"
- **내용**: "연차 발생 기준을 변경하면 모든 직원의 연차 현황이 자동으로 재계산됩니다. 변경 전에 신중히 검토해주세요."

#### 액션 버튼 영역

**1. 저장 버튼**
- **버튼명**: "저장" (저장 중: "저장 중...")
- **활성화 조건**: 변경 사항이 있을 때만 활성화
- **클릭 시**:
  1. confirm 노출
     - 문구: "⚠️ 연차 발생 기준을 변경하면 모든 직원의 연차 현황에 영향을 미칩니다.\n모든 직원의 연차가 자동으로 재계산됩니다.\n\n변경하시겠습니까?"
  2. 확인 선택 시:
     - 설정 업데이트 (연차 발생 기준, 비례부여 소수점 처리)
     - 모든 재직중인 직원의 연차 자동 재계산 (퇴사자 제외)
     - 토스트 노출: "연차 발생 기준이 변경되었습니다. (N명 재계산)"
     - 설정 다시 로드 (변경 사항 초기화)
  3. 취소 선택 시: 아무 동작 없음

**2. 취소 버튼**
- **표시 조건**: 변경 사항이 있을 때만 표시
- **클릭 시**: 설정 다시 로드 (변경 사항 초기화)

**3. 전체 직원 연차 재계산 버튼**
- **클릭 시**:
  1. confirm 노출
     - 문구: "⚠️ 모든 재직 직원의 연차를 현재 설정 기준으로 재계산합니다.\n\n계속하시겠습니까?"
  2. 확인 선택 시:
     - 모든 재직중인 직원의 연차 재계산 (현재 설정 기준, 퇴사자 제외)
     - 토스트 노출: "N명의 연차가 재계산되었습니다."
  3. 취소 선택 시: 아무 동작 없음
- **활성화 조건**: 재계산 중이거나 저장 중일 때 비활성화
