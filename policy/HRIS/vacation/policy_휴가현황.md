# 휴가 현황 화면(HRIS_VAC_01) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_VAC_01
- **화면명**: 휴가 현황
- **목적**: 재직중인 전체 직원의 연차 발생/사용/잔여 현황을 조회하고, 직원별 휴가 신청 내역을 관리하는 화면
- **주요 기능**: 요약 통계 카드, 성명/사번 검색, 부서 필터, 연차 현황 리스트 조회, 직원별 상세 관리
- **진입 경로**: 인사정보 모듈 > 연차/휴가관리 메뉴 > 휴가 현황 탭 선택 시 진입
- **연관 화면**:
  - HRIS_VAC_01_DLG (직원 휴가 관리 Dialog) — 행 클릭 시 팝업

---

## 2. 요청한 화면 기능

### 2.1 요약 카드 (3개)

화면 상단에 전체 현황을 요약하는 3개의 카드를 표시한다.

#### 카드 1: 전체 직원
- **표시 내용**: 재직중인 전체 직원 수
- **표시 형식**: "N명" (예: "50명")
- **계산 기준**: 재직상태가 "재직중"인 직원 수

#### 카드 2: 평균 연차 사용률
- **표시 내용**: 전체 직원의 평균 연차 사용률
- **표시 형식**: "N%" (예: "68%")
- **계산 기준**: 각 직원의 (사용일수 / 총발생일수 * 100)의 평균값 (소수점 반올림)

#### 카드 3: 전체 잔여 연차
- **표시 내용**: 전체 직원의 잔여 연차 합계
- **표시 형식**: "N일" (예: "250일")
- **계산 기준**: 모든 직원의 잔여일수 합계

### 2.2 검색 영역

- 구성: 인풋박스 (좌측 검색 아이콘 포함)
- 검색항목: **성명 또는 사번**
- placeholder 문구: `성명, 사번 검색`
- Enter 입력 시 검색 실행
- 검색 방식: **완전 일치** — 입력된 검색어가 성명 또는 사번과 정확히 일치하는 경우에만 조회 (대소문자 구분 없음)
- 검색 인풋박스 빈 값/공백 처리:
  - 빈 값 또는 공백만 입력 시 전체 조회
  - 앞뒤 공백은 자동 trim 처리
- 검색 시 하단 리스트를 검색 결과 기준으로 갱신한다.
- **검색 실행 후 입력 필드 포커스 유지** (공통 정책 1.1 적용)
- **부서 필터(2.3)와 동시 적용 가능** (AND 조건)
  - 검색과 부서 필터가 모두 적용된 경우, 두 조건을 모두 만족하는 직원만 조회
  - 예: "홍길동" 검색 + "개발팀" 선택 → 개발팀에 소속된 "홍길동"만 조회

### 2.3 부서 필터

- 구성: Combobox (shadcn Command 컴포넌트)
- 기능: 특정 부서의 직원만 필터링
- **옵션**:
  - "전체 부서" (기본 선택, value: 'all')
  - 각 부서별 전체 경로 (예: "본사 > 개발팀 > 프론트엔드팀")
  - "해당 없음" (부서 미배정 직원, value: 'none')
- **정렬**: 부서 전체 경로 기준 정렬
- 선택 시 리스트를 선택한 부서 기준으로 필터링한다.
- 부서 Combobox 내부 검색 기능 제공 (CommandInput)

### 2.4 리스트(테이블)

- **조회 대상**: 재직중인 직원만 (퇴사자 제외)
- **기본 정렬**: 부서명 > 사번 (부서별로 그룹화, 같은 부서 내에서는 사번 순)
- **체크박스**: 제공하지 않음 (선택 삭제 등의 기능 없음)
- 컬럼 정의:

| 컬럼명 | 표시형식 | 비고 |
|--------|---------|------|-------|------|
| 사번 | 텍스트 | |
| 성명 | 텍스트 | |
| 부서 | 텍스트 | 전체 경로 표시 (예: "본사 > 개발팀"), 미배정 시 "-" |
| 발생 | "N일" | 총 발생일수 (자동 계산) |
| 사용 | "N일" | 사용일수 (승인된 신청 기준) |
| 대기중 | "N일" | 승인 대기중인 신청 일수 |
| 잔여 | "N일" | 잔여일수 (발생 - 사용 - 대기중) |
| 사용률 | Progress bar + "N%" | 프로그레스바(w-20 h-2) + 퍼센트(w-12) |

#### 연차 계산 로직
- **발생일수**: 연차발생 기준으로 시스템에서 자동 계산
- **사용일수**: 승인 완료된 휴가 신청의 차감 일수를 모두 합산
- **대기중일수**: 승인 대기중인 휴가 신청의 차감 일수를 모두 합산
- **잔여일수**: 발생일수 - 사용일수 - 대기중일수
- **사용률**: (사용일수 ÷ 발생일수 × 100) 소수점 반올림

### 2.5 행 클릭 - 직원 휴가 관리 Dialog (HRIS_VAC_01_DLG)

행 클릭 시 해당 직원의 휴가 신청 내역을 관리할 수 있는 Dialog가 열린다.

#### Dialog 헤더
- **제목**: "{직원명}"
- **설명**: "승인된 신청은 취소할 수 있고, 반려된 신청은 승인할 수 있습니다."

#### Dialog 내용
- **조회 대상**: 해당 직원의 승인/반려/승인취소된 신청만 (대기중 제외)
- **정렬**: 신청일시 기준 (최근 순)

#### 테이블 컬럼

| 컬럼명 | 표시형식 | 비고 |
|--------|---------|------|
| 신청일 | YYYY. M. D. | |
| 유형 | 텍스트 | 연차, 반차(오전), 반차(오후) |
| 기간 | YYYY-MM-DD ~ YYYY-MM-DD | |
| 일수 | "N일" | 평일 기준 실제 차감 일수 |
| 사유 | 텍스트 | |
| 상태 | Badge | 승인, 반려, 승인취소 |
| 액션 | 버튼 | 상태별 버튼 표시 |

#### 액션 버튼

1. **승인 취소 버튼** (상태가 '승인'인 경우)
   - 버튼명: "승인 취소"
   - 클릭 시 confirm 노출: "승인을 취소하시겠습니까? 연차가 복원됩니다."
   - 확인 시:
     - 상태를 '승인취소'로 변경
     - 연차 잔액 재계산 (복원)
     - 토스트 노출: "승인 취소가 완료되었습니다."
     - Dialog 내용 새로고침
     - 부모 화면(휴가 현황 리스트) 새로고침
   - 처리 중 버튼 비활성화 및 문구 변경: "처리 중..."

2. **승인 버튼** (상태가 '반려' 또는 '승인취소'인 경우)
   - 버튼명: "승인"
   - 클릭 시 confirm 노출: "반려된 신청을 승인하시겠습니까? 연차가 차감됩니다."
   - 확인 시:
     - 상태를 '승인'으로 변경
     - 연차 잔액 재계산 (차감)
     - 토스트 노출: "승인이 완료되었습니다."
     - Dialog 내용 새로고침
     - 부모 화면(휴가 현황 리스트) 새로고침
   - 처리 중 버튼 비활성화 및 문구 변경: "처리 중..."

#### 빈 화면 처리
- 데이터 0건: "승인 또는 반려된 신청이 없습니다." 
- 로딩 중: "로딩 중..."

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
|--------------|--------------|------------|------|
| 1.1 검색 | 검색 실행 후 입력 필드 포커스 유지 | 그대로 적용 | 충돌 없음 |
| 1.2 리스트 개수 | 카운트 표시 | **미적용** | 요약 카드로 대체 (전체 직원 수 표시) |
| 1.3 리스트(테이블) | 체크박스, 행 클릭 | **부분 적용** | 체크박스 미제공, 행 클릭만 적용 |
| 1.4 페이지 사이즈 | 페이지네이션 | **미적용** | 전체 직원 한 화면에 표시 (페이징 없음) |
| 1.5 엑셀 다운로드 | 다운로드 기능 | **미적용** | 엑셀 다운로드 기능 없음 |
| 1.6 선택 삭제 | 삭제 기능 | **미적용** | 삭제 기능 없음 (체크박스 미제공) |

> 본 화면은 직원 리스트 화면과 달리, 연차 현황 조회에 특화된 화면으로 페이징/선택 삭제/엑셀 다운로드 기능이 없으며, 대신 요약 통계 카드와 부서 필터를 제공한다.
