# 직원 상세 - 연차/휴가 탭(HRIS_EMP_03_VAC) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_EMP_03_VAC
- **화면명**: 직원 상세 - 연차/휴가 탭
- **목적**: 선택한 직원의 연차 현황, 신청 내역, 이력을 조회하고 연차를 신청하는 탭
- **주요 기능**: 연차 현황 조회, 신청 내역 조회, 연차 신청, 신청 취소, 연차 이력 조회
- **진입 경로**: HRIS_EMP_03 (직원 상세) 화면에서 "연차/휴가" 탭 선택 시 표시
- **연관 화면**:
  - HRIS_EMP_03 (직원 상세) — 부모 화면
  - 연차/휴가 신청 다이얼로그 — '+ 연차 신청' 버튼 클릭 시 노출
  - 반려 사유 다이얼로그 — '사유 보기' 버튼 클릭 시 노출

---

## 2. 요청한 화면 기능

### 2.1 연차 현황 카드

**기본 구성**
- 제목: "{현재년도}년 연차 현황"
- 우측 상단: "+ 연차 신청" 버튼
- "+ 연차 신청" 버튼 클릭 시: 2.3 연차 신청 다이얼로그 노출

#### 2.1.1 현황 표시 (4개 항목, 4열 그리드)

| 항목 | 표시값 | 
|------|--------|
| 총 발생 | {totalDays}일 | 
| 사용 | {usedDays}일 | 
| 대기중 | {pendingDays}일 | 
| 잔여 | {remainingDays - pendingDays}일 | 

**계산 로직**
- 총 발생: 당해년도 발생한 연차 총 일수
- 사용: 승인된 연차 사용 일수
- 대기중: 승인 대기 중인 신청 일수
- 잔여: 총 발생 - 사용 - 대기중

### 2.2 신청 내역

**기본 구성**
- 제목: "신청 내역"
- 테이블 형식으로 표시

#### 2.2.1 테이블 컬럼

| 컬럼명 | 표시값 | 형식 |
|--------|--------|------|
| 신청일 | {requestedAt} | MM/DD/YYYY |
| 유형 | {휴가유형명} | - |
| 기간 | {startDate} ~ {endDate} | YYYY-MM-DD |
| 일수 | {workingDays}일 | - |
| 사유 | {reason} | 최대 200px, 초과 시 말줄임(...) |
| 상태 | 뱃지 | 2.2.2 참조 |
| 액션 | 버튼 | 2.2.3, 2.2.4 참조 |

#### 2.2.2 상태 뱃지 및 액션 버튼

| 상태값 | 뱃지 텍스트 | 우측 버튼 |
|--------|-------------|-----------|
| pending | 대기중 | 신청 취소 버튼 (2.2.3) |
| approved | 승인 | - |
| rejected | 반려 | 사유 보기 버튼 (2.2.4) |
| cancelled | 승인취소 | - | 

#### 2.2.3 신청 취소 버튼

**표시 조건**
- 상태가 "pending" (대기중)인 경우에만 표시

**클릭 시 동작**
1. Confirm 메시지: "휴가 신청을 취소하시겠습니까?"
2. "확인" 선택:
   - 신청 취소 API 호출
   - 취소 성공 시:
     - 토스트 메시지: "휴가 신청이 취소되었습니다."
     - 데이터 새로고침
   - 취소 실패 시:
     - Alert 경고 메시지: "휴가 신청 취소에 실패했습니다."
3. "취소" 선택: 동작 없음

#### 2.2.4 사유 보기 버튼

**표시 조건**
- 상태가 "rejected" (반려)이고, 반려사유가 있는 경우에만 표시

**클릭 시 동작**
- 2.5 반려 사유 다이얼로그 노출

#### 2.2.5 빈 상태
- 신청 내역이 없는 경우: "신청 내역이 없습니다." 메시지 표시

### 2.3 연차 신청

#### 2.3.1 연차 신청 다이얼로그

**다이얼로그 제목**
- "연차/휴가 신청"

**다이얼로그 설명**
- "{직원명}님의 휴가를 신청합니다."

**연차 현황 요약**
- 다이얼로그 상단에 연차 현황 표시 
- 총 발생, 사용, 대기중, 잔여 (4개 항목)

#### 2.3.2 입력 필드

| 필드명 | 필수 여부 | 입력 타입 | 비고 |
|--------|-----------|-----------|------|
| 휴가 유형 | 필수 (*) | 셀렉트박스 | 휴가 유형 마스터 데이터 (현재: 연차, 오전반차, 오후반차) |
| 시작일 | 필수 (*) | date input | YYYYMMDD 형식으로 저장 |
| 종료일 | 필수 (*) | date input | YYYYMMDD 형식으로 저장 |
| 사유 | 필수 (*) | textarea (3행) | - |

#### 2.3.3 반차 선택 시 특수 동작

- 휴가 유형을 오전반차 또는 오후반차를 선택한 경우, 시작일 입력 시 종료일 자동으로 시작일과 동일하게 설정 후 종료일 비활성화

#### 2.3.4 예상 차감 일수 계산

**표시 조건**
- 휴가 유형, 시작일, 종료일이 모두 입력된 경우

**표시 영역**
- 텍스트: "예상 차감 일수: N일"

**계산 로직**
- **반차**: 무조건 0.5일
  - 추가 안내문구: "(반차는 0.5일 고정)"
- **연차**: 평일 계산 (공휴일 제외)
  - 시작일 ~ 종료일 범위의 평일(월~금) 일수 계산
  - 공휴일은 제외
  - 추가 안내문구: "(평일 기준, 공휴일 제외)"

#### 2.3.5 신청 버튼

**활성화 조건**
- 모든 필수 필드 입력
- 제출 중이 아닐 때

**비활성화 조건**
- 필수 필드 미입력
- 제출 중일 때

**클릭 시 동작**
1. **필수 필드 검증**
   - 미입력 시 Alert 경고 메시지: "모든 필드를 입력해주세요."
   - 검증 실패 시 중단

2. **잔여 연차 확인**
   - 예상 차감 일수 > 잔여 연차 - 대기중 일수
   - 부족 시 Alert 경고 메시지: "잔여 연차가 부족합니다."
   - 부족 시 중단

3. **신청 API 호출**
   - 신청 성공 시:
     - 토스트 메시지: "휴가 신청이 완료되었습니다. (대기중)"
     - 데이터 새로고침
     - 다이얼로그 닫기
   - 신청 실패 시:
     - Alert 경고 메시지: "휴가 신청에 실패했습니다."

**제출 중일 때**
- 버튼 텍스트: "신청 중..."
- 버튼 비활성화

#### 2.3.6 취소 버튼

**클릭 시 동작**
- 입력값 초기화
- 다이얼로그 닫기

### 2.4 연차 이력

#### 2.4.1 표시 조건
- **이력 보기 토글이 ON인 경우**
- **연차/휴가 탭이 선택된 경우**
- 두 조건이 모두 만족해야 이력 섹션 표시

#### 2.4.2 이력 표시

**제목**
- "연차이력"

**타임라인 형식**
- 각 이력 항목을 시간순으로 세로로 나열
- 발생 시간 기준 내림차순 (최신순)

**이력 항목 구성**
1. **구분 및 발생시간** (상단)
   - 표시 형식: {구분} · {발생시간}
   - 구분: "발생" / "사용" / "취소"
   - 발생시간: YYYY-MM-DD HH:mm 형식

2. **변경 내용 카드**
   - 일수: {days}일 (양수: 녹색 "+", 음수: 빨간색)
   - 사유: {reason}

#### 2.4.3 이력 구분별 표시

| 구분 (type) | 표시명 | 일수 색상 | 사유 표시 |
|-------------|--------|-----------|-----------|
| grant | 발생 | 녹색 (+) | 이력의 사유 |
| use | 사용 | 빨간색 (-) | **연차/휴가 신청을 통해 사용한 경우: 신청 시 입력한 사유 제공**<br>그 외: 이력의 사유 |
| cancel | 취소 | 녹색 (+) | 이력의 사유 |

**사용(use) 타입의 사유 표시 우선순위**
1. 연차/휴가 신청을 통해 사용한 경우: 신청 다이얼로그(2.3)에서 입력한 사유 제공
2. 그 외의 경우: 이력의 사유 표시

#### 2.4.4 빈 상태
- 이력이 없는 경우: "연차 이력이 없습니다." 메시지 표시

### 2.5 반려 사유 다이얼로그

**다이얼로그 제목**
- "반려 사유"

**다이얼로그 설명**
- "휴가 신청이 반려된 사유입니다."

**사유 표시**
- 반려 사유 텍스트 표시
- 줄바꿈 유지 (whitespace-pre-wrap)

**닫기 버튼**
- 클릭 시: 다이얼로그 닫기

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
