# 직원 상세 화면(HRIS_EMP_03) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_EMP_03
- **화면명**: 직원 상세
- **목적**: 선택한 직원의 상세 정보를 조회하고 수정하는 화면
- **주요 기능**: 직원 정보 조회, 탭별 정보 확인 (개인정보, 조직정보, 급여정보, 연차/휴가), 이력 보기 토글
- **진입 경로**: HRIS_EMP_01 (직원관리) 화면에서 행 클릭 시 진입
- **화면 전달 파라미터**: URL 경로 방식 (예: `/hr/employee/{employeeId}`)
  - URL 파라미터 없이 직접 접근 시: HRIS_EMP_01 (직원관리) 화면으로 이동
  - 유효하지 않은 employeeId 전달 시: HRIS_EMP_01 (직원관리) 화면으로 이동
- **연관 화면**:
  - HRIS_EMP_01 (직원관리) — 뒤로가기 버튼 클릭 시 이동
  - HRIS_EMP_03_PER (개인정보 탭) — 개인정보 탭 선택 시 표시
  - HRIS_EMP_03_ORG (조직정보 탭) — 조직정보 탭 선택 시 표시
  - HRIS_EMP_03_PAY (급여정보 탭) — 급여정보 탭 선택 시 표시
  - HRIS_EMP_03_VAC (연차/휴가 탭) — 연차/휴가 탭 선택 시 표시

## 2. 요청한 화면 기능

### 2.1 PageHeader

#### 2.1.1 제목 영역
- **구성**: 직원 이름 + 재직상태 뱃지
- **직원 이름**: 텍스트 표시
- **재직상태 뱃지**:
  - "재직중": 녹색 뱃지 (variant="success")
  - "퇴사": 회색 뱃지 (variant="secondary")
- **재직상태 판정 기준**:
  - 재직중: 퇴사일이 없거나 퇴사일이 오늘 이후인 경우
  - 퇴사: 퇴사일이 오늘 이전인 경우

#### 2.1.2 PageHeader 클릭 동작
- **PageHeader 전체 영역** (제목 영역 + 뒤로가기 버튼) 클릭 시 HRIS_EMP_01 (직원관리) 화면으로 이동
- 뒤로가기 버튼 표시

### 2.2 프로필 헤더

- **위치**: PageHeader 바로 아래, 탭 위
- **표시 정보**:
  - 사번: "사번: {employeeNumber}"
  - 근속기간: "근속기간: {tenure}"
- **근속기간 계산**:
  - 입사일 ~ 현재일 (재직중인 경우)
  - 입사일 ~ 퇴사일 (퇴사한 경우)
  - 표시 형식: "N년 N개월" (1개월 미만: "1개월 미만")

### 2.3 탭 구성

#### 2.3.1 탭 목록
- **4개 탭**: 개인정보, 조직정보, 급여정보, 연차/휴가
- **기본 선택 탭**: 개인정보
- **탭 전환**: 클릭 시 해당 탭 콘텐츠로 전환

#### 2.3.2 탭 레이아웃
- **위치**: 프로필 헤더 아래
- **구성**:
  - 좌측: 탭 목록 (TabsList)
  - 우측: 이력 보기 토글

#### 2.3.3 탭별 정책서
각 탭의 상세 내용은 별도 정책서 참조:
- **개인정보 탭**: HRIS_EMP_03_PER 정책서
- **조직정보 탭**: HRIS_EMP_03_ORG 정책서
- **급여정보 탭**: HRIS_EMP_03_PAY 정책서
- **연차/휴가 탭**: HRIS_EMP_03_VAC 정책서

### 2.4 이력 보기 토글

#### 2.4.1 위치 및 구성
- **위치**: 탭 목록 우측
- **구성**: 레이블("이력 보기") + Switch 컴포넌트
- **기본값**: 최초 진입 시 OFF, 이후 마지막 설정값 유지
- **상태 저장**: LocalStorage에 저장하여 전역적으로 유지
  - 한 직원 상세 화면에서 ON → 다른 직원 상세 화면 진입 시에도 ON 유지
  - 한 직원 상세 화면에서 OFF → 다른 직원 상세 화면 진입 시에도 OFF 유지

#### 2.4.2 동작 방식
- Switch ON: 현재 선택된 탭의 이력 섹션 표시
- Switch OFF: 이력 섹션 숨김
- 탭 전환 시: 이력 보기 상태 유지 (각 탭별로 이력 표시/숨김)

#### 2.4.3 이력 표시 대상
- **개인정보 탭**: 개인정보 관련 이력 (category="personal")
- **조직정보 탭**: 조직정보 관련 이력 (category="organization")
- **급여정보 탭**: 급여정보 관련 이력 (category="salary")
- **연차/휴가 탭**: 연차 이력 (타임라인 형식)

### 2.5 데이터 로딩

#### 2.5.1 정보 수정 후 새로고침
- 각 탭의 수정 다이얼로그에서 정보 수정 완료 시
- 직원 정보 다시 로딩
- 연관 정보 (직급, 근로형태) 다시 로딩
- 이력 새로고침 (historyRefreshKey 증가)

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
|--------------|--------------|------------|------|
| PageHeader | 뒤로가기 버튼 표시 | 그대로 적용 | 충돌 없음 |
| 재직상태 | 퇴사일 기준 판정 | 그대로 적용 | 충돌 없음 |
| 근속기간 | 입사일 ~ 현재/퇴사일 계산 | 그대로 적용 | 충돌 없음 |

> 본 화면은 공통 정책과 대부분 일치하며, 주요 충돌 없이 공통 정책을 그대로 적용한다.
