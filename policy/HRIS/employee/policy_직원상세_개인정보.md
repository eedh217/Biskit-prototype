# 직원 상세 - 개인정보 탭(HRIS_EMP_03_PER) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_EMP_03_PER
- **화면명**: 직원 상세 - 개인정보 탭
- **목적**: 선택한 직원의 개인정보를 조회하고 수정하는 탭
- **주요 기능**: 개인정보 조회, 개인정보 수정, 개인정보 이력 조회
- **진입 경로**: HRIS_EMP_03 (직원 상세) 화면에서 "개인정보" 탭 선택 시 표시
- **기본 선택**: HRIS_EMP_03 화면 진입 시 기본으로 선택되는 탭
- **연관 화면**:
  - HRIS_EMP_03 (직원 상세) — 부모 화면
  - 개인정보 수정 다이얼로그 — '수정' 버튼 클릭 시 노출

---

## 2. 요청한 화면 기능

### 2.1 개인정보 카드

**기본 구성**
- 카드 우측 상단에 "수정" 버튼 배치
- "수정" 버튼 클릭 시: 2.2 개인정보 수정 다이얼로그 노출

#### 2.1.1 공통 필드 (내외국인 모두 표시)

| 필드명 | 표시 조건 | 형식 | 비고 |
|--------|-----------|------|------|
| 내외국인 여부 | 항상 표시 | "내국인" / "외국인" | - |
| 거주구분 | 항상 표시 | "거주자" / "비거주자" | - |
| 장애여부 | 항상 표시 | "비장애인" / "장애인복지법상 장애인" / "국가유공자 중증환자" / "중증환자" | - |
| 이메일 | 항상 표시 | - | - |
| 연락처 | 항상 표시 | - | 값 없음: "-" |
| 휴대폰번호 | 항상 표시 | 하이픈 포함 (예: 010-1234-5678) | 값 없음: "-" |
| 입사일 | 항상 표시 | YYYY-MM-DD | - |
| 퇴사일 | 퇴사일 값이 있는 경우만 | YYYY-MM-DD | - |
| 주소 | zipCode와 address가 모두 있는 경우만 | ({zipCode}) {address} {detailAddress} | 값 없음: "-" |

#### 2.1.2 내국인 전용 필드

| 필드명 | 표시 조건 | 형식 | 비고 |
|--------|-----------|------|------|
| 주민등록번호 | 내외국인 여부가 "내국인" | 전체 표시 (마스킹 없음) | - |

#### 2.1.3 외국인 전용 필드

| 필드명 | 표시 조건 | 형식 | 비고 |
|--------|-----------|------|------|
| 외국인등록번호 | 내외국인 여부가 "외국인" AND 외국인등록번호 값이 있음 | 전체 표시 (마스킹 없음) | - |
| 여권번호 | 내외국인 여부가 "외국인" AND 여권번호 값이 있음 | - | - |
| 생년월일 | 내외국인 여부가 "외국인" AND 여권번호 값이 있음 | YYYY-MM-DD | - |
| 성별 | 내외국인 여부가 "외국인" AND 여권번호 값이 있음 | "남" / "여" | - |
| 국적 | 내외국인 여부가 "외국인" | ISO 3166-1 alpha-2 코드의 한글명 (예: "미국", "일본") | - |

### 2.2 개인정보 수정

#### 2.2.1 수정 다이얼로그 기본 정책
- **기본 기능**: HRIS_EMP_02 (직원 추가) 화면의 개인정보 영역과 동일
  - 모든 필드 구성 및 입력 규칙
  - 유효성 검증 규칙 (필드별 검증 시점 포함)
  - 필수 필드 규칙
  - 한글 입력 처리
  - 내외국인 구분 로직
  - 상세 내용은 HRIS_EMP_02 정책서 참조

#### 2.2.2 수정 다이얼로그 차이점

**화면 형태**
- 다이얼로그 형태로 표시
- 다이얼로그 제목: "개인정보 수정"

**사번**
- **읽기 전용** (변경 불가)
- 중복 체크 미수행

**초기 데이터**
- **전달 데이터**: 현재 직원 정보 (employee)
- 폼 필드: 기존 직원 정보로 채움

**다이얼로그 이탈 방지**
- 다이얼로그 닫기 시도 시 (X 버튼, ESC 키, 외부 클릭) 확인 메시지 표시
  - 입력값 변경이 있었을 경우 :
    - Confirm 메시지: "개인정보 수정을 취소하시겠습니까?"
    - "확인" 선택: 다이얼로그 닫기
    - "취소" 선택: 다이얼로그 유지
  - 입력값 변경이 없었을 경우:
    - 즉시 다이얼로그 닫기

**하단 버튼**
- 취소 버튼:
  - 클릭 시:
    - 입력값 변경이 있었을 경우 :
      - Confirm 메시지: "개인정보 수정을 취소하시겠습니까?"
      - "확인" 선택: 다이얼로그 닫기
      - "취소" 선택: 다이얼로그 유지
    - 입력값 변경이 없었을 경우:
      - 즉시 다이얼로그 닫기
- 수정 버튼:
  - **활성화 조건**: 필수 필드가 모두 입력되고 유효성 검증을 통과한 경우
  - **비활성화 조건**: 필수 필드 미입력 또는 유효성 검증 실패
  - 클릭 시 처리 순서:
    1. **유효성 검증**:
       - 필수 필드 입력 확인
       - 신분증 번호 체크디짓 검증
       - 이메일 형식 검증
       - 이메일 중복 체크
       - **퇴사일 + 부서장 검증**: 현재 부서장으로 지정된 직원의 퇴사일을 오늘 이전으로 변경하려는 경우 에러 표시
         - 에러 위치: 퇴사일 입력 영역
         - 에러 메시지: "부서장으로 지정된 직원은 퇴사일을 오늘 이전 날짜로 지정할 수 없습니다. 조직정보 탭에서 부서장 지정을 먼저 해제해주세요."
    2. **직원 수정**:
       - 직원 수정 API 호출
       - 수정 성공 시:
         - 토스트 메시지: "직원정보 수정을 완료했습니다."
         - 다이얼로그 닫기
         - 직원 정보 새로고침
         - 개인정보 이력 새로고침
       - 수정 실패 시:
         - Alert 경고 메시지: {에러 메시지}
  - 제출 중일 때 버튼 텍스트: "수정 중..."
  - 제출 중일 때 버튼 비활성화

### 2.3 개인정보 이력

#### 2.3.1 표시 조건
- **이력 보기 토글이 ON인 경우**
- **개인정보 탭이 선택된 경우**
- 두 조건이 모두 만족해야 이력 섹션 표시

#### 2.3.2 이력 표시

**타임라인 형식**
- 각 이력 항목을 시간순으로 세로로 나열
- 수정일시 기준 내림차순 (최신순)
- **무한 스크롤**:
  - 초기 표시: 10개
  - 스크롤 시: 10개씩 추가 로드
  - 최대 개수: 제한 없음 (전체 이력 표시)
  - 10개 초과 시 스크롤 발생

**이력 항목 구성**
1. **수정자 및 날짜** (상단)
   - 표시 형식: 수정자 이름 · 수정날짜
   - 수정날짜: YYYY-MM-DD HH:mm 형식
   - 예시: "홍길동 · 2024-01-15 14:30"

2. **변경 내용 카드**
   - 각 변경 사항을 행으로 나열
   - 변경 사항 표시 형식:
     - 필드명 
     - 변경 전 값 (취소선 표시)
     - 화살표 아이콘 (→)
     - 변경 후 값 (강조 표시)
   - 예시: "이메일    ~~old@example.com~~ → new@example.com"

#### 2.3.3 빈 상태
- 이력이 없는 경우: "수정 이력이 없습니다." 메시지 표시

#### 2.3.4 이력 새로고침
- **자동 새로고침 조건**: 개인정보 수정 완료 시
- **동작**: 컴포넌트 재마운트하여 최신 이력 다시 조회

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
|--------------|--------------|------------|------|
| 내외국인 구분 | 내국인/외국인에 따른 필드 표시 | 그대로 적용 | 충돌 없음 |
| 날짜 형식 | YYYY-MM-DD | 그대로 적용 | 충돌 없음 |
| 값 없음 표시 | "-" | 그대로 적용 | 충돌 없음 |

> 본 화면은 공통 정책과 대부분 일치하며, 주요 충돌 없이 공통 정책을 그대로 적용한다.
