# 직원 추가 화면(HRIS_EMP_02) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_EMP_02
- **화면명**: 직원 추가
- **목적**: 새로운 직원의 정보를 입력하여 등록하는 화면
- **주요 기능**: 직원 정보 입력 (개인정보, 조직정보, 급여정보), 유효성 검증, 직원 등록
- **진입 경로**: HRIS_EMP_01 (직원관리) 화면에서 '추가하기' 버튼 클릭 시 진입
- **연관 화면**:
  - HRIS_EMP_01 (직원관리) — 취소 버튼 클릭 시 이동
  - HRIS_EMP_03 (직원 상세) — 직원 추가 완료 후 이동
  - 주소 검색 다이얼로그 — '주소 검색' 버튼 클릭 시 노출

---

## 2. 요청한 화면 기능

### 2.1 화면 구성

- **3개 섹션 구분**: 개인정보, 조직정보, 급여정보
- 각 섹션은 Card 컴포넌트로 구분하여 표시
- 하단에 고정 버튼 영역 배치 (취소, 추가)

### 2.2 개인정보 섹션

#### 2.2.1 사번
- **필수 입력** (*)
- 최대 길이: 30자
- 허용 문자: 한글, 영문, 숫자, 특수문자 (!"#$%&'()*+,-./:;<>=?@[\]^_`{|}~)
- **유효성 검증 시점**: Submit 시 (추가 버튼 클릭)
- 중복 체크: 등록 시 기존 사번과 중복되면 Alert 경고 메시지 표시
  - **중복 체크 대상**: 전체 직원 (퇴사자 포함)
  - Alert 메시지: "중복된 사번은 사용하실 수 없습니다."

#### 2.2.2 성명
- **필수 입력** (*)
- 최대 길이: 30자
- 허용 문자: 한글, 영문, 숫자, 공백
- **유효성 검증 시점**: Blur 시 (입력란을 벗어날 때)
- 특수문자 입력 시 자동 필터링 및 에러 메시지 표시
  - 에러 메시지: "특수문자는 입력하실 수 없습니다."

#### 2.2.3 내외국인 여부
- **필수 선택** (*)
- 라디오 버튼: 내국인 / 외국인
- 기본값: 내국인

#### 2.2.4 주민등록번호 (내국인 선택 시)
- **필수 입력** (*)
- 입력 형식: 앞 6자리 - 뒤 7자리
- 앞 6자리 입력 완료 시 뒤 7자리 입력란으로 자동 포커스 이동
- **유효성 검증 시점**: Blur 시 (입력란을 벗어날 때)
- 체크디짓 검증 수행
  - 실패 시 에러 메시지: "유효하지 않은 주민등록번호입니다."

#### 2.2.5 외국인 신분증 (외국인 선택 시)

**신분증 선택**
- **필수 선택** (*)
- 라디오 버튼: 외국인등록번호 / 여권번호
- 기본값: 외국인등록번호

**외국인등록번호 선택 시**
- **외국인등록번호 필수 입력** (*)
- 입력 형식: 앞 6자리 - 뒤 7자리
- 앞 6자리 입력 완료 시 뒤 7자리 입력란으로 자동 포커스 이동
- **유효성 검증 시점**: Blur 시 (입력란을 벗어날 때)
- 체크디짓 검증 수행
  - 실패 시 에러 메시지: "유효하지 않은 외국인등록번호입니다."
- **국적 필수 선택** (*)
  - ISO 3166-1 alpha-2 국가 코드 리스트 제공 (194개국)

**여권번호 선택 시**
- **여권번호 필수 입력** (*)
- **국적 필수 선택** (*)
  - ISO 3166-1 alpha-2 국가 코드 리스트 제공 (194개국)
- **성별 필수 선택** (*)
  - 라디오 버튼: 남 / 여
  - 기본값: 남
- **생년월일 필수 입력** (*)
  - 형식: YYYYMMDD (8자리)
  - **유효성 검증 시점**: Blur 시 (입력란을 벗어날 때)
  - 날짜 유효성 검증
    - 실패 시 에러 메시지: "유효하지 않은 날짜입니다."

#### 2.2.6 거주구분
- **필수 선택** (*)
- 라디오 버튼: 거주자 / 비거주자
- 기본값: 거주자
- **비거주자 선택 시 거주지국 입력 필드 표시**:
  - **필수 선택** (*)
  - 셀렉트박스: ISO 3166-1 alpha-2 국가 코드 리스트 (194개국, 한글 국가명 표시)
  - 거주자로 변경 시 거주지국 값 초기화
- **필수 필드 조건**: 비거주자 선택 시에만 거주지국이 필수 필드로 활성화됨

#### 2.2.7 장애여부
- **필수 선택** (*)
- 셀렉트박스: 비장애인 / 장애인복지법상 장애인 / 국가유공자 중증환자 / 중증환자
- 기본값: 비장애인

#### 2.2.8 이메일
- **필수 입력** (*)
- **유효성 검증 시점**:
  - Blur 시: 이메일 형식 검증
    - 실패 시 에러 메시지: "이메일 형식이 맞지 않습니다."
  - Submit 시: 중복 체크 (추가 버튼 클릭)
    - 재직 중인 직원 중 중복된 이메일이 있으면 Alert 경고 메시지 표시
    - Alert 메시지: "근로자 중 중복된 이메일이 등록되어 있습니다."

#### 2.2.9 연락처
- 선택 입력
- 최대 길이: 30자
- 허용 문자: 숫자, +, (, ), -, 공백

#### 2.2.10 휴대폰번호
- 선택 입력
- 입력 형식: 3자리 - 4자리 - 4자리 (예: 010-1234-5678)
- 첫 번째 입력란 3자리 입력 완료 시 두 번째 입력란으로 자동 포커스 이동
- 두 번째 입력란 4자리 입력 완료 시 세 번째 입력란으로 자동 포커스 이동
- 숫자만 입력 가능

#### 2.2.11 입사일
- **필수 입력** (*)
- 형식: YYYYMMDD (8자리)
- **유효성 검증 시점**: Blur 시 (입력란을 벗어날 때)
- 날짜 유효성 검증
  - 실패 시 에러 메시지: "유효하지 않은 날짜입니다."
- **퇴사일과의 관계**:
  - 퇴사일이 이미 입력되어 있고, 입사일을 퇴사일 이후 날짜로 입력한 경우:
    - 입사일 입력 영역 하단에 에러 표시
    - 에러 메시지: "입사일은 퇴사일 이전 날짜만 입력할 수 있습니다."

#### 2.2.12 퇴사일
- 선택 입력
- 형식: YYYYMMDD (8자리)
- **유효성 검증 시점**: Blur 시 (입력란을 벗어날 때)
- 날짜 유효성 검증
  - 실패 시 에러 메시지: "유효하지 않은 날짜입니다."
- **입사일과의 관계**:
  - 퇴사일을 입사일 이전 날짜로 입력한 경우:
    - 퇴사일 입력 영역 하단에 에러 표시
    - 에러 메시지: "퇴사일은 입사일 이후 날짜만 입력할 수 있습니다."
- **부서장과의 관계**:
  - 부서장 체크 후 퇴사일을 오늘 이전 날짜로 입력한 경우:
    - 퇴사일 입력 영역 하단에 에러 표시
    - 에러 메시지: "퇴사일을 오늘 이전 날짜로 지정한 경우, 부서장으로 선택할 수 없습니다."
- **에러 우선순위**:
  - 여러 에러가 동시에 발생할 경우, 가장 먼저 발견된 에러 1개만 표시
  - 검증 순서: 날짜 형식 → 입사일과의 관계 → 부서장과의 관계

#### 2.2.13 주소
- 선택 입력
- '주소 검색' 버튼 클릭 시 주소 검색 다이얼로그 노출
- 우편번호, 주소: 읽기 전용 (주소 검색으로만 입력)
- 상세주소: 직접 입력 가능
- 주소 검색 완료 후 상세주소 입력란으로 자동 포커스 이동

### 2.3 조직정보 섹션

#### 2.3.0 안내문구
- 조직정보 섹션 상단에 안내문구를 표시한다.
- 문구: `※ 조직관리 메뉴에서 부서, 직급, 근로형태를 관리할 수 있습니다.`

#### 2.3.1 부서
- 선택 입력
- 셀렉트박스: 전체 부서 목록 (계층 경로 표시)
  - 표시 형식: "본사 > 개발팀 > 프론트엔드팀"
  - **데이터 출처**: 조직관리 메뉴 > 부서 관리에 저장된 리스트 순서대로 표시
- 부서 변경 시 부서장 체크박스 자동 초기화

#### 2.3.2 부서장
- 선택 입력 (체크박스)
- 부서 미선택 시 비활성화
- 체크 상태로 등록 시 중복 확인 로직 실행:
  - 해당 부서에 이미 부서장이 있는 경우 (submit):
    - Confirm 메시지: "{부서 경로}의 부서장은 {기존 부서장 이름}님으로 지정되어 있습니다. {신규 직원 이름}님으로 변경하시겠습니까?\n\n아니오 선택 시, 부서장은 {기존 부서장 이름}님으로 유지됩니다."
    - "예" 선택: 기존 부서장 해제 후 신규 직원을 부서장으로 등록
    - "아니오" 선택: 신규 직원은 부서장이 아닌 상태로 등록
- **퇴사일과의 관계**:
  - 퇴사일이 **오늘 이전**이고 부서장 체크된 경우 (blur):
    - 유효성 검증 에러 발생
    - 에러 위치: 부서장 체크 영역
    - 에러 메시지: "퇴사일을 오늘 이전 날짜로 지정한 경우, 부서장으로 선택할 수 없습니다."
  - 퇴사일이 **오늘 이후**(오늘 포함)이고 부서장 체크된 경우 (submit):
    - 저장 전 Confirm 팝업 노출 (2.6.2 참조)

#### 2.3.3 직급
- 선택 입력
- 셀렉트박스: 전체 직급 목록
  - **데이터 출처**: 조직관리 메뉴 > 직급 관리에 저장된 리스트 순서대로 표시

#### 2.3.4 근로형태
- 선택 입력
- 셀렉트박스: 전체 근로형태 목록
  - **데이터 출처**: 조직관리 메뉴 > 근로형태 관리에 저장된 리스트 순서대로 표시

### 2.4 급여정보 섹션

#### 2.4.1 연봉
- 선택 입력
- 금액 입력란:
  - 양수만 입력 가능
  - 천 단위 콤마 자동 표시
  - 숫자 이외 문자 입력 시 자동 필터링
  - 최대 12자리 (콤마 제외한 숫자 기준)
- **자동 계산 기능**:
  - 연봉 입력 시 급여항목 중 "급여" 항목이 있으면 월 급여 자동 계산
  - 계산식: `Math.round(연봉 ÷ 12)`
  - 자동 계산 후에도 월 급여 수동 변경 가능
- **불일치 경고**:
  - 연봉과 월 급여가 일치하지 않을 경우 경고 메시지 표시
  - 경고 메시지: "연봉({연봉}원)과 월 급여({월급여}원 × 12 = {월급여×12}원)가 일치하지 않습니다."
  - 경고가 표시되어도 저장 가능 (불일치 상태로 저장 허용)

#### 2.4.2 급여항목
- 선택 입력 (다중 항목 추가 가능)
- **급여항목 추가 버튼**: "급여항목 추가" 버튼 클릭 시 리스트 하단에 새로운 급여항목 입력 행 추가
- **급여항목 선택**:
  - 콤보박스 형태 (검색 가능)
  - 과세/비과세 그룹으로 구분
  - 과세 항목: 급여, 상여, 인정상여 등
  - 비과세 항목: 식대(P01), 차량유지비(P02), 육아수당(P03) 등
  - 비과세 항목은 코드와 함께 표시 (예: "P01 - 식대")
  - 검색 시 코드 또는 항목명으로 검색 가능
- **금액 입력**:
  - 양수만 입력 가능
  - 천 단위 콤마 자동 표시
  - 숫자 이외 문자 입력 시 자동 필터링
  - 최대 12자리 (콤마 제외한 숫자 기준)
- **삭제**: 각 급여항목 행의 삭제 버튼(휴지통 아이콘)으로 삭제 가능, confirm 없음 
- **유효성 검증**:
  - 급여항목만 선택하고 금액 미입력 시: 금액 입력란에 빨간 테두리 표시
  - 금액만 입력하고 급여항목 미선택 시: 급여항목 콤보박스에 빨간 테두리 표시
  - 급여항목과 금액 둘 다 비어있는 경우: 저장 시 해당 항목 제외 (에러 없음)
- **빈 항목 처리**:
  - 급여항목을 선택하지 않고 금액도 입력하지 않은 행은 저장 시 자동으로 제외됨

#### 2.4.3 계좌정보

**은행**
- 선택 입력
- 셀렉트박스: 은행 목록 (BANKS 상수 사용)

**예금주**
- 선택 입력
- 허용 문자: 한글, 영문, 숫자, 공백
- 특수문자 입력 시 자동 필터링

**계좌번호**
- 선택 입력
- 허용 문자: 숫자, 하이픈(-)
- 기타 문자 입력 시 자동 필터링

### 2.5 페이지 이탈 방지

#### 2.5.1 새로고침, 탭 닫기
- 페이지 이탈 시도 시:
  - 입력값 변경이 있었을 경우 :
    - 브라우저 기본 확인 메시지 표시 (커스텀 메시지 설정 불가)
    - "확인" 선택: 페이지 이탈
    - "취소" 선택: 현재 화면 유지
  - 입력값 변경이 없었을 경우:
    - 즉시 페이지 이탈

#### 2.5.2 브라우저 뒤로가기/앞으로가기
- 뒤로가기/앞으로가기 시도 시:
  - 입력값 변경이 있었을 경우 :
    - Confirm 메시지: "직원 추가를 취소하시겠습니까?"
    - "확인" 선택: 페이지 이동
    - "취소" 선택: 현재 화면 유지
  - 입력값 변경이 없었을 경우:
    - 즉시 페이지 이동

#### 2.5.3 메뉴 클릭 (사이드바, LNB)
- 메뉴 클릭 시:
  - 입력값 변경이 있었을 경우 :
    - Confirm 메시지: "직원 추가를 취소하시겠습니까?"
    - "확인" 선택: 메뉴 이동
    - "취소" 선택: 현재 화면 유지
  - 입력값 변경이 없었을 경우:
    - 즉시 메뉴 이동
- 현재 페이지 내부 버튼 클릭(추가, 취소 등)은 이탈 방지 대상에서 제외

### 2.6 하단 고정 버튼

#### 2.6.1 취소 버튼
- 클릭 시:
  - 입력값 변경이 있었을 경우 :
    - Confirm 메시지: "직원 추가를 취소하시겠습니까?"
    - "확인" 선택: HRIS_EMP_01 (직원관리 리스트) 화면으로 이동
    - "취소" 선택: 현재 화면 유지
  - 입력값 변경이 없었을 경우:
    - 즉시 HRIS_EMP_01 (직원관리 리스트) 화면으로 이동

#### 2.6.2 추가 버튼
- **활성화 조건**: 필수 필드가 모두 입력되고 필드별 형식 검증(blur 시점)을 통과한 경우
- **비활성화 조건**: 필수 필드 미입력 또는 형식 검증 실패 (단, 중복 체크는 submit 시점에 수행)
- 클릭 시 처리 순서:
  1. **유효성 검증**:
     - 아래 순서대로 검증을 진행하며, **가장 먼저 발견된 오류만 Alert로 표시**하고 검증을 중단한다.
     - **검증 순서**:
       1. 필수 필드 입력 확인
       2. 신분증 번호 체크디짓 검증
       3. 이메일 형식 검증
       4. **사번 중복 체크**
          - 중복 체크 대상: 전체 직원 (퇴사자 포함)
          - 중복 시 Alert 메시지: "중복된 사번은 사용하실 수 없습니다."
       5. **이메일 중복 체크**
          - 중복 시 Alert 메시지: "근로자 중 중복된 이메일이 등록되어 있습니다."
       6. **부서장 중복 체크** 
          - 해당 부서에 이미 부서장 존재 시 Confirm 메시지: "{부서 경로}의 부서장은 {기존 부서장 이름}님으로 지정되어 있습니다. {신규 직원 이름}님으로 변경하시겠습니까?\n\n아니오 선택 시, 부서장은 {기존 부서장 이름}님으로 유지됩니다."
       7. **퇴사일 + 부서장 검증**: 퇴사일이 오늘 이전이고 부서장 체크된 경우 에러 표시 (2.3.2 참조)
     - **예시**: 사번과 이메일이 모두 중복된 경우, 사번 중복 Alert만 노출하고 이메일 중복 체크는 수행하지 않음
  2. **저장 전 확인** (유효성 검증 통과 시):
     - **퇴사일이 오늘 이후이고 부서장 체크된 경우**:
       - Confirm 메시지: "퇴사일 이후에는 부서장이 체크해제됩니다. 부서장으로 추가하시겠습니까?"
       - "확인" 선택: 저장 진행
       - "취소" 선택: 폼 유지 (저장 취소)
  3. **직원 등록**:
     - 직원 등록 API 호출
     - 등록 성공 시:
       - 토스트 메시지: "직원 추가를 완료했습니다."
       - HRIS_EMP_03 (직원 상세) 화면으로 이동
     - 등록 실패 시:
       - Alert 경고 메시지: {에러 메시지}
- 제출 중일 때 버튼 텍스트: "추가 중..."
- 제출 중일 때 버튼 비활성화

- 에러가 있는 경우 폼 제출 불가

### 2.9 부서장 자동 체크 해제

- **정책**: 매일 자정에 전체 직원의 재직 상태를 확인하여, 퇴사자가 된 직원의 부서장 체크를 자동으로 해제한다.
- **판정 기준**: 퇴사일이 오늘 이전인 경우 퇴사자로 판정
- **처리 내용**:
  - 퇴사자가 된 직원의 부서장 체크를 해제
  - 해당 부서의 부서장이 "미지정" 상태로 변경됨
- **구현 시점**: 추후 서버 API 구축 시 배치 작업으로 구현 (현재는 정책만 명시)

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
