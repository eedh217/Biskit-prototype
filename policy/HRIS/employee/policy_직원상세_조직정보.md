# 직원 상세 - 조직정보 탭(HRIS_EMP_03_ORG) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_EMP_03_ORG
- **화면명**: 직원 상세 - 조직정보 탭
- **목적**: 선택한 직원의 조직정보를 조회하고 수정하는 탭
- **주요 기능**: 조직정보 조회, 조직정보 수정, 조직정보 이력 조회
- **진입 경로**: HRIS_EMP_03 (직원 상세) 화면에서 "조직정보" 탭 선택 시 표시
- **연관 화면**:
  - HRIS_EMP_03 (직원 상세) — 부모 화면
  - 조직정보 수정 다이얼로그 — '수정' 버튼 클릭 시 노출

---

## 2. 요청한 화면 기능

### 2.1 조직정보 카드

**기본 구성**
- 카드 우측 상단에 "수정" 버튼 배치
- "수정" 버튼 클릭 시: 2.2 조직정보 수정 다이얼로그 노출

#### 2.1.1 표시 필드

| 필드명 | 표시 형식 | 비고 |
|--------|-----------|------|
| 부서 | {부서 전체 경로} + 부서장 뱃지 (조건부) | 부서 전체 경로: "본사 > 개발팀 > 프론트엔드팀" 형식<br>값 없음: "-" |
| 직급 | {직급명} | 값 없음: "-" |
| 근로형태 | {근로형태명} | 값 없음: "-" |

#### 2.1.2 부서장 뱃지
- **표시 조건**: 부서장으로 지정된 경우
- **표시 위치**: 부서 전체 경로 우측

#### 2.1.3 부서 전체 경로 계산
- 선택된 부서부터 최상위 부서까지 역순으로 탐색
- " > " 구분자로 연결
- 예시: "본사 > 개발팀 > 프론트엔드팀"

### 2.2 조직정보 수정

#### 2.2.1 수정 다이얼로그 기본 정책
- **기본 기능**: HRIS_EMP_02 (직원 추가) 화면의 조직정보 영역과 동일
  - 모든 필드 구성 및 입력 규칙
  - 부서장 중복 확인 로직
  - 부서 변경 시 부서장 체크박스 초기화
  - 상세 내용은 HRIS_EMP_02 정책서 2.3 조직정보 섹션 참조

#### 2.2.2 수정 다이얼로그 차이점

**화면 형태**
- 다이얼로그 형태로 표시
- 다이얼로그 제목: "조직정보 수정"

**초기 데이터**
- **전달 데이터**: 현재 직원 정보 (employee)
- 폼 필드: 기존 직원의 조직정보로 채움

**다이얼로그 이탈 방지**
- 다이얼로그 닫기 시도 시 (X 버튼, ESC 키, 외부 클릭) 확인 메시지 표시
  - 입력값 변경이 있었을 경우 :
    - Confirm 메시지: "조직정보 수정을 취소하시겠습니까?"
    - "확인" 선택: 다이얼로그 닫기
    - "취소" 선택: 다이얼로그 유지
  - 입력값 변경이 없었을 경우:
    - 즉시 다이얼로그 닫기

**하단 버튼**
- 취소 버튼:
  - 클릭 시:
    - 입력값 변경이 있었을 경우 :
      - Confirm 메시지: "조직정보 수정을 취소하시겠습니까?"
      - "확인" 선택: 다이얼로그 닫기
      - "취소" 선택: 다이얼로그 유지
    - 입력값 변경이 없었을 경우:
      - 즉시 다이얼로그 닫기
- 수정 버튼:
  - 클릭 시 처리 순서:
    1. **유효성 검증**:
       - 필수 필드 입력 확인
       - 부서장 중복 체크 (해당 부서에 이미 부서장 존재 시 Confirm)
       - **퇴사일 + 부서장 검증**: 퇴사일이 오늘 이전이고 부서장 체크된 경우 에러 표시 (HRIS_EMP_02 정책서 2.3.2 참조)
    2. **저장 전 확인** (유효성 검증 통과 시):
       - **퇴사일이 오늘 이후이고 부서장 체크된 경우**:
         - Confirm 메시지: "퇴사일 이후에는 부서장이 체크해제됩니다. 부서장으로 수정하시겠습니까?"
         - "확인" 선택: 수정 진행
         - "취소" 선택: 다이얼로그 유지
    3. **직원 수정**:
       - 직원 수정 API 호출
       - 수정 성공 시:
         - 토스트 메시지: "조직정보 수정을 완료했습니다."
         - 다이얼로그 닫기
         - 직원 정보 새로고침
         - 조직정보 이력 새로고침
       - 수정 실패 시:
         - Alert 경고 메시지: {에러 메시지}
  - 제출 중일 때 버튼 텍스트: "수정 중..."
  - 제출 중일 때 버튼 비활성화

### 2.3 조직정보 이력

#### 2.3.1 표시 조건
- **이력 보기 토글이 ON인 경우**
- **조직정보 탭이 선택된 경우**
- 두 조건이 모두 만족해야 이력 섹션 표시

#### 2.3.2 이력 표시

**타임라인 형식**
- 각 이력 항목을 시간순으로 세로로 나열
- 수정일시 기준 내림차순 (최신순)

**이력 항목 구성**
1. **수정자 및 날짜** (상단)
   - 표시 형식: 수정자 이름 · 수정날짜
   - 수정날짜: YYYY-MM-DD HH:mm 형식
   - 예시: "홍길동 · 2024-01-15 14:30"

2. **변경 내용 카드**
   - 각 변경 사항을 행으로 나열
   - 변경 사항 표시 형식:
     - 필드명
     - 변경 전 값 (취소선 표시)
     - 화살표 아이콘 (→)
     - 변경 후 값 (강조 표시)
   - 예시: "부서    ~~개발팀~~ → 마케팅팀"

**변경 추적 필드**
- 부서: 부서 전체 경로로 표시
- 직급: 직급명으로 표시
- 근로형태: 근로형태명으로 표시
- 부서장 여부: "부서장" / "일반"으로 표시

#### 2.3.3 빈 상태
- 이력이 없는 경우: "수정 이력이 없습니다." 메시지 표시

#### 2.3.4 이력 새로고침
- **자동 새로고침 조건**: 조직정보 수정 완료 시
- **동작**: 컴포넌트 재마운트하여 최신 이력 다시 조회

#### 2.3.5 자동 이력 생성 (시스템)

**부서 삭제 시 자동 이력 생성**
- 조건: 퇴사자가 소속된 부서(또는 하위 부서)가 삭제되는 경우
- 자동 생성 내용:
  - 카테고리: 조직정보
  - 변경 항목: 부서
  - 변경 전: 부서 전체 경로 (예: "본사 > 개발팀")
  - 변경 후: **-(부서 삭제)**
  - 수정자: 시스템
  - 변경일: 부서 삭제 시점
- 표시 예시: "부서    ~~본사 > 개발팀~~ → -(부서 삭제)"

**직급 삭제 시 자동 이력 생성**
- 조건: 퇴사자가 소속된 직급이 삭제되는 경우
- 자동 생성 내용:
  - 카테고리: 조직정보
  - 변경 항목: 직급
  - 변경 전: 직급명 (예: "부장")
  - 변경 후: **-(직급 삭제)**
  - 수정자: 시스템
  - 변경일: 직급 삭제 시점
- 표시 예시: "직급    ~~부장~~ → -(직급 삭제)"

**근로형태 삭제 시 자동 이력 생성**
- 조건: 퇴사자가 소속된 근로형태가 삭제되는 경우
- 자동 생성 내용:
  - 카테고리: 조직정보
  - 변경 항목: 근로형태
  - 변경 전: 근로형태명 (예: "계약직")
  - 변경 후: **-(근로형태 삭제)**
  - 수정자: 시스템
  - 변경일: 근로형태 삭제 시점
- 표시 예시: "근로형태    ~~계약직~~ → -(근로형태 삭제)"

**자동 이력 생성 원칙**
- 재직자는 부서/직급/근로형태 삭제 불가하므로 자동 이력이 생성되지 않음
- 퇴사자만 자동 이력 대상
- 수정자는 "시스템"으로 표시되어 사용자 수정과 구분됨
- 조직정보 탭에서 이력 보기 토글 ON 시 다른 수정이력과 함께 시간순으로 표시됨

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
|--------------|--------------|------------|------|
