# 직원관리 화면(HRIS_EMP_01) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_EMP_01
- **화면명**: 직원관리 
- **목적**: 회사에 등록된 전체 직원의 리스트를 조회·관리하는 화면
- **주요 기능**: 이름/사번 검색, 직원 리스트 조회, 선택 삭제, 엑셀 다운로드, 직원 추가, 직원 상세 조회
- **진입 경로**: 인사정보 모듈 > 직원관리 메뉴 선택 시 진입
- **연관 화면**:
  - HRIS_EMP_02 (직원 추가) — '추가하기' 버튼 클릭 시 이동
  - HRIS_EMP_03 (직원 상세) — 행 클릭 시 이동

---

## 2. 요청한 화면 기능

### 2.1 검색영역

- 구성: 인풋박스 + 검색 버튼
- 검색항목: **이름 또는 사번**
- placeholder 문구: `이름 또는 사번을 입력해주세요.`
- Enter 입력 시 검색 실행 (공통 정책 1.1 권장사항 적용)
- 검색 방식: **완전 일치** — 입력된 검색어가 이름 또는 사번과 정확히 일치하는 경우에만 조회
- 검색 인풋박스 빈 값/공백 처리:
  - 빈 값 또는 공백만 입력 시 전체 조회
  - 앞뒤 공백은 자동 trim 처리
- 검색 시 하단 리스트 및 리스트 개수를 검색 결과 기준으로 갱신한다.
- 검색 실행 시 페이지네이션 1페이지로 초기화 (공통 정책 1.1 적용)
- 검색 실행 시 체크박스 선택 상태 모두 초기화 (공통 정책 1.1 적용)
- **검색 실행 후 입력 필드 포커스 유지** (공통 정책 1.1 적용)

### 2.2 리스트 개수(카운트)

- 공통 정책 1.2 적용
- 기본: 등록된 총 리스트 개수 제공
- 검색 상태: 검색 결과 개수 제공
- **표시 형식**: "N개" (예: "50개")
- 위치: 리스트 상단 좌측

### 2.3 리스트(테이블)

- 테이블 컴포넌트 사용 (공통 정책 1.3 적용)
- 기본 정렬: 최근 등록 순 (공통 정책 1.3 적용)
- 가장 좌측에 "행 선택" 체크박스 제공(멀티 선택) (공통 정책 1.3 적용)
- 헤더에 전체 선택/해제 체크박스 제공 (현재 페이지 전체 선택) (공통 정책 1.3 적용)
- 체크박스 영역 클릭 시 체크박스만 토글하고, 행 클릭(상세 화면)은 발생하지 않는다.
- **컬럼 고정**: 체크박스 컬럼과 이름 컬럼은 좌측 고정 (스크롤 시에도 고정)
- 컬럼 정의:

| 컬럼명 | 표시형식 | 정렬 | 비고 |
|--------|---------|------|------|
| 이름 | 텍스트 | 좌측 | 좌측 고정 컬럼 |
| 사번 | 텍스트 | 좌측 | |
| 재직상태 | "재직중" / "퇴사" | 좌측 | |
| 입사일 | 날짜 (YYYY-MM-DD) | 좌측 | |
| 이메일 | 텍스트 | 좌측 | |
| 휴대폰번호 | 텍스트 | 좌측 | 하이픈(-) 포함 형식 (예: 010-1234-5678) |
| 직급 | 텍스트 | 좌측 | 직급 미지정 시 "-" 표시 |
| 부서 | 텍스트 | 좌측 | 전체 경로 표시 (예: "본사 > 개발팀 > 프론트엔드팀"), 부서 미지정 시 "-" 표시 |

- **재직상태 판정 기준**:
  - 재직중: 퇴사일이 없거나 퇴사일이 오늘 이후인 경우
  - 퇴사: 퇴사일이 오늘 이전인 경우

### 2.4 리스트 개수 선택(페이지 사이즈)

- 공통 정책 1.4 적용
- **위치: 페이지네이션 영역 내부 (테이블 하단)**
- 셀렉박스 제공
- 옵션: 15개 / 30개 / 50개 / 100개
- 기본 선택: 30개
- 페이지 사이즈 변경 시 현재 보고 있던 데이터 위치를 유지한다. (공통 정책 1.4 적용)

### 2.5 엑셀 다운로드

- 공통 정책 1.5 적용
- 다운로드 범위 규칙:
  1. 검색하지 않았을 경우: 전체 리스트 엑셀 다운로드
  2. 검색했을 경우: 검색 결과 리스트 엑셀 다운로드
  3. 리스트 선택 후 버튼 클릭 시: 선택한 리스트만 엑셀 다운로드
- 엑셀 정렬은 화면과 동일
- 데이터 0건일 경우: "다운로드할 데이터가 없습니다" 안내 메시지 표시 (공통 정책 1.5 적용)
- **파일명 규칙**: `직원관리` (예: "직원관리.xlsx")
- **엑셀 포함 컬럼**: 이름, 사번, 재직상태, 입사일, 이메일, 휴대폰번호, 직급, 부서 (화면과 동일)

### 2.6 선택 삭제

- 공통 정책 1.6 적용
- 좌측 체크박스 선택 시 활성화
- 데이터 0건일 경우 삭제 버튼 비활성화 (공통 정책 1.6 적용)
- 버튼 클릭 시 confirm 노출
  - 문구: "총 N개의 리스트를 삭제하시겠습니까?" (N = 선택한 행 개수)
- **삭제 방식: 물리삭제** (LocalStorage에서 완전 삭제)
- 삭제 성공 시 토스트 1.5초 노출 — 문구: "삭제가 완료되었습니다." (공통 정책 1.6 적용)
- 삭제 후 리스트/카운트를 최신 상태로 갱신한다.
- 삭제 실패 시 처리 정책 (공통 정책 1.6 적용):
  - 건별 트랜잭션 적용, 부분 성공 허용
  - 실패 시 에러 메시지 표시: "삭제 실패" + 에러 내용

### 2.7 직원 추가

- **위치: 리스트 상단 우측 (삭제/다운로드 버튼 영역)**
- 버튼명: "추가하기"
- 버튼 클릭 시 HRIS_EMP_02 '직원 추가' 화면으로 이동한다.

### 2.8 행 클릭

- 행 클릭 시 HRIS_EMP_03 '직원 상세' 화면으로 이동한다. (공통 정책 1.3 적용)
- 이동 방식: `/hr/employee/{employeeId}` URL로 이동
- 체크박스 영역 클릭 시 행 클릭 이벤트는 발생하지 않는다.

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
|--------------|--------------|------------|------|
| 1.1 검색 | 검색 실행 후 입력 필드 포커스 유지 | 그대로 적용 | 충돌 없음 |
| 1.2 리스트 개수 | 카운트 표시 | 그대로 적용 | 충돌 없음 |
| 1.3 리스트(테이블) | 기본 정렬, 체크박스, 행 클릭 | 그대로 적용 | 충돌 없음 |
| 1.4 페이지 사이즈 | 페이지네이션 영역에 셀렉박스 배치 | 그대로 적용 | 충돌 없음 |
| 1.5 엑셀 다운로드 | 다운로드 범위 규칙 | 그대로 적용 | 충돌 없음 |
| 1.6 선택 삭제 | 물리삭제, confirm 문구 | 그대로 적용 | 충돌 없음 |

> 본 화면은 공통 정책과 대부분 일치하며, 주요 충돌 없이 공통 정책을 그대로 적용한다.
