# 직원 상세 - 급여정보 탭(HRIS_EMP_03_PAY) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_EMP_03_PAY
- **화면명**: 직원 상세 - 급여정보 탭
- **목적**: 선택한 직원의 급여정보를 조회하고 수정하는 탭
- **주요 기능**: 급여정보 조회, 급여정보 수정, 급여정보 이력 조회
- **진입 경로**: HRIS_EMP_03 (직원 상세) 화면에서 "급여정보" 탭 선택 시 표시
- **연관 화면**:
  - HRIS_EMP_03 (직원 상세) — 부모 화면
  - 급여정보 수정 다이얼로그 — '수정' 버튼 클릭 시 노출

---

## 2. 요청한 화면 기능

### 2.1 급여정보 카드

**기본 구성**
- 카드 우측 상단에 "수정" 버튼 배치
- "수정" 버튼 클릭 시: 2.2 급여정보 수정 다이얼로그 노출

#### 2.1.1 표시 필드

| 필드명 | 비고 |
|--------|-----------|------|
| 계약급여 타입 | "연봉" / "시급"<br>값 없음: "-" |
| 계약급여 금액 | 천 단위 콤마<br>값 없음: "-" |
| 식대 | 천 단위 콤마<br>급여 타입이 연봉일 경우, "(월)"<br>급여 타입이 시급일 경우, "(일)"<br>급여 타입 없음: 단위 표시 안 함<br>값 없음: "-" |
| 은행 | 값 없음: "-" |
| 예금주 | 값 없음: "-" |
| 계좌번호 | 값 없음: "-" |

### 2.2 급여정보 수정

#### 2.2.1 수정 다이얼로그 기본 정책
- **기본 기능**: HRIS_EMP_02 (직원 추가) 화면의 급여정보 영역과 동일
  - 모든 필드 구성 및 입력 규칙
  - 급여 타입 변경 시 식대 초기화
  - 예금주: 한글/영문/숫자/공백만 허용
  - 계좌번호: 숫자와 하이픈(-)만 허용
  - 상세 내용은 HRIS_EMP_02 정책서 2.4 급여정보 섹션 참조

#### 2.2.2 수정 다이얼로그 차이점

**화면 형태**
- 다이얼로그 형태로 표시
- 다이얼로그 제목: "급여정보 수정"

**초기 데이터**
- **전달 데이터**: 현재 직원 정보 (employee)
- 폼 필드: 기존 직원의 급여정보로 채움
- 금액 필드: 천 단위 콤마 표시

**다이얼로그 이탈 방지**
- 다이얼로그 닫기 시도 시 (X 버튼, ESC 키, 외부 클릭) 확인 메시지 표시
  - 입력값 변경이 있었을 경우 :
    - Confirm 메시지: "급여정보 수정을 취소하시겠습니까?"
    - "확인" 선택: 다이얼로그 닫기
    - "취소" 선택: 다이얼로그 유지
  - 입력값 변경이 없었을 경우:
    - 즉시 다이얼로그 닫기

**하단 버튼**
- 취소 버튼:
  - 클릭 시:
    - 입력값 변경이 있었을 경우 :
      - Confirm 메시지: "급여정보 수정을 취소하시겠습니까?"
      - "확인" 선택: 다이얼로그 닫기
      - "취소" 선택: 다이얼로그 유지
    - 입력값 변경이 없었을 경우:
      - 즉시 다이얼로그 닫기
- 수정 버튼:
  - 클릭 시:
    - 직원 수정 API 호출
    - 수정 성공 시:
      - 토스트 메시지: "급여정보 수정을 완료했습니다."
      - 다이얼로그 닫기
      - 직원 정보 새로고침
      - 급여정보 이력 새로고침
    - 수정 실패 시:
      - Alert 경고 메시지: {에러 메시지}
  - 제출 중일 때 버튼 텍스트: "수정 중..."
  - 제출 중일 때 버튼 비활성화

### 2.3 급여정보 이력

#### 2.3.1 표시 조건
- **이력 보기 토글이 ON인 경우**
- **급여정보 탭이 선택된 경우**
- 두 조건이 모두 만족해야 이력 섹션 표시

#### 2.3.2 이력 표시

**타임라인 형식**
- 각 이력 항목을 시간순으로 세로로 나열
- 수정일시 기준 내림차순 (최신순)

**이력 항목 구성**
1. **수정자 및 날짜** (상단)
   - 표시 형식: 수정자 이름 · 수정날짜
   - 수정날짜: YYYY-MM-DD HH:mm 형식
   - 예시: "홍길동 · 2024-01-15 14:30"

2. **변경 내용 카드**
   - 각 변경 사항을 행으로 나열
   - 변경 사항 표시 형식:
     - 필드명
     - 변경 전 값 (취소선 표시)
     - 화살표 아이콘 (→)
     - 변경 후 값 (강조 표시)
   - 예시: "계약급여 금액    ~~30,000,000원~~ → 35,000,000원"

**변경 추적 필드**
- 계약급여 타입: "연봉" / "시급"으로 표시
- 계약급여 금액: 천 단위 콤마 + "원"
- 식대: 천 단위 콤마 + "원" + 단위 (급여 타입에 따라 "(월)" 또는 "(일)")
- 은행: 은행명
- 예금주: 예금주 이름
- 계좌번호: 계좌번호

#### 2.3.3 빈 상태
- 이력이 없는 경우: "수정 이력이 없습니다." 메시지 표시

#### 2.3.4 이력 새로고침
- **자동 새로고침 조건**: 급여정보 수정 완료 시
- **동작**: 컴포넌트 재마운트하여 최신 이력 다시 조회
