# 부서관리 화면(HRIS_ORG_01) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_ORG_01
- **화면명**: 부서관리
- **목적**: 회사의 조직도를 계층형 트리 구조로 관리하고, 각 부서의 소속 인원 및 통계를 조회하는 화면
- **주요 기능**: 부서 트리 조회/검색, 부서 추가/수정/삭제, 순서 변경, 부서별 직원 목록 조회, 조직 통계
- **진입 경로**: 인사정보 모듈 > 부서 메뉴 선택 시 진입
- **연관 화면**: 없음

---

## 2. 요청한 화면 기능

### 2.1 레이아웃

- **2패널 구조**: 좌측 패널(부서 트리) + 우측 패널(부서 상세)
- **최초 진입 시**: 가장 최상위 부서(첫 번째 부서) 자동 선택

### 2.2 좌측 패널 - 검색 영역

- **위치**: 트리 상단
- **구성**: 검색 아이콘 + 인풋박스
- **placeholder**: "부서명 검색"
- **검색 방식**: 부분 일치 (입력된 문자열이 부서명에 포함되면 매칭)
- **검색 결과 표시**: 매칭된 부서와 그 부모 부서들을 트리 구조로 표시
- **검색 결과 없을 경우**: "검색 결과가 없습니다."

### 2.3 좌측 패널 - 부서 트리

#### 2.3.1 트리 구조

- **계층형 구조**: 최상위 부서 → 하위 부서 → 하위 부서...
- **Depth 제한**: 최대 5단계 (depth 0~4)
  - depth 0: 최상위 부서
  - depth 1~4: 하위 부서
  - depth 4에서는 하위 부서 추가 불가
- **표시 형식**: `부서명 (직원 수)`
  - 예: "개발팀 (15)"
  - 직원 수는 **재직중인 직원만** 카운트
- **선택 상태**: 선택된 부서는 파란색 배경(`bg-blue-50`) 표시
- **Expand/Collapse**:
  - 하위 부서가 있는 경우 펼침/접기 아이콘 표시
  - 클릭 시 하위 부서 표시/숨김 토글
  - 기본 상태: 모두 펼쳐진 상태

#### 2.3.2 부서 선택

- **클릭 시**: 해당 부서 선택 + 우측 패널에 상세 정보 표시
- **선택 표시**: 파란색 배경

#### 2.3.3 부서 추가

##### 최상위 부서 추가

- **위치**: 검색영역 우측
- **동작**:
  1. 버튼 클릭 시 트리 하단에 입력 필드 생성
  2. 포커스 자동 이동
  3. 부서명 입력 후 Enter 키 또는 포커스 아웃(blur) 시 저장
  4. Escape 키 시 취소
- **검증**:
  - 빈 값 또는 공백만 입력 시: 취소 처리 (에러 표시 없음)
  - 중복 체크: 같은 레벨(최상위)에 동일한 부서명이 있으면 입력 필드 하단에 에러 표시 (blur 또는 엔터 시)
    - 에러 문구: "같은 레벨에 동일한 이름의 조직이 이미 존재합니다."
  - 최대 길이: 50자
- **저장 후 동작**:
  - 좌측 트리만 갱신 (새로 추가된 부서 표시)
  - 우측 패널(부서 상세)은 유지 (현재 선택된 부서 변경 없음)

##### 하위 부서 추가

- **진입 방법**: 부서 아이템 호버 시 "+" 아이콘 표시 → 클릭
- **Depth 제한**: depth 4인 부서에서는 "+" 아이콘 표시 안 됨 (최대 5단계 제한)
- **동작**:
  1. 해당 부서의 하위 레벨에 입력 필드 생성
  2. 포커스 자동 이동
  3. 부서명 입력 후 Enter 키 또는 포커스 아웃(blur) 시 저장
  4. Escape 키 시 취소
- **검증**: 최상위 부서 추가와 동일 (단, 같은 부모 아래의 형제 부서끼리만 중복 체크)
  - 에러 문구: "같은 레벨에 동일한 이름의 조직이 이미 존재합니다."
- **저장 후 동작**:
  - 좌측 트리만 갱신 (새로 추가된 부서 표시)
  - 우측 패널(부서 상세)은 유지 (현재 선택된 부서 변경 없음)

#### 2.3.4 부서 수정

- **진입 방법**: 부서 아이템 호버 시 연필 아이콘 표시 → 클릭
- **동작**:
  1. 부서명이 입력 필드로 변경
  2. 기존 부서명이 선택된 상태로 포커스 이동
  3. 수정 후 Enter 키 또는 포커스 아웃(blur) 시 저장
  4. Escape 키 시 취소
- **검증**: 부서 추가와 동일
- **취소 처리**: 빈 값 또는 공백만 입력 시 수정 취소 (원래 값 유지)
- **저장 후 동작**:
  - 좌측 트리만 갱신 (수정된 부서명 반영)
  - 우측 패널(부서 상세)은 유지 (현재 선택된 부서가 수정된 경우에도 재로드 없이 유지)

#### 2.3.5 부서 삭제

- **진입 방법**: 부서 아이템 호버 시 휴지통 아이콘 표시 → 클릭

##### 삭제 로직

**1. 하위 부서가 없을 경우**
- **소속 직원 없음**: 삭제 가능
  - Confirm 메시지: `'{부서명}' 부서를 삭제하시겠습니까?`
  - 확인 시 물리 삭제 (LocalStorage에서 완전 삭제)
- **소속 직원 있음**: 삭제 불가
  - Alert 메시지: `소속 직원이 존재하여 삭제할 수 없습니다.`

**2. 하위 부서가 있을 경우**
- **삭제 대상**: 선택한 부서 + 모든 하위 부서 (재귀적으로 전체 삭제)
- **삭제하려는 부서와 모든 하위 부서에 소속 직원 없음**: 삭제 가능
  - Confirm 메시지: `'{부서명}' 부서와 하위부서를 모두 삭제하시겠습니까?`
  - 확인 시 물리 삭제 (선택한 부서 + 모든 하위 부서 완전 삭제)
- **하위 부서 중 하나라도 소속 직원 있음**: 삭제 불가
  - Alert 메시지: `하위부서에 소속직원이 존재하여 삭제할 수 없습니다.`

##### 소속 직원 판정 기준

- **재직중인 직원만** 카운트 (퇴사일이 없거나 퇴사일이 오늘 이후)

##### 삭제 후 처리

- 삭제된 부서가 현재 선택된 부서인 경우: 첫 번째 부서 자동 선택
- 좌측 트리만 갱신 (로딩 표시 없음)
- 우측 패널은 새로 선택된 부서 정보 표시

##### 퇴사자 이력 처리

- 삭제된 부서(및 하위 부서)에 소속된 퇴사자가 있는 경우, 각 퇴사자의 조직정보 수정이력에 자동으로 기록 추가
  - 카테고리: 조직정보
  - 변경 항목: 부서
  - 변경 전: 부서 전체 경로 (예: "본사 > 개발팀")
  - 변경 후: **-(부서 삭제)**
  - 수정자: 시스템
  - 변경일: 부서 삭제 시점

#### 2.3.6 부서 순서 변경 (DnD)

- **기능**: 드래그 앤 드롭으로 같은 레벨 내에서 부서 순서 변경
- **제약 조건**:
  - 같은 부모를 가진 부서끼리만 순서 변경 가능
  - 다른 레벨로 이동 불가
- **불가능한 동작 시 처리**: alert 표시 — "같은 레벨 내에서만 순서를 변경할 수 있습니다."
- **저장**: 순서 변경 즉시 LocalStorage에 저장

### 2.4 좌측 패널 - 빈 상태 (Empty State)

- **조건**: 등록된 부서가 하나도 없는 경우
- **표시 문구**:
  - "등록된 부서가 없습니다."
  - "추가하기 버튼을 눌러 부서를 등록해보세요." (작은 글씨)

### 2.5 우측 패널 - 선택 전 상태

- **표시 문구**: "부서를 선택하면 상세 정보가 표시됩니다."
- **위치**: 화면 중앙

### 2.6 우측 패널 - 소속 인원 정보 카드

#### 2.6.1 카드 헤더

- **제목**: "소속 인원 정보"
- **부제**: "* 재직중인 직원만 표시됩니다."

#### 2.6.2 기본 정보

- **부서장**:
  - 표시 형식: "{이름} ({사번})"
  - 부서장이 없는 경우: "미지정"
  - 부서장 퇴사 시 처리: 부서장인 직원이 퇴사하게 되는 경우, 부서장 체크 해제되고 부서장 미지정으로 변경됨
- **소속 인원 수**:
  - 표시 형식: "{N}명"
  - 카운트 대상: 해당 부서 및 모든 하위 부서의 재직중인 직원

#### 2.6.3 소속 직원 목록 테이블

- **제목**: "소속 직원 목록"
- **대상**: 해당 부서 및 **모든 하위 부서**의 재직중인 직원
- **컬럼 구성**:

| 컬럼명 | 표시형식 | 정렬 | 비고 |
|--------|---------|------|------|
| 사번 | 텍스트 | 좌측 | |
| 이름 | 텍스트 | 좌측 | |
| 부서 | 텍스트 | 좌측 | 전체 경로 표시 (예: "본사 > 개발팀 > 프론트엔드팀") |
| 직급 | 텍스트 | 좌측 | 직급 미지정 시 "-" 표시 |
| 이메일 | 텍스트 | 좌측 | |
| 전화번호 | 텍스트 | 좌측 | 하이픈(-) 포함 형식 (예: 010-1234-5678) |

- **정렬 기준** (우선순위 순):
  1. 부서 depth 오름차순 (상위 부서가 먼저)
  2. 같은 depth 내에서 부서 order 오름차순
  3. 같은 부서 내에서 사번 오름차순

- **무한 스크롤**:
  - 초기 표시: 20개
  - 스크롤 시: 20개씩 추가 로드
  - 최대 개수: 제한 없음 (전체 직원 표시)
  - 높이: 화면 아래까지 꽉 차게 표시

- **빈 상태**:
  - 조건: 소속 직원이 없는 경우
  - 표시 문구: "소속 직원이 없습니다."

### 2.7 우측 패널 - 조직 통계 카드

#### 2.7.1 카드 헤더

- **제목**: "조직 통계"

#### 2.7.2 기본 통계

- **하위 부서 개수**:
  - 표시 형식: "{N}개"
  - 카운트 대상: 해당 부서 아래의 모든 하위 부서 (재귀적)
- **전체 인원 수 (하위 부서 포함)**:
  - 표시 형식: "{N}명"
  - 카운트 대상: 해당 부서 및 모든 하위 부서의 재직중인 직원

#### 2.7.3 직급별 인원 분포

- **제목**: "직급별 인원 분포"
- **표시 형식**: "{직급명}: {N}명" (예: "부장: 3명, 과장: 5명")
- **정렬**: 직급 순서(order) 기준
- **카운트 대상**: 해당 부서 및 모든 하위 부서의 재직중인 직원
- **직급 미지정 직원**: "미지정: {N}명"으로 표시

#### 2.7.4 고용형태별 인원 분포

- **제목**: "고용형태별 인원 분포"
- **표시 형식**: "{고용형태명}: {N}명" (예: "정규직: 10명, 계약직: 2명")
- **정렬**: 고용형태 순서(order) 기준
- **카운트 대상**: 해당 부서 및 모든 하위 부서의 재직중인 직원
- **고용형태 미지정 직원**: "미지정: {N}명"으로 표시

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
|--------------|--------------|------------|------|
| 재직 상태 판정 | 퇴사일 기준으로 재직중/퇴사 판정 | 그대로 적용 | 충돌 없음 |
| 전화번호 표시 | 하이픈(-) 포함 형식 (예: 010-1234-5678) | 그대로 적용 | 충돌 없음 |
| 부서 경로 표시 | " > "로 구분된 전체 경로 (예: "본사 > 개발팀 > 프론트엔드팀") | 그대로 적용 | 충돌 없음 |
| 토스트 메시지 | 성공/실패 시 1.5초 노출 | 그대로 적용 | 충돌 없음 |
| 물리 삭제 | LocalStorage에서 완전 삭제 | 그대로 적용 | 충돌 없음 |

> 본 화면은 공통 정책과 대부분 일치하며, 주요 충돌 없이 공통 정책을 그대로 적용한다.
