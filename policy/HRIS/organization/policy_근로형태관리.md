# 근로형태관리 화면(HRIS_ORG_03) - 정책서

---

## 1. 화면 개요

- **화면 ID**: HRIS_ORG_03
- **화면명**: 근로형태관리
- **목적**: 회사의 근로형태 코드를 관리하고, 각 근로형태의 소속 인원 정보를 조회하는 화면
- **주요 기능**: 근로형태 리스트 조회, 근로형태 추가/수정/삭제, 순서 변경, 근로형태별 직원 목록 조회
- **진입 경로**: 인사정보 모듈 > 근로형태 메뉴 선택 시 진입
- **연관 화면**: 없음

---

## 2. 요청한 화면 기능

### 2.1 레이아웃

- **2패널 구조**: 좌측 패널(근로형태 리스트) + 우측 패널(근로형태 상세)
- **최초 진입 시**: 가장 첫 번째 근로형태 자동 선택

### 2.2 좌측 패널 - 근로형태 리스트

#### 2.2.1 리스트 헤더

- **총 개수 표시**: "총 N개" 형식으로 좌측 상단에 표시
- **추가하기 버튼**: 우측 상단에 배치
  - 버튼명: "추가하기"
  - 아이콘: Plus 아이콘
  - 근로형태 추가 모드 진입 시 버튼 숨김

#### 2.2.2 근로형태 아이템

- **표시 형식**: `근로형태명 (직원 수)`
  - 예: "정규직 (15)"
  - 직원 수는 **재직중인 직원만** 카운트 (퇴사일이 없거나 퇴사일이 오늘 이후)
- **선택 상태**: 선택된 근로형태는 파란색 배경(`bg-blue-50`) 표시
- **드래그 핸들**: 좌측에 GripVertical 아이콘 표시 (순서 변경용)
- **호버 상태**: 마우스 오버 시 수정/삭제 아이콘 표시
  - 연필 아이콘: 수정 기능
  - 휴지통 아이콘: 삭제 기능 (빨간색)

#### 2.2.3 근로형태 선택

- **클릭 시**: 해당 근로형태 선택 + 우측 패널에 상세 정보 표시
- **선택 표시**: 파란색 배경
- **수정 모드 중**: 클릭 이벤트 비활성화

### 2.3 근로형태 추가

- **진입 방법**: "추가하기" 버튼 클릭
- **동작**:
  1. 리스트 하단에 입력 필드 생성
  2. 포커스 자동 이동
  3. placeholder: "근로형태명 입력 후 Enter"
  4. 근로형태명 입력 후 Enter 키 또는 포커스 아웃(blur) 시 저장
  5. Escape 키 시 취소
- **검증**:
  - 빈 값 또는 공백만 입력 시: 취소 처리 (에러 표시 없음)
  - 중복 체크: 동일한 근로형태명이 있으면 입력 필드 하단에 에러 표시 (blur 또는 엔터 시)
    - 에러 문구: "이미 동일한 근로형태가 등록되어 있습니다."
    - 중복 체크 시 대소문자 무시
  - 최대 길이: 20자
- **저장 후 동작**:
  - 추가 모드 종료
  - 좌측 리스트 갱신
  - 우측 패널은 현재 선택된 근로형태 유지
  - 새로 추가된 근로형태는 리스트 맨 하단에 표시 (order 자동 증가)

### 2.4 근로형태 수정

- **진입 방법**: 근로형태 아이템 호버 시 연필 아이콘 표시 → 클릭
- **동작**:
  1. 해당 근로형태명이 입력 필드로 변경
  2. 기존 근로형태명이 선택된 상태로 포커스 이동
  3. 수정 후 Enter 키 또는 포커스 아웃(blur) 시 저장
  4. Escape 키 시 취소
- **검증**: 근로형태 추가와 동일
  - 중복 체크 시 자기 자신 제외
- **취소 처리**: 빈 값 또는 공백만 입력 시 수정 취소 (원래 값 유지)
- **저장 후 동작**:
  - 수정 모드 종료
  - 좌측 리스트 갱신
  - 우측 패널은 현재 선택된 근로형태 유지

### 2.5 근로형태 삭제

- **진입 방법**: 근로형태 아이템 호버 시 휴지통 아이콘 표시 → 클릭

#### 삭제 로직

- **소속 직원 없음**: 삭제 가능
  - Confirm 메시지: `'{근로형태명}' 근로형태를 삭제하시겠습니까?`
  - 확인 시 물리 삭제 (LocalStorage에서 완전 삭제)
- **소속 직원 있음**: 삭제 불가
  - Alert 메시지: `소속 직원이 존재하여 삭제할 수 없습니다.`

#### 소속 직원 판정 기준

- **재직중인 직원만** 카운트 (퇴사일이 없거나 퇴사일이 오늘 이후)

#### 삭제 후 처리

- 삭제된 근로형태가 현재 선택된 근로형태인 경우: 첫 번째 근로형태 자동 선택
- 좌측 리스트 갱신
- 근로형태가 하나도 없는 경우: 빈 상태 표시

#### 퇴사자 이력 처리

- 해당 근로형태에 소속된 퇴사자가 있는 경우, 각 퇴사자의 조직정보 수정이력에 자동으로 기록 추가
  - 카테고리: 조직정보
  - 변경 항목: 근로형태
  - 변경 전: 근로형태명
  - 변경 후: **-(근로형태 삭제)**
  - 수정자: 시스템
  - 변경일: 근로형태 삭제 시점

### 2.6 순서 변경 (DnD)

- **기능**: 드래그 앤 드롭으로 근로형태 순서 변경
- **드래그 핸들**: 좌측 GripVertical 아이콘
- **드래그 중**: 해당 아이템 투명도 50% (opacity: 0.5)
- **저장**: 순서 변경 즉시 LocalStorage에 저장
- **저장 후 동작**: 리스트 갱신, 현재 선택된 근로형태 유지

### 2.7 좌측 패널 - 빈 상태 (Empty State)

- **조건**: 등록된 근로형태가 하나도 없고, 추가 모드가 아닌 경우
- **표시 문구**:
  - "등록된 근로형태가 없습니다."
  - "추가하기 버튼을 눌러 근로형태를 등록해보세요." (작은 글씨)

### 2.8 우측 패널 - 선택 전 상태

- **표시 문구**: "근로형태를 선택하면 상세 정보가 표시됩니다."
- **위치**: 화면 중앙

### 2.9 우측 패널 - 소속 인원 정보 카드

#### 2.9.1 카드 헤더

- **제목**: "소속 인원 정보"
- **부제**: "* 재직중인 직원만 표시됩니다."

#### 2.9.2 기본 정보

- **근로형태명**:
  - 표시 형식: "{근로형태명}"
- **소속 인원 수**:
  - 표시 형식: "{N}명"
  - 카운트 대상: 해당 근로형태의 재직중인 직원

#### 2.9.3 소속 직원 목록 테이블

- **제목**: "소속 직원 목록"
- **대상**: 해당 근로형태의 재직중인 직원
- **컬럼 구성**:

| 컬럼명 | 표시형식 | 정렬 | 비고 |
|--------|---------|------|------|
| 사번 | 텍스트 | 좌측 | |
| 이름 | 텍스트 | 좌측 | |
| 부서 | 텍스트 | 좌측 | 전체 경로 표시 (예: "본사 > 개발팀 > 프론트엔드팀"), 부서 미지정 시 "-" 표시 |
| 직급 | 텍스트 | 좌측 | 직급 미지정 시 "-" 표시 |
| 이메일 | 텍스트 | 좌측 | |
| 전화번호 | 텍스트 | 좌측 | 하이픈(-) 포함 형식 (예: 010-1234-5678) |

- **정렬 기준** (우선순위 순):
  1. 부서 depth 오름차순 (상위 부서가 먼저)
  2. 같은 depth 내에서 부서 order 오름차순
  3. 같은 부서 내에서 사번 오름차순
  4. 부서가 없는 직원은 맨 뒤로

- **무한 스크롤**:
  - 초기 표시: 20개
  - 스크롤 시: 20개씩 추가 로드
  - 최대 개수: 제한 없음 (전체 직원 표시)
  - 높이: 화면 아래까지 꽉 차게 표시

- **빈 상태**:
  - 조건: 소속 직원이 없는 경우
  - 표시 문구: "소속 직원이 없습니다."

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
|--------------|--------------|------------|------|
| 재직 상태 판정 | 퇴사일 기준으로 재직중/퇴사 판정 (퇴사일이 없거나 퇴사일이 오늘 이후) | 그대로 적용 | 충돌 없음 |
| 전화번호 표시 | 하이픈(-) 포함 형식 (예: 010-1234-5678) | 그대로 적용 | 충돌 없음 |
| 부서 경로 표시 | " > "로 구분된 전체 경로 (예: "본사 > 개발팀 > 프론트엔드팀") | 그대로 적용 | 충돌 없음 |
| 물리 삭제 | LocalStorage에서 완전 삭제 | 그대로 적용 | 충돌 없음 |

> 본 화면은 공통 정책과 대부분 일치하며, 주요 충돌 없이 공통 정책을 그대로 적용한다.
