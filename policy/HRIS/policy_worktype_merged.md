# 근로형태 관리 화면 - 최종 정책서

---

## 1. 화면 개요

- 모듈명: 근로형태 관리
- 기능: 근로형태 추가 / 수정 / 삭제
- 화면 ID
  - 리스트: HRIS_WT_01
  - 추가 팝업: HRIS_WT_02
  - 수정 팝업: HRIS_WT_03

---

## 2. 리스트 화면 (HRIS_WT_01)

### 2.1 검색영역
- 구성: 인풋박스 + 검색 버튼 `[공통]`
- 검색 대상: 근로형태명 OR 근로형태 코드 `[화면별]`
- 검색 입력 제한 `[화면별]`
  - 최대 10자
  - 특수문자 입력 불가
- placeholder: `근로형태명 또는 근로형태 코드를 입력해주세요.` `[공통]`
- Enter 입력 시 검색 실행 `[공통 권장]`

### 2.2 리스트 개수(카운트)
- 텍스트로 제공 `[공통]`
- 기본: 등록된 총 리스트 개수 제공 `[공통]`
- 검색 상태: 검색 결과 개수 제공 `[공통]`

### 2.3 리스트(테이블)
- 테이블 컴포넌트 사용 `[공통]`
- 기본 정렬: 최근 등록 순 `[공통]`
- 행 클릭 시 수정 팝업(HRIS_WT_03) 노출 `[공통]`
  - 선택 영역: 행 전체 `[공통]`
  - 전달 키값: 별도 DB PK(id) `[공통]`
- 가장 좌측에 "행 선택" 체크박스 제공(멀티 선택) `[공통]`

#### 테이블 컬럼 `[화면별]`

| 컬럼명 | 타입 | 비고 |
|--------|------|------|
| 근로형태명 | 문자열 | 최대 10자 |
| 근로형태 코드 | 문자열 | 최대 10자 |

> 추가 컬럼 없음. 위 2개 컬럼만 사용

### 2.4 리스트 개수 선택(페이지 사이즈)
- 셀렉박스 제공 `[공통]`
- 옵션: 15개 / 30개 / 50개 / 100개 `[공통]`
- 기본 선택: 30개 `[공통]`

### 2.5 엑셀 다운로드
- 버튼 제공 `[공통]`
- 파일명: `HRIS_근로형태_리스트` `[화면별]`
- 다운로드 범위 규칙 `[공통]`
  1. 검색하지 않았을 경우: 전체 리스트 엑셀 다운로드
  2. 검색했을 경우: 검색 결과 리스트 엑셀 다운로드
  3. 리스트 선택 후 버튼 클릭 시: 선택한 리스트만 엑셀 다운로드
- 엑셀 정렬(sorting)은 화면과 동일 `[공통]`

### 2.6 선택 삭제
- 버튼 제공 `[공통]`
- 좌측 체크박스 선택 시 활성화 `[공통]`
- 버튼 클릭 시 confirm 노출 `[공통]`
  - 문구: `총 N개의 리스트를 삭제하시겠습니까?` `[공통]`
- 삭제 후 리스트/카운트 UI는 최신 상태로 갱신 `[공통]`
- **삭제 정책** `[화면별]`
  - 물리삭제
  - 삭제 후 복구 불가
  - **삭제된 근로형태를 사용 중인 임직원의 근로형태 값을 NULL로 변경** `[화면별]`
- 삭제 시 임직원 연관 경고 confirm 노출 `[화면별]`
  - 삭제 대상 근로형태를 사용 중인 임직원이 있을 경우 추가 경고 confirm 노출
  - 문구: `해당 근로형태를 사용 중인 임직원이 N명 있습니다. 삭제 시 해당 임직원의 근로형태가 초기화됩니다. 삭제하시겠습니까?`
- **삭제 실패 시 처리** `[공통]`
  - 건별 트랜잭션 적용, 부분 성공 허용
  - 삭제 실패 리스트 팝업 제공
  - 실패한 리스트 개수 제공
  - 실패한 리스트 정보와 실패 사유 제공

## 3. 근로형태 추가 팝업 (HRIS_WT_02)

### 3.1 닫기/이탈 방지 (Confirm)
- 팝업 외 영역 클릭 또는 닫기 버튼 클릭 시 confirm 노출 `[공통]`
- ESC, 브라우저 뒤로가기 등은 비활성/차단 `[공통 기본값]`
- confirm 문구: `근로형태 추가를 취소하시겠습니까?` `[공통 - 항목명 치환]`

### 3.2 추가 완료 토스트
- 추가 성공 시 토스트 1.5초 노출 `[공통]`
- 문구: `근로형태 추가를 완료했습니다.` `[공통 - 항목명 치환]`

### 3.3 유효성 검사 (Validation) 공통 규칙
- 검증 타이밍: blur / submit `[공통]`
- 에러 표시 `[공통]`
  - 컴포넌트 하단에 에러 문구 노출
  - 재입력 후 검증 통과 시 에러 문구 제거
  - 포커스 아웃 시 정상 입력으로 판단되면 에러 문구 제거

### 3.4 서버 에러 처리 `[공통]`
- 서버 에러(500 등) 발생 시 에러 alert 노출
- alert 문구: `서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요.` `[공통]`
- alert 확인 후 팝업 유지 (입력값 보존)

### 3.5 입력 필드 상세 `[화면별]`

#### 근로형태명
- 필수
- 최대 10자
- 특수문자 불가
  - 에러 문구: `특수문자는 입력하실 수 없습니다.`
- 최대 글자 수 초과 시
  - 에러 문구: `최대 10자까지 입력 가능합니다.`

#### 근로형태 코드
- 필수
- 최대 10자
- 특수문자 불가
  - 에러 문구: `특수문자는 입력하실 수 없습니다.`
- 최대 글자 수 초과 시
  - 에러 문구: `최대 10자까지 입력 가능합니다.`
- 중복 불가 (근로형태 코드 기준)
  - 검증 시점: submit (저장 버튼 클릭 시) `[화면별]`
  - 에러 문구: `중복된 근로형태 코드는 사용하실 수 없습니다.`

## 4. 근로형태 수정 팝업 (HRIS_WT_03)

### 4.1 기본 동작
- 기능/검증/토스트/confirm 정책은 "추가 팝업"과 동일 `[공통]`

### 4.2 닫기/이탈 방지 (Confirm)
- 팝업 외 영역 클릭 또는 닫기 버튼 클릭 시 confirm 노출 `[공통]`
- confirm 문구: `근로형태 수정을 취소하시겠습니까?` `[공통 - 항목명 치환]`

### 4.3 수정 완료 토스트
- 수정 성공 시 토스트 1.5초 노출 `[공통]`
- 문구: `근로형태 수정을 완료했습니다.` `[공통 - 항목명 치환]`

### 4.4 수정 제한 `[화면별]`
- 근로형태 코드: 수정 가능
- 나머지 입력 필드 정책은 추가 팝업과 동일

### 4.5 근로형태 코드 중복 검사 (수정 시)
- 자기 자신을 포함하여 중복 검사
- 근로형태 코드를 변경하지 않으면 중복 검사 통과

### 4.6 서버 에러 처리 `[공통]`
- 추가 팝업과 동일 (에러 alert 노출 + 팝업 유지)

## 5. 임직원 관리 연관 정책 `[화면별]`

### 5.1 임직원 추가/수정 팝업 - 근로형태 입력 필드 추가
- **리스트 컬럼에는 근로형태를 추가하지 않음**
- 임직원 추가 팝업(HRIS_EP_02) 및 수정 팝업(HRIS_EP_03)에 근로형태 입력 영역 추가
- 입력 방식: 단일선택 셀렉박스
- 셀렉박스 옵션: 근로형태 관리에 등록된 근로형태명 리스트
  - 근로형태명만 표시 (코드 미표시)
  - 가나다순 정렬
- **필수 여부: 선택값 (필수 아님)**
- placeholder: `선택` `[공통]`

### 5.2 근로형태 삭제 시 임직원 연관 처리
- 근로형태 관리에서 특정 근로형태 삭제 시, 해당 근로형태가 지정된 임직원의 근로형태 값을 NULL로 변경
- 삭제 API에서 연관 임직원 근로형태 NULL 처리를 함께 수행

## 6. 충돌 지점 정리

| 항목 | 공통 정책 | 화면별 정책 | 적용 결과 |
|------|-----------|-------------|-----------|
| 충돌 없음 | - | - | 공통 정책과 화면별 정책 간 충돌 지점 없음 |

> 본 화면은 공통 정책과 화면별 정책 간 충돌이 발견되지 않았습니다. 화면별 정책은 공통 정책을 보완하는 형태로 적용됩니다.
