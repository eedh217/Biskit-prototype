# 기타소득 월별 리스트 화면(SPS_OI_02) - 최종 정책서

---

## 1. 화면 개요

- **화면 ID**: SPS_OI_02
- **화면명**: 기타소득 월별 리스트
- **목적**: SPS_OI_01(기타소득 합산) 화면에서 특정 월 행을 클릭했을 때, 해당 월의 기타소득 개인별 리스트를 조회·관리하는 화면
- **주요 기능**: 성명(상호) 검색, 월별 합산 요약 정보 표시, 기타소득 리스트 조회, 선택 삭제, 전체 삭제, 기타소득 추가(팝업 호출), 간이지급명세서 생성(Popover → 팝업 호출), 엑셀 다운로드
- **진입 경로**: SPS_OI_01 화면에서 특정 월 행 클릭 시 이동
- **화면 전달 파라미터**: URL 쿼리 파라미터 방식 (예: `?year=2026&month=3`)
  - URL 파라미터 없이 직접 접근하거나 비정상 값이 전달된 경우: 에러 화면을 표시한다.
  - 에러 화면 구성 (공통 정책 7.1 적용):
    - 화면 중앙에 에러 아이콘 표시
    - 메시지: **"죄송합니다. 잘못된 접근이거나 요청하신 페이지를 찾을 수 없습니다."**
    - "기타소득 합산으로 이동" 버튼 제공 (클릭 시 SPS_OI_01로 이동)
- **연관 화면**:
  - SPS_OI_01 (기타소득 합산) — 부모 화면
  - SPS_OI_03 (기타소득 추가 팝업) — '기타소득 추가' 버튼 클릭 시 노출
  - SPS_OI_04 (기타소득 수정 팝업) — 행 클릭 시 노출
  - SPS_OI_08 (간이지급명세서 생성 팝업) — '간이지급명세서 생성' Popover에서 대상 선택 후 노출

## 2. 요청한 화면 기능

### 2.1 검색영역

- 구성: 인풋박스 + 검색 버튼
- 검색항목: **성명(상호)**
- placeholder 문구: `성명(상호)를 입력해주세요.` (공통 정책 1.1 — 검색항목 1개)
- Enter 입력 시 검색 실행 (공통 정책 1.1 권장사항 적용)
- 검색 방식: **부분 일치** (LIKE '%keyword%') — 입력된 검색어가 성명(상호)에 포함되어 있으면 조회
- 허용 문자: 한글·영문·숫자·공백·허용 특수문자(&, ', -, ., ·, (, )), 허용되지 않은 문자는 입력 불가
- 검색 인풋박스 빈 값/공백 처리, 최대 입력 길이: 공통 정책 1.1 적용
- 검색 시 하단 리스트 및 리스트 개수를 검색 결과 기준으로 갱신한다.
- 검색 실행 시 페이지네이션 1페이지로 초기화 (공통 정책 1.1 적용)
- 검색 실행 시 체크박스 선택 상태 모두 초기화 (공통 정책 1.1 적용)

### 2.2 상단 요약 정보

- 해당 월의 합산 데이터를 상단에 요약 표시한다.
- 하단 리스트 데이터의 합산값이다.
- **검색 시에도 상단 요약은 항상 전체 데이터 기준을 유지한다.** (검색 결과 기준으로 갱신하지 않음)
- 표시 항목:

| 항목명 | 표시형식 | 비고 |
|--------|---------|------|
| 건수(소득자 건수) | 숫자 | 해당 월 소득자 행 개수 (row 개수) |
| 총 지급액 | 금액(천 단위 콤마 + "원") | 하단 리스트 지급액 합계 |
| 총 필요경비 | 금액(천 단위 콤마 + "원") | 하단 리스트 필요경비 합계 |
| 총 소득금액 | 금액(천 단위 콤마 + "원") | 하단 리스트 소득금액 합계 |
| 총 소득세 | 금액(천 단위 콤마 + "원") | 하단 리스트 소득세 합계 |
| 총 지방소득세 | 금액(천 단위 콤마 + "원") | 하단 리스트 지방소득세 합계 |
| 총 실소득금액 | 금액(천 단위 콤마 + "원") | 하단 리스트 실소득금액 합계 |

- 금액 표시: 공통 정책 5.1(천 단위 콤마), 5.3("원" 단위 표기) 적용
  - 예: 1,000,000원


### 2.3 안내문구

- 리스트 상단에 안내문구를 표시한다.
- 문구: `※ 간이지급명세서 신고파일 생성 후 신고파일 데이터에서 값이 수정된 대상자 또는 추가된 대상자는 상단 '상세검색'을 통해 검색 후 간이지급명세서 개별 생성이 가능합니다.`

### 2.4 리스트 개수(카운트)

- 공통 정책 1.2 적용
- 기본: 등록된 총 리스트 개수 제공
- 검색 상태: 검색 결과 개수 제공
- 표시 형식: "총 N건"

### 2.5 리스트(테이블)

- 테이블 컴포넌트 사용 (공통 정책 1.3 적용)
- 기본 정렬: 최근 등록 순 (공통 정책 1.3 적용)
- 가장 좌측에 "행 선택" 체크박스 제공(멀티 선택) (공통 정책 1.3 적용)
- 헤더에 전체 선택/해제 체크박스 제공 (현재 페이지 전체 선택) (공통 정책 1.3 적용)
- 체크박스 영역 클릭 시 체크박스만 토글하고, 행 클릭(수정 팝업)은 발생하지 않는다.
- 컬럼 정의:

| 컬럼명 | 표시형식 | 정렬 | 비고 |
|--------|---------|------|------|
| 귀속연도 | "YYYY년" | 좌측 | |
| 귀속월 | "MM월" | 좌측 | |
| 성명(상호) | 텍스트 | 좌측 | |
| 주민(사업자)등록번호 | 텍스트 | 좌측 | 마스킹 없이 전체 표시 |
| 소득구분 | 텍스트 | 좌측 | "자문/고문" 또는 "자문/고문 외 인적용역" |
| 지급건수 | 숫자 | 우측 | 사용자가 입력한 지급건수 |
| 지급액(A) | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 5.2, 5.3 적용 |
| 필요경비(B) | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 5.2, 5.3 적용 |
| 소득금액(A-B) | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 5.2, 5.3 적용 |
| 소득세 | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 5.2, 5.3 적용 |
| 지방소득세 | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 5.2, 5.3 적용 |
| 실소득금액 | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 5.2, 5.3 적용 |
| 신고파일 최종 생성일 | 날짜 (YYYY-MM-DD HH:MM:SS) | 중앙 | 미생성 시 "-" 표시 |

### 2.6 리스트 개수 선택(페이지 사이즈)

- 공통 정책 1.4 적용
- **위치: 페이지네이션 영역 내부 (테이블 하단)**
- 셀렉박스 제공
- 옵션: 15개 / 30개 / 50개 / 100개
- 기본 선택: 30개
- 페이지 사이즈 변경 시 현재 보고 있던 데이터 위치를 유지한다. (공통 정책 1.4 적용)

### 2.7 엑셀 다운로드

- 공통 정책 1.5 적용
- 다운로드 범위 규칙:
  1. 검색하지 않았을 경우: 전체 리스트 엑셀 다운로드
  2. 검색했을 경우: 검색 결과 리스트 엑셀 다운로드
  3. 리스트 선택 후 버튼 클릭 시: 선택한 리스트만 엑셀 다운로드
- 엑셀 정렬은 화면과 동일
- 데이터 0건일 경우: "다운로드할 데이터가 없습니다" 안내 메시지 표시 (공통 정책 1.5 적용)
- 엑셀 파일 내 금액 컬럼: 숫자 형식(순수 숫자)으로 저장 (콤마, "원" 없이) (공통 정책 1.5 적용)
- **파일명 규칙**: `YYYY년 M월 기타소득` (예: "2026년 3월 기타소득.xlsx")

### 2.8 선택 삭제

- 공통 정책 1.6 적용
- 좌측 체크박스 선택 시 활성화
- 데이터 0건일 경우 삭제 버튼 비활성화 (공통 정책 1.6 적용)
- 버튼 클릭 시 confirm 노출
  - 문구: "총 N건의 리스트를 삭제하시겠습니까?"
- **삭제 방식: 물리삭제** (DB에서 완전 삭제)
- 삭제 성공 시 토스트 1.5초 노출 — 문구: "삭제가 완료되었습니다." (공통 정책 1.6 적용)
- 삭제 후 리스트/카운트/상단 요약 정보를 최신 상태로 갱신한다.
- **해당 화면은 삭제 실패 케이스가 없습니다.**

### 2.9 전체 삭제

- **선택 삭제 버튼 옆에 '전체 삭제' 버튼을 배치한다.**
- 데이터 0건일 경우 전체 삭제 버튼 비활성화 (공통 정책 1.6 적용)
- 버튼 클릭 시 confirm 노출
  - 문구: `총 N건의 기타소득을 전체 삭제하시겠습니까?`
  - N = 해당 월의 전체 기타소득 리스트 건수
- 확인 버튼 클릭 시 해당 월의 기타소득 리스트를 전체 삭제한다.
- **삭제 방식: 물리삭제** (DB에서 완전 삭제)
- 삭제 성공 시 토스트 1.5초 노출 — 문구: "삭제가 완료되었습니다." (공통 정책 1.6 적용)
- 전체 삭제 후 리스트/카운트/상단 요약 정보를 최신 상태로 갱신한다.
- **해당 화면은 삭제 실패 케이스가 없습니다.**

### 2.10 기타소득 추가

- **리스트 상단에 '기타소득 추가' 버튼을 배치한다.**
- 버튼 클릭 시 SPS_OI_03 '기타소득 추가' 팝업을 노출한다.

### 2.11 행 클릭

- 행 클릭 시 SPS_OI_04 '기타소득 수정' 팝업을 노출한다. (공통 정책 1.3 적용)
- 수정 팝업 호출 시 해당 행의 레코드 ID를 전달한다. (공통 정책 4.2 적용)

### 2.12 간이지급명세서 생성

- **위치**: 상단 title 영역 우측 (PageHeader의 actions 영역)
- **버튼명**: "간이지급명세서 생성"
- **버튼 클릭 시**: Popover가 노출된다.

#### 2.12.1 Popover 구성

- **형태**: 메뉴 아이템 리스트 형태 (제목 없음)
- **크기**: 256px (w-64)
- **옵션 구성**:

| 옵션명 | 동작 |
|--------|------|
| 조회된 전체 대상자(N건) | 클릭 즉시 Popover 닫히고 SPS_OI_08 팝업 노출 (N = 현재 조회된 리스트 행 개수) |
| 선택 대상자(N건) | 클릭 즉시 Popover 닫히고 SPS_OI_08 팝업 노출 (N = 선택된 체크박스 개수, 단, 선택된 대상자가 없을 경우 경고 toast 표시) |

#### 2.12.2 조회된 전체 대상자

- **대상 범위**:
  - **검색하지 않은 경우**: 해당 월의 전체 리스트
  - **검색한 경우**: 검색 결과 리스트
- **전달 데이터**: 대상 리스트의 모든 레코드 ID 배열
- **건수 표시**: "조회된 전체 대상자(N건)" (N = 현재 조회된 리스트 행 개수)

#### 2.12.3 선택 대상자

- **대상 범위**: 리스트 좌측 체크박스로 선택된 대상자
- **전달 데이터**: 선택된 레코드 ID 배열 (`selectedIds`)
- **건수 표시**: "선택 대상자(N건)" (N = 선택된 체크박스 개수, 행 개수 기준)
- **유효성 검증**:
  - 선택된 대상자가 없을 경우 (체크박스 선택 없음):
    - Toast 경고 메시지 표시: "선택된 대상자가 없습니다. 목록 내 좌측 대상자를 선택해주세요."
    - SPS_OI_08 팝업은 노출되지 않음

#### 2.12.4 SPS_OI_08 팝업 호출

- **호출 조건**: 유효성 검증 통과 시
- **전달 파라미터**:
  - `year`: 현재 조회 중인 연도
  - `month`: 현재 조회 중인 월
  - `targetIds`: 선택된 레코드 ID 배열
  - `targetType`: 'all' (조회된 전체 대상자) 또는 'selected' (선택 대상자)
- **팝업 상세 정책**: SPS_OI_08 정책서 참조

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
|--------------|--------------|------------|------|
| 1.6 선택 삭제 confirm 문구 | "총 N개의 리스트를 삭제하시겠습니까?" | 그대로 적용 | 충돌 없음 |
| (추가) 전체 삭제 | 공통 정책에 없음 | **추가** — "총 N건의 기타소득을 전체 삭제하시겠습니까?" | 화면별 추가 기능. 공통 정책에는 전체 삭제 기능이 없으므로 화면 전용 기능으로 추가 |

> 본 화면은 공통 정책과 대부분 일치하며, 주요 충돌 없이 공통 정책을 그대로 적용한다.
> '전체 삭제' 기능은 공통 정책에 없는 화면 전용 추가 기능이다.
