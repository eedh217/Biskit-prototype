# 전체 기타소득 수정 팝업(SPS_OI_07) - 최종 정책서

---

## 1. 화면 개요

- **화면 ID**: SPS_OI_07
- **화면명**: 전체 기타소득 수정 팝업
- **목적**: 전체 기타소득 화면(SPS_OI_05)에서 등록된 기타소득 데이터를 수정 또는 삭제하기 위한 팝업
- **기본 기능**: SPS_OI_06(전체 기타소득 추가 팝업)과 동일하며, 지급액을 **0원으로 입력 가능**하다.
- **종속 조건**: SPS_OI_04와 달리 특정 지급연월에 종속되지 않으며, 지급연도와 지급월이 팝업 내에 표시되고 수정 가능하다.
- **진입 경로**: SPS_OI_05 화면의 리스트 행 클릭 시 노출
- **데이터 연동**: 수정/삭제된 데이터는 SPS_OI_01(기타소득 합산) 및 SPS_OI_02(기타소득 월별 리스트)에서도 즉시 반영된다.
- **연관 화면**:
  - SPS_OI_05 (전체 기타소득) — 부모 화면
  - SPS_OI_06 (전체 기타소득 추가 팝업) — 기본 기능 동일
  - SPS_OI_04 (기타소득 수정 팝업) — 월별 리스트용 수정 팝업 (지급연월 고정)

## 2. 요청한 화면 기능

### 2.1 수정 팝업 호출 키 (공통 정책 4.2 적용)

- 수정 팝업 호출 시 전달하는 키값은 별도 DB PK(id)를 사용한다.

### 2.2 닫기/이탈 방지 (공통 정책 4.3 적용)

- 팝업을 벗어나려는 모든 행동(팝업 외 영역 클릭, ESC 키 입력, 닫기 버튼 클릭, 브라우저 뒤로가기 등) 발생 시 confirm 노출
- 단, 기존 입력값에서 변경된 내용이 없는 경우에는 confirm 없이 팝업을 바로 닫는다.
- confirm 문구: **"기타소득 수정을 취소하시겠습니까?"**

### 2.3 수정 완료 토스트 (공통 정책 4.4 적용)

- 수정 성공 시 토스트 1.5초 노출
- 문구: **"기타소득 수정을 완료했습니다."**
- 수정 완료 후 SPS_OI_05 리스트/카운트를 최신 상태로 갱신한다.
- 수정 완료 후 SPS_OI_05의 기존 스크롤 위치를 유지한다.

### 2.5 자동 계산 영역

입력 영역 하단에 다음 항목을 제공한다. 모두 **readonly**로 제공되며, 사용자가 직접 수정할 수 없다.

| 항목명 | 표시형식 | 상태 | 비고 |
|--------|---------|------|------|
| 소득금액(A-B) | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 5.3 적용 |
| 세율(%) | 퍼센트 | readonly | **20% 고정** |
| 소득세 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 5.3 적용 |
| 지방소득세 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 5.3 적용 |
| 실소득금액 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 5.3 적용 |

### 2.6 자동 계산 동작 규칙

#### 2.6.1 계산 트리거

- 팝업 진입 시 기존 등록값을 기반으로 자동 계산 결과를 표시한다.
- 지급액(A) 또는 필요경비(B) 입력값이 변경될 때마다 **실시간으로** 자동 계산을 수행한다.

#### 2.6.2 계산 순서

1. **소득금액 계산**: 소득금액(A-B) = 지급액(A) - 필요경비(B)
2. **세율 적용**: 세율 = **20% 고정**
3. **소득세 계산**: 소득세 = Math.floor(소득금액 × 0.2)
4. **소액부징수 판정**: 소득세가 1,000원 미만인 경우 → 소득세 = 0원
5. **지방소득세 계산**:
   - 소득세 = 0원인 경우 → 지방소득세 = 0원
   - 소득세 > 0원인 경우 → 지방소득세 = Math.floor(소득세 × 0.1)
6. **실소득금액 계산**: 실소득금액 = 소득금액(A-B) - 소득세 - 지방소득세

#### 2.6.3 계산 예시

**예시 1. 지급액 0원 / 필요경비 0원**

| 항목 | 계산 | 결과 |
|------|------|------|
| 소득금액(A-B) | 0 - 0 | **0원** |
| 소득세 | 0 × 0.2 | **0원** |
| 소액부징수 판정 | 0 < 1,000 | 소득세 = **0원** |
| 지방소득세 | 소득세 0원이므로 | **0원** |
| 실소득금액 | 0 - 0 - 0 | **0원** |

**예시 2. 지급액 1,000,000원 / 필요경비 600,000원**

| 항목 | 계산 | 결과 |
|------|------|------|
| 소득금액(A-B) | 1,000,000 - 600,000 | **400,000원** |
| 소득세 | 400,000 × 0.2 | **80,000원** |
| 소액부징수 판정 | 80,000 >= 1,000 | 소득세 유지 |
| 지방소득세 | 80,000 × 0.1 | **8,000원** |
| 실소득금액 | 400,000 - 80,000 - 8,000 | **312,000원** |

### 2.7 유효성 검사 (공통 정책 4.3 적용)

- 검증 타이밍: **blur / submit**
- 에러 표시: 컴포넌트 하단에 에러 문구 노출
- 재입력 후 검증 통과 시 에러 문구 제거
- 포커스 아웃 시 정상 입력으로 판단되면 에러 문구 제거

#### 2.7.1 필수값 검증 (blur 시)

- 모든 필수 입력 항목이 미입력 상태에서 blur 시, 해당 필드 하단에 에러 문구 노출
- **공백만 입력된 경우 trim 후 빈값으로 판정**하여 동일하게 필수 에러 처리
- 에러 문구: **"필수 입력 항목입니다."**

#### 2.7.2 귀속연월 유효성 검사 (blur 시)

- 귀속연월(귀속연도 + 귀속월)이 지급연월보다 **이후** 날짜인 경우 에러 (동일 연월은 허용, 귀속연월 ≤ 지급연월)
- 에러 문구: **"귀속연월은 지급연월보다 이전 날짜여야합니다."**

#### 2.7.3 주민(사업자)등록번호 자릿수 검사 (blur 시)

- 주민(사업자)등록번호는 **10자리 또는 13자리**만 유효하다.
- 10자리, 13자리가 아닌 경우 blur 시 에러 처리
- 에러 문구: **"유효한 주민(사업자)등록번호가 아닙니다."**
- **체크디지트 검증은 수행하지 않는다.**

#### 2.7.4 필요경비 60% 검증 (blur 시)

- 필요경비(B)가 지급액(A)의 60% 미만인 경우 에러 처리
- 검증식: 필요경비 < 지급액 × 0.6
- 에러 문구: **"필요경비는 지급액의 60/100이상이어야 합니다."**
- **예외**: 지급액이 0원인 경우, 필요경비도 반드시 0원이어야 한다.
  - 지급액 = 0원이고 필요경비 > 0원인 경우 에러 처리
  - 에러 문구: **"필요경비는 지급액보다 적어야 합니다."**

#### 2.7.5 필요경비 > 지급액 검증 (blur 시)

- 필요경비(B)가 지급액(A)보다 큰 경우 에러 처리
- 검증식: 필요경비 > 지급액
- 에러 문구: **"필요경비는 지급액보다 적어야 합니다."**

#### 2.7.6 중복 데이터 검사 (submit 시)

- **지급연월 + 귀속연월 + 주민(사업자)등록번호 + 소득구분**이 모두 동일한 데이터가 이미 존재하는 경우 (자기 자신 제외)
- submit 시 수정을 차단하고 **개별 필드에 에러 표시**를 한다.
  - 에러 표시 위치: 지급연월, 귀속연월, 주민(사업자)등록번호, 소득구분 각 필드 하단
  - 에러 문구: **"지급연월, 귀속연월, 주민(사업자)등록번호, 소득구분이 동일한 기타소득이 존재합니다."**

### 2.8 삭제 (공통 정책 4.5 적용)

- 수정 팝업 내 **"삭제"** 버튼을 제공한다.
- 삭제 버튼 클릭 시 confirm 팝업 노출
  - 데이터 수정 상태(변경사항 존재)와 무관하게, 수정 취소 confirm 없이 바로 삭제 confirm을 표시한다.
  - 문구: **"기타소득을 삭제하시겠습니까? 삭제한 정보는 복구할 수 없습니다."**
- confirm 확인 버튼 클릭 시
  1. **삭제 가능 여부 조건: 없음 (항상 삭제 가능)**
  2. 삭제 처리 후 팝업을 닫는다.
  3. SPS_OI_05 리스트/카운트를 최신 상태로 갱신한다.
  4. 토스트: **"기타소득 삭제를 완료했습니다."**

### 2.9 안내 문구

팝업 하단에 다음 안내 문구를 **고정 표시**한다.

> ※ 지급액과 필요경비 입력 시 소득금액이 자동 계산되며, 세율(20%)가 자동 적용되어 소득세, 지방소득세, 실소득 금액이 제공됩니다.
> ※ 필요경비는 지급액의 60/100 이상이어야 하며, 지급액보다 클 수 없습니다.
> ※ 소액부징수(소득세액이 1천원 미만)인 경우, 소득세가 면제됩니다.

- 위 문구는 정보 제공 목적이다.

### 2.10 저장 실패 처리

- 저장(submit) 실패 시 (중복 검사 실패, 비즈니스 로직 실패 등) **팝업을 유지**하고 에러를 표시한다.
- 중복 검사 실패: alert 노출 후 팝업 유지
- 기타 비즈니스 로직 실패: 해당 필드 하단에 에러 문구 노출 후 팝업 유지
- 서버 에러: 공통 정책 6.1 alert 노출 후 팝업 유지

### 2.11 전체 동작 흐름

```
1. SPS_OI_05 리스트 행 클릭 시 해당 행의 레코드 ID를 전달하며 수정 팝업을 호출한다.
2. 팝업 진입 시 기존 등록값이 각 필드에 바인딩된다.
3. 팝업 진입 시 기존값 기반으로 자동 계산 결과(소득금액/세율/소득세/지방소득세/실소득금액)를 표시한다.
4. 모든 필드(지급연도, 지급월, 귀속연도, 귀속월, 성명(상호), 주민번호, 내외국인, 소득구분, 지급건수, 지급액, 필요경비)를 변경할 수 있다.
5. 필드 변경 후 focus out 시 자동 재계산을 수행한다.
6. blur 이벤트마다 개별 유효성 검사를 수행한다.
7. "수정" 버튼 클릭 시 검증을 수행한다.
8. 수정 성공 시 토스트("기타소득 수정을 완료했습니다.") 노출 후 팝업을 닫고 부모 화면을 갱신한다.
9. 수정 실패 시 팝업을 유지하고 에러를 표시한다.
10. 삭제 버튼 클릭 시 삭제 confirm → 삭제 처리 → 팝업 닫힘 → 부모 화면 갱신.
```

### 2.12 서버 에러 처리 (공통 정책 6.1 적용)

- 문구: **"서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."**
