# 기타소득 엑셀 업로드 기능(SPS_OI_01, SPS_OI_05) - 최종 정책서

---

## 1. 기능 개요

- **화면 ID**: SPS_OI_01 (기타소득 합산), SPS_OI_05 (전체 기타소득)
- **화면명**: 기타소득 합산 / 전체 기타소득
- **기능명**: 기타소득 엑셀 업로드
- **목적**: 기타소득 합산 화면 및 전체 기타소득 화면에서 엑셀 파일을 통해 기타소득 데이터를 대량으로 업로드하는 기능
- **공통 정책 참조**: excel-upload-policy (공통 엑셀 업로드 정책)
- **연관 정책**: policy_기타소득추가팝업.md (세율/계산 로직 동일)
- **적용 범위**: 두 화면(SPS_OI_01, SPS_OI_05)의 엑셀 업로드 기능은 **완전히 동일**하게 작동한다.

---

## 2. 요청한 기능 정의

### 2.1 엑셀 업로드 버튼 (excel-upload-policy 적용)

- 기타소득 합산 화면 및 전체 기타소득 화면에 **엑셀 업로드** 버튼을 제공한다.
- 버튼 클릭 시 2가지 옵션을 제공한다:
  - **파일 올리기**
  - **엑셀 양식 받기**

---

### 2.2 파일 올리기 (excel-upload-policy 1-1 적용)

- 시스템 파일 선택 팝업 노출
- 엑셀 파일만 업로드 가능
  - 허용 확장자: `.xlsx`, `.xls`
  - 파일 선택 팝업에서 `.xlsx`/`.xls`만 필터링하여 허용되지 않는 확장자는 **선택 자체 불가**
- 파일은 **1개만** 업로드 가능
- 다중 선택 불가
- **파일 크기 제한**: 최대 **10MB**

---

### 2.3 엑셀 양식 받기 (excel-upload-policy 1-2 적용)

- 양식 파일 즉시 다운로드
- 양식 파일명: `간이지급명세서_기타소득_업로드_양식.xlsx`

---

### 2.4 엑셀 양식 구조

- **Row 1~9**: 헤더/안내 영역
- **Row 10**: 컬럼명
- **Row 11**: 컬럼코드
- **Row 12**: 샘플 데이터 (업로드 시 **자동 무시**, 데이터로 처리되지 않음)
- **Row 13부터**: 실제 데이터 입력 영역 (13행부터 데이터로 인식)

#### 컬럼 정의

| 컬럼코드 | 컬럼명 | 필수여부 | 비고 |
|----------|--------|---------|------|
| name | *성명(상호) | 필수 | 최대 50자 |
| attributionYear | *귀속연도 | 필수 | 숫자만 허용 |
| attributionMonth | *귀속월 | 필수 | 1~12 숫자 |
| paymentYear | *지급연도 | 필수 | 숫자만 허용 |
| paymentMonth | *지급월 | 필수 | 1~12 숫자 |
| iino | *주민(사업자)등록번호 | 필수 | 10자리 또는 13자리, **체크디지트 검증 제외** |
| isForeign | *내외국인 구분 (내국인:N, 외국인:Y) | 필수 | N 또는 Y (정보 표시용) |
| incomeType | *소득구분 | 필수 | "자문/고문" 또는 "자문/고문 외 인적용역" |
| paymentCount | *지급건수 | 필수 | 양의 정수만 허용, 최대 10자리 |
| paymentAmount | *지급액 | 필수 | 0 또는 양의 정수 허용, 최대 12자리 |
| necessaryExpense | *필요경비 | 필수 | 0 또는 양의 정수 허용, 최대 12자리 |

---

### 2.5 업로드 기본 검증 (excel-upload-policy 2 적용)

- 엑셀 컬럼 구조는 양식과 **완전히 일치해야 한다**
- 컬럼 불일치 시 → **전체 업로드 실패**

#### 전체 실패 조건 및 에러 메시지

| 조건 | 에러 메시지 |
|------|------------|
| 컬럼 구조 불일치 | "양식에 맞지 않는 엑셀파일입니다." |
| 양식 외 입력 데이터가 없음 | "데이터가 없습니다. 데이터 입력 후 파일을 업로드해주세요." |

| 2 | 귀속연도 숫자 검증 | 정수가 아닐 경우 (소수점 포함 시 에러) | "귀속연도는 숫자로 입력해주세요." |
| 3 | 귀속연도 범위 검증 | 2026년 미만이거나 현재연도를 초과할 경우 | "귀속연도는 2026년부터 YYYY년까지 입력 가능합니다." (YYYY = 현재연도) |
| 4 | 지급연도 숫자 검증 | 정수가 아닐 경우 (소수점 포함 시 에러) | "지급연도는 숫자로 입력해주세요." |
| 5 | 지급연도 범위 검증 | 2026년 미만이거나 (현재연도+1)년을 초과할 경우 | "지급연도는 2026년부터 YYYY년까지 입력 가능합니다." (YYYY = 현재연도+1) |
| 6 | 귀속월 범위 검증 | 1~12 이내 정수가 아닐 경우 | "귀속월은 1~12 내 숫자로 입력해주세요." |
| 7 | 지급월 범위 검증 | 1~12 이내 정수가 아닐 경우 | "지급월은 1~12 내 숫자로 입력해주세요." |
| 8 | 지급연월 vs 귀속연월 선후관계 | 지급연월이 귀속연월보다 이전 날짜일 경우 | "지급연월은 귀속연월 이후 날짜여야 합니다." |
| 9 | 내외국인 구분 값 검증 | N, Y 이외 문자일 경우 | "내외국인 구분은 N, Y로 입력해주세요." |
| 10 | 소득구분 유효성 검증 | "자문/고문" 또는 "자문/고문 외 인적용역" 이외의 값일 경우 | "소득구분은 '자문/고문' 또는 '자문/고문 외 인적용역'으로 입력해주세요." |
| 11 | 지급건수 숫자 검증 | 정수가 아닌 문자일 경우 (소수점 포함 시 에러) | "지급건수는 숫자만 입력 가능합니다." |
| 12 | 지급건수 양수 검증 | 지급건수가 0 이하일 경우 | "지급건수를 양수로 입력해주세요." |
| 13 | 지급건수 자릿수 검증 | 지급건수가 10자리를 초과할 경우 | "지급건수는 최대 10자리까지 입력 가능합니다." |
| 14 | 지급액 숫자 검증 | 정수가 아닌 문자일 경우 (소수점 포함 시 에러) | "지급액은 숫자만 입력 가능합니다." |
| 15 | 지급액 양수 검증 | 지급액이 0 이하일 경우 | "지급액은 양수로 입력해주세요." |
| 16 | 지급액 자릿수 검증 | 지급액이 12자리를 초과할 경우 | "지급액은 최대 12자리까지 입력 가능합니다." |
| 17 | 필요경비 숫자 검증 | 정수가 아닌 문자일 경우 (소수점 포함 시 에러) | "필요경비는 숫자만 입력 가능합니다." |
| 18 | 필요경비 음수 검증 | 필요경비가 0 미만일 경우 | "필요경비는 0 이상으로 입력해주세요." |
| 19 | 필요경비 자릿수 검증 | 필요경비가 12자리를 초과할 경우 | "필요경비는 최대 12자리까지 입력 가능합니다." |
| 20 | 필요경비 60% 검증 | 필요경비가 지급액의 60% 미만일 경우 | "필요경비는 지급액의 60/100 이상이어야 합니다." |
| 21 | 필요경비 > 지급액 검증 | 필요경비가 지급액보다 클 경우 | "필요경비는 지급액보다 적어야 합니다." |
| 22 | 성명(상호) 길이 검증 | name이 50자를 초과할 경우 | "성명(상호)는 최대 50자까지 입력 가능합니다." |
| 23 | 주민(사업자)등록번호 숫자 검증 | iino가 숫자가 아닐 경우 | "주민(사업자)등록번호는 숫자만 입력 가능합니다." |
| 24 | 주민(사업자)등록번호 자릿수 검증 | iino가 10자리 또는 13자리가 아닐 경우 | "주민(사업자)등록번호는 10자리 또는 13자리로 입력해주세요." |
| 25 | 중복 데이터 검사 | 지급연월 + 귀속연월 + 주민(사업자)등록번호 + 소득구분이 동일한 데이터가 이미 존재할 경우 (파일 내 중복 시 첫 번째 row 성공, 이후 row 실패) | "지급연월, 귀속연월, 주민(사업자)등록번호, 소득구분이 동일한 기타소득이 존재합니다." |

> **복수 에러 처리**: 한 row에서 여러 유효성 검사가 실패할 경우, 검증 순서(순번)에 따라 **첫 번째 에러만** 표시한다.

> **체크디지트 검증 제외**: 기타소득은 주민(사업자)등록번호 체크디지트 검증을 수행하지 않는다. 자릿수만 확인한다.

### 2.9 자동 계산 로직 (기타소득 추가 팝업과 동일)

각 row의 데이터가 유효성 검사를 통과한 후, 서버에서 아래 순서로 자동 계산하여 저장한다.

#### 2.9.1 계산 순서

1. **소득금액 계산**: 소득금액(A-B) = 지급액(A) - 필요경비(B)
2. **세율 적용**: 세율 = **20% 고정**
3. **소득세 계산**: 소득세 = Math.floor(소득금액 × 0.2)
4. **소액부징수 판정**: 소득세가 1,000원 미만인 경우 → 소득세 = **0원**
5. **지방소득세 계산**:
   - 소득세 = 0원인 경우 → 지방소득세 = **0원**
   - 소득세 > 0원인 경우 → 지방소득세 = Math.floor(소득세 × 0.1)
6. **실소득금액 계산**: 실소득금액 = 소득금액(A-B) - 소득세 - 지방소득세

> ※ Math.floor는 소수점 이하 버림(원단위 이하 절사)을 의미한다.

#### 2.9.2 계산 예시

**예시 1. 지급액 1,000,000원 / 필요경비 600,000원**

| 항목 | 계산 | 결과 |
|------|------|------|
| 소득금액(A-B) | 1,000,000 - 600,000 | **400,000원** |
| 소득세 | 400,000 × 0.2 | **80,000원** |
| 소액부징수 판정 | 80,000 >= 1,000 | 소득세 유지 |
| 지방소득세 | 80,000 × 0.1 | **8,000원** |
| 실소득금액 | 400,000 - 80,000 - 8,000 | **312,000원** |

**예시 2. 지급액 1,000,000원 / 필요경비 996,000원**

| 항목 | 계산 | 결과 |
|------|------|------|
| 소득금액(A-B) | 1,000,000 - 996,000 | **4,000원** |
| 소득세 | 4,000 × 0.2 | **800원** |
| 소액부징수 판정 | 800 < 1,000 | 소득세 = **0원** |
| 지방소득세 | 소득세 0원이므로 | **0원** |
| 실소득금액 | 4,000 - 0 - 0 | **4,000원** |

### 2.10 데이터 저장 정책

- **저장 모드**: 신규 등록
- 성공 row는 **즉시** 신규 저장 (부분 실패 시에도 성공 row는 즉시 저장)
- **지급연도/지급월 기준 저장**: 엑셀 내 지급연도, 지급월 값에 따라 해당 연도/월의 기타소득 데이터로 저장한다.
  - 예: 엑셀의 지급연도/지급월이 2026년 2월일 경우 → 기타소득 합산 화면의 2026년 2월 데이터로 저장
- **대표 식별값**: 주민(사업자)등록번호

### 2.10.1 예외 규칙 없음

- 기타소득은 사업소득과 달리 **예외 업종 규칙이 없다**.
- 모든 데이터는 지급연월 기준으로 저장되고 표시된다.
- 귀속 기준 12월 재귀속 로직은 적용되지 않는다.

### 2.11 업로드 결과 화면 (excel-upload-policy 적용)

업로드 완료 후 반드시 다음을 표시한다:

- **총 N건**
- **성공 건수**
- **실패 건수**
- **실패 row 목록**

#### 실패 row 표시 규칙

각 실패 row는 다음을 포함한다:

- **대표 식별값**: 주민(사업자)등록번호
- **실패 사유**: 해당 row의 유효성 검사 실패 메시지

---

### 2.12 실패 데이터 다운로드 (excel-upload-policy 적용)

부분 실패가 발생한 경우 반드시 다음 기능을 제공한다:

- **버튼명**: "실패 데이터 다운로드"
- 실패 row만 포함된 엑셀 파일 생성
- **파일명**: `기타소득업로드_실패목록_YYYYMMDD.xlsx` (업로드 일시 기준)
- **파일 구성**:
  - 기존 업로드 컬럼 유지 (성명, 귀속연도, 귀속월, 지급연도, 지급월, 주민번호, 내외국인, 소득구분, 지급건수, 지급액, 필요경비)
  - 마지막 컬럼에 **"실패사유"** 컬럼 추가
  - 각 실패 row에 실패 사유 입력
  - 금액 컬럼(지급건수, 지급액, 필요경비)은 숫자 형식(순수 숫자)으로 저장 (콤마 없이)
- **목적**: 사용자가 수정 후 재업로드 가능하도록 지원

### 2.13 업로드 완료 처리

#### 전체 성공 시

- 토스트 팝업 **1.5초** 노출
- 문구: **"엑셀 업로드를 완료했습니다."**
- 기타소득 합산 화면 또는 전체 기타소득 화면 데이터 새로고침
- 해당 지급연월의 데이터 반영

#### 부분 실패 시

- 토스트 **미노출**
- 성공/실패 집계 표시
- 실패 목록 표시
- **실패 데이터 다운로드** 버튼 노출
- **확인** 버튼 제공 (클릭 시 해당 화면으로 이동, 성공 건 반영된 상태로 새로고침)

#### 전체 실패 시 (row 단위)

- 부분 실패와 **동일한 UI** 표시 (성공 0건, 실패 N건)
- 실패 목록 표시
- **실패 데이터 다운로드** 버튼 노출
- **확인** 버튼 제공

---

### 2.14 업로드 진행 중 이탈 처리

- 업로드가 진행 중일 때 사용자가 페이지를 이탈(뒤로가기, 새로고침, 다른 메뉴 클릭)하려 할 경우 **확인 팝업**을 표시한다.
  - 팝업 문구: "업로드가 진행 중입니다. 페이지를 떠나시겠습니까?"
  - **확인** 버튼 클릭 시: 업로드 취소, 페이지 이탈
  - **취소** 버튼 클릭 시: 팝업 닫기, 업로드 계속 진행

---

### 2.15 동시 업로드 처리

- 동일 사용자 또는 다른 사용자가 동시에 엑셀 업로드를 수행할 경우 **큐잉 처리**한다.
- 먼저 요청된 업로드가 완료된 후 다음 업로드가 처리된다.

---

### 2.16 제한 정책 (excel-upload-policy 적용)

- 여러 파일 업로드 불가
- 자동 컬럼 매핑 금지
- 컬럼 순서 무시 로직 금지
- 임의 컬럼 허용 금지

---

### 2.17 서버 에러 처리 (공통 정책 6.1 적용)

- 서버 에러 발생 시 alert 팝업 노출
- 문구: **"서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."**

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 기능 적용 | 사유 |
|--------------|--------------|------------|------|
| excel-upload-policy 대표 식별값 | 화면에서 정의 | **주민(사업자)등록번호** | 기타소득의 고유 식별값으로 주민(사업자)등록번호 사용 |
| excel-upload-policy 실패 파일명 | 파일명 예시: 급여업로드_실패목록_20260204 | **변경** → 기타소득업로드_실패목록_YYYYMMDD | 화면명에 맞게 파일명 변경 |
| excel-upload-policy 저장 모드 | 화면에서 정의 | **신규 등록** | 엑셀 업로드 데이터는 항상 신규 등록 |
| (추가) 체크디지트 검증 제외 | 사업소득은 체크디지트 검증 수행 | **제외** | 기타소득은 주민(사업자)등록번호 자릿수만 확인, 체크디지트 검증 없음 |
| (추가) 세율 20% 고정 | 사업소득은 세율 조건부 | **20% 고정** | 기타소득은 모든 경우 세율 20% 적용 |
| (추가) 필요경비 60% 규칙 | 사업소득에 없음 | **추가** | 필요경비는 지급액의 60% 이상이어야 함 |
| (추가) 예외 업종 규칙 없음 | 사업소득은 12월 예외 처리 있음 | **예외 규칙 없음** | 기타소득은 모두 지급연월 기준 표시 |

> 본 기능은 excel-upload-policy(공통 엑셀 업로드 정책)를 기본으로 적용하며, 화면별 세율/계산 로직은 기타소득 추가 팝업(SPS_OI_03)과 동일하게 적용한다.
