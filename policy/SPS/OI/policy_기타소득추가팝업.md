# 기타소득 추가 팝업(SPS_OI_03) - 최종 정책서

---

## 1. 화면 개요

- **화면 ID**: SPS_OI_03
- **화면명**: 기타소득 추가 팝업
- **목적**: 기타소득 월별 리스트 화면(SPS_OI_02)에서 신규 기타소득 데이터를 등록하기 위한 입력 팝업
- **종속 조건**: 해당 팝업은 특정 지급연월(YYYY년 MM월)에 종속되며, 등록되는 데이터는 현재 리스트 화면의 지급연월에 포함된다.
- **진입 경로**: SPS_OI_02 화면의 '기타소득 추가' 버튼 클릭 시 노출
- **접근 권한**: 별도 권한 제한 없음 (모든 사용자 접근 가능)
- **연관 화면**:
  - SPS_OI_02 (기타소득 월별 리스트) - 부모 화면

## 2. 요청한 화면 기능

### 2.1 닫기/이탈 방지 (공통 정책 3.1 적용)

- 팝업을 벗어나려는 모든 행동(팝업 외 영역 클릭, ESC 키 입력, 닫기 버튼 클릭, 브라우저 뒤로가기 등) 발생 시 confirm 노출
- 단, 사용자가 아무런 입력도 하지 않은 초기 상태에서는 confirm 없이 팝업을 바로 닫는다.
- **초기 상태 판정 기준**: 모든 입력 항목의 최종값이 초기값과 동일하면 초기상태로 판정한다.
- confirm 문구: **"기타소득 추가를 취소하시겠습니까?"**

### 2.2 추가 완료 토스트 (공통 정책 3.2 적용)

- 추가 성공 시 토스트 1.5초 노출
- 문구: **"기타소득 추가를 완료했습니다."**
- 추가 완료 후 **팝업을 자동으로 닫고**, SPS_OI_02 리스트/카운트/상단 요약 정보를 최신 상태로 갱신한다.

### 2.3 입력 항목 정의

다음 항목은 **모두 필수 입력값**이다.

- **지급연도, 지급월**: 팝업에 직접 표시되지 않으며, 부모 화면(SPS_OI_02)의 지급연월이 자동으로 바인딩된다. 유효성 검사(귀속연월 비교)에 사용된다.

| 항목명 | 입력 타입 | 제약 조건 | 기본값 | 비고 |
|--------|----------|----------|--------|------|
| 귀속연도 | 단일 셀렉트박스 | 2026년 ~ 현재연도 | 없음 (placeholder: "선택") | |
| 귀속월 | 단일 셀렉트박스 | 1월 ~ 12월 | 없음 (placeholder: "선택") | |
| 성명(상호) | 단일 인풋박스 | 최대 50자, 허용 문자: 한글·영문·숫자·공백·허용 특수문자(&, ', -, ., ·, (, )), 공백만 입력 시 trim 후 빈값 판정(필수 에러) | 없음 | 허용되지 않은 문자는 입력 불가 |
| 주민(사업자)등록번호 | 단일 인풋박스 | 숫자만 입력 가능, 최대 13자리, 10자리 또는 13자리만 허용 | 없음 | **체크디지트 검증 제외** |
| 내외국인 구분 | 라디오 버튼 | 내국인 / 외국인 | **내국인** | 단순 정보 표시용 |
| 소득구분 | 라디오 버튼 | 자문/고문 / 자문/고문 외 인적용역 | **자문/고문** | "자문/고문 외 인적용역" 옆에 도움말 아이콘 제공 |
| 지급건수 | 숫자 입력 필드 | 숫자만 입력 가능, 최대 10자리, 양수만 허용 | 없음 | |
| 지급액(A) | 숫자 입력 필드 | 숫자만 입력 가능, 최대 12자리, 양수만 허용 | 없음 | 공통 정책 5.1(천 단위 콤마) 적용 |
| 필요경비(B) | 숫자 입력 필드 | 숫자만 입력 가능, 최대 12자리, 0 또는 양수 허용 | 없음 | 공통 정책 5.1(천 단위 콤마) 적용 |

#### 소득구분 도움말

- **"자문/고문 외 인적용역"** 옆에 도움말 아이콘(`ℹ️`)을 제공한다.
- 클릭 시 말풍선/툴팁으로 다음 설명을 표시한다:
  - 고용관계 없이 다수인에게 강연을 하고 강연료 등 대가를 받는 용역
  - 라디오/텔리비전방송 등을 통하여 해설/계몽 또는 연기의 심사 등을 하고 보수 또는 이와 유사한 성질의 대가를 받는 용역
  - 변호사/공인회계사/세무사 그 밖에 전문적 지식 또는 특별한 기능을 가진 자가 그 지식 또는 기능을 활용하여 보수 또는 그 밖의 대가를 받고 제공하는 용역
  - 그 밖에 고용관계 없이 수당 또는 이와 유사한 성질의 대가를 받고 제공하는 용역
- 다른 영역 클릭 시 툴팁이 닫힌다.

### 2.4 자동 계산 영역

입력 영역 하단에 다음 항목을 제공한다. 모두 **readonly**로 제공되며, 사용자가 직접 수정할 수 없다.

| 항목명 | 표시형식 | 상태 | 비고 |
|--------|---------|------|------|
| 소득금액(A-B) | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 5.3 적용 |
| 세율(%) | 퍼센트 | readonly | **20% 고정** |
| 소득세 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 5.3 적용 |
| 지방소득세 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 5.3 적용 |
| 실소득금액 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 5.3 적용 |

### 2.5 자동 계산 동작 규칙

#### 2.5.1 계산 트리거

- 지급액(A) 또는 필요경비(B) 입력값이 변경될 때마다 **실시간으로** 자동 계산을 수행한다.
  1. 입력값 변경 시 즉시 자동 계산이 수행된다.
  2. 필요경비 유효성 검증(2.6.4, 2.6.5)은 blur 시 수행된다.

#### 2.5.2 계산 순서

1. **소득금액 계산**: 소득금액(A-B) = 지급액(A) - 필요경비(B)
2. **세율 적용**: 세율 = **20% 고정**
3. **소득세 계산**: 소득세 = Math.floor(소득금액 × 0.2)
4. **소액부징수 판정**: 소득세가 1,000원 미만인 경우 → 소득세 = 0원
5. **지방소득세 계산**:
   - 소득세 = 0원인 경우 → 지방소득세 = 0원
   - 소득세 > 0원인 경우 → 지방소득세 = Math.floor(소득세 × 0.1)
6. **실소득금액 계산**: 실소득금액 = 소득금액(A-B) - 소득세 - 지방소득세

> ※ Math.floor는 소수점 이하 버림(원단위 이하 절사)을 의미한다.

#### 2.5.3 계산 예시

**예시 1. 지급액 1,000,000원 / 필요경비 600,000원**

| 항목 | 계산 | 결과 |
|------|------|------|
| 소득금액(A-B) | 1,000,000 - 600,000 | **400,000원** |
| 소득세 | 400,000 × 0.2 | **80,000원** |
| 소액부징수 판정 | 80,000 >= 1,000 | 소득세 유지 |
| 지방소득세 | 80,000 × 0.1 | **8,000원** |
| 실소득금액 | 400,000 - 80,000 - 8,000 | **312,000원** |

**예시 2. 지급액 1,000,000원 / 필요경비 996,000원**

| 항목 | 계산 | 결과 |
|------|------|------|
| 소득금액(A-B) | 1,000,000 - 996,000 | **4,000원** |
| 소득세 | 4,000 × 0.2 | **800원** |
| 소액부징수 판정 | 800 < 1,000 | 소득세 = **0원** |
| 지방소득세 | 소득세 0원이므로 | **0원** |
| 실소득금액 | 4,000 - 0 - 0 | **4,000원** |

### 2.6 유효성 검사 (공통 정책 3.3 적용)

- 검증 타이밍: **blur / submit**
- 에러 표시: 컴포넌트 하단에 에러 문구 노출
- 재입력 후 검증 통과 시 에러 문구 제거
- 포커스 아웃 시 정상 입력으로 판단되면 에러 문구 제거

#### 2.6.1 필수값 검증 (blur 시)

- 모든 필수 입력 항목이 미입력 상태에서 blur 시, 해당 필드 하단에 에러 문구 노출
- **공백만 입력된 경우 trim 후 빈값으로 판정**하여 동일하게 필수 에러 처리
- 에러 문구: **"필수 입력 항목입니다."**
- **검증 순서**: blur 시 (1) 필수값 검증을 먼저 수행 → (2) 필수값 통과 시에만 추가 유효성 검사를 진행한다.

#### 2.6.2 귀속연월 유효성 검사 (blur 시)

- 귀속연도와 귀속월이 **둘 다 선택된 상태**에서 blur 시에만 검사한다. (한쪽만 선택된 경우 검사하지 않음)
- 귀속연월(귀속연도 + 귀속월)이 지급연월보다 **이후** 날짜인 경우 에러 (동일 연월은 허용, 귀속연월 ≤ 지급연월)
- 에러 표시 위치: **귀속연도+귀속월 영역 전체 하단**
- 에러 문구: **"귀속연월은 지급연월보다 이전 날짜여야합니다."**

#### 2.6.3 주민(사업자)등록번호 자릿수 검사 (blur 시)

- 주민(사업자)등록번호는 **10자리 또는 13자리**만 유효하다.
- 10자리, 13자리가 아닌 경우 blur 시 에러 처리
- 에러 문구: **"유효한 주민(사업자)등록번호가 아닙니다."**
- **체크디지트 검증은 수행하지 않는다.**

#### 2.6.4 필요경비 60% 검증 (blur 시)

- 필요경비(B)가 지급액(A)의 60% 미만인 경우 에러 처리
- 검증식: 필요경비 < 지급액 × 0.6
- 에러 문구: **"필요경비는 지급액의 60/100이상이어야 합니다."**

#### 2.6.5 필요경비 > 지급액 검증 (blur 시)

- 필요경비(B)가 지급액(A)보다 큰 경우 에러 처리
- 검증식: 필요경비 > 지급액
- 에러 문구: **"필요경비는 지급액보다 적어야 합니다."**

#### 2.6.6 중복 데이터 검사 (submit 시)

- submit 시점에 **서버에서 최종 중복 체크**를 수행한다.
- **지급연월 + 귀속연월 + 주민(사업자)등록번호 + 소득구분**이 모두 동일한 데이터가 이미 존재하는 경우
- submit 시 등록을 차단하고 **개별 필드에 에러 표시**를 한다.
  - 에러 표시 위치: 지급연월, 귀속연월, 주민(사업자)등록번호, 소득구분 각 필드 하단
  - 에러 문구: **"지급연월, 귀속연월, 주민(사업자)등록번호, 소득구분이 동일한 기타소득이 존재합니다."**

### 2.7 안내 문구

팝업 하단에 다음 안내 문구를 **고정 표시**한다.

> ※ 지급액과 필요경비 입력 시 소득금액이 자동 계산되며, 세율(20%)가 자동 적용되어 소득세, 지방소득세, 실소득 금액이 제공됩니다.
> ※ 필요경비는 지급액의 60/100 이상이어야 하며, 지급액보다 클 수 없습니다.
> ※ 소액부징수(소득세액이 1천원 미만)인 경우, 소득세가 면제됩니다.

- 위 문구는 정보 제공 목적이다.

### 2.8 저장 실패 처리

- 저장(submit) 실패 시 (중복 검사 실패, 비즈니스 로직 실패 등) **팝업을 유지**하고 에러를 표시한다.
- 중복 검사 실패: alert 노출 후 팝업 유지
- 기타 비즈니스 로직 실패: 해당 필드 하단에 에러 문구 노출 후 팝업 유지
- 서버 에러: 공통 정책 6.1 alert 노출 후 팝업 유지

### 2.9 전체 동작 흐름

```
1. 사용자가 모든 필수값을 입력한다.
2. 지급액(A) 또는 필요경비(B) 입력 시 실시간으로 자동 계산을 수행한다.
3. blur 이벤트마다 개별 유효성 검사를 수행한다.
4. submit 시 서버에서 중복 검사를 수행한다.
5. 모든 조건을 통과하면 데이터를 등록한다.
6. 등록 성공 시 토스트 노출 후 팝업을 자동으로 닫고 부모 화면을 갱신한다.
7. 등록 실패 시 팝업을 유지하고 에러를 표시한다.
```

### 2.10 서버 에러 처리 (공통 정책 6.1 적용)

- 서버 에러 발생 시 alert 팝업 노출
- 문구: **"서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."**

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
|--------------|--------------|------------|------|
| 3.3 유효성 검사 타이밍 | blur / submit | 그대로 적용 + **자동 계산 트리거 추가** (지급액·필요경비 입력 시 실시간 자동 계산) | 자동 계산은 공통 정책에 없는 화면 전용 동작. 입력값 변경 시 즉시 자동 계산이 수행됨 |

| (추가) 세율 고정 | 공통 정책에 없음 | **추가** - 세율 20% 고정 적용 | 화면 전용 비즈니스 로직 |
