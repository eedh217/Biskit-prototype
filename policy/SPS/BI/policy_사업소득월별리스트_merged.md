# 사업소득 월별 리스트 화면(SPS_BI_02) - 최종 정책서

---

## 1. 화면 개요

- **화면 ID**: SPS_BI_02
- **화면명**: 사업소득 월별 리스트
- **목적**: SPS_BI_01(사업소득 합산) 화면에서 특정 월 행을 클릭했을 때, 해당 월의 사업소득 개인별 리스트를 조회·관리하는 화면
- **주요 기능**: 성명(상호) 검색, 월별 합산 요약 정보 표시, 사업소득 리스트 조회, 선택 삭제, 전체 삭제, 사업소득 추가(팝업 호출), 간이지급명세서 생성(Popover → 팝업 호출)
- **진입 경로**: SPS_BI_01 화면에서 특정 월 행 클릭 시 이동
- **화면 전달 파라미터**: URL 쿼리 파라미터 방식 (예: `?year=2026&month=3`)
  - URL 파라미터 없이 직접 접근 시: 에러 페이지를 표시한다.
  - URL 파라미터 비정상 값(year=abcd, month=13 등) 전달 시: 에러 페이지를 표시한다.
- **연관 화면**:
  - SPS_BI_01 (사업소득 합산) — 부모 화면
  - SPS_BI_03 (사업소득 추가 팝업) — '사업소득 추가' 버튼 클릭 시 노출
  - SPS_BI_04 (사업소득 수정 팝업) — 행 클릭 시 노출 (단건/다건 통합)
  - SPS_BI_08 (간이지급명세서 생성 팝업) — '간이지급명세서 생성' Popover에서 대상 선택 후 노출

## 2. 요청한 화면 기능

### 2.1 검색영역

- 구성: 인풋박스 + 검색 버튼
- 검색항목: **성명(상호)**
- placeholder 문구: `성명(상호)를 입력해주세요.` (공통 정책 1.1 — 검색항목 1개)
- Enter 입력 시 검색 실행 (공통 정책 1.1 권장사항 적용)
- 검색 방식: **부분 일치** (LIKE '%keyword%') — 입력된 검색어가 성명(상호)에 포함되어 있으면 조회
- 허용 문자: 한글·영문·숫자·공백·허용 특수문자(&, ', -, ., ·, (, )), 허용되지 않은 문자는 입력 불가
- 검색 인풋박스 빈 값/공백 처리, 최대 입력 길이: 공통 정책 1.1 적용
- 검색 시 하단 리스트 및 리스트 개수를 검색 결과 기준으로 갱신한다.
- 검색 실행 시 페이지네이션 1페이지로 초기화 (공통 정책 1.1 적용)
- 검색 실행 시 체크박스 선택 상태 모두 초기화 (공통 정책 1.1 적용)

### 2.2 상단 요약 정보

- 해당 월의 합산 데이터를 상단에 요약 표시한다.
- 하단 리스트 데이터의 합산값이다. (귀속 기준 예외 규칙(2.12)에 의해 재귀속된 데이터도 포함하여 합산)
- **검색 시에도 상단 요약은 항상 전체 데이터 기준을 유지한다.** (검색 결과 기준으로 갱신하지 않음)
- 표시 항목:

| 항목명 | 표시형식 | 비고 |
|--------|---------|------|
| 건수(소득자 건수) | 숫자 | 해당 월 소득자 행 개수 (합산 후 row 개수) |
| 총 지급액 | 금액(천 단위 콤마 + "원") | 하단 리스트 지급액 합계 |
| 총 소득세 | 금액(천 단위 콤마 + "원") | 하단 리스트 소득세 합계 |
| 총 지방소득세 | 금액(천 단위 콤마 + "원") | 하단 리스트 지방소득세 합계 |
| 총 실지급액 | 금액(천 단위 콤마 + "원") | 하단 리스트 실지급액 합계 |

- 금액 표시: 공통 정책 5.1(천 단위 콤마), 4.3("원" 단위 표기) 적용
  - 예: 1,000,000원

### 2.3 안내문구

- 리스트 상단에 안내문구를 표시한다.
- 문구: `※ 간이지급명세서 신고 후 데이터가 수정되거나 추가된 대상자는 좌측 체크박스에서 체크 후 간이지급명세서 개별 생성을 진행해주세요.`

### 2.4 리스트 개수(카운트)

- 공통 정책 1.2 적용
- 기본: 등록된 총 리스트 개수 제공
- 검색 상태: 검색 결과 개수 제공
- **표시 형식**:
  - **1~11월**: "N건" 형식으로만 표시 (N = 합산 후 행 개수)
  - **12월**:
    - 예외규칙 데이터가 없는 경우: "N건" 형식으로만 표시
    - 예외규칙 데이터가 있는 경우: "N건(X개)" 형식으로 표시
      - N = 합산 후 행 개수 (화면에 보이는 row 개수)
      - X = 합산하기 전 총 데이터 개수 (실제 레코드 개수)
      - 예시: 4개 데이터 중 2개가 합산되어 3개 행으로 표시되는 경우 → "3건(4개)"
  - 예외규칙 데이터: 보험설계사(940906), 음료배달(940907), 방문판매원(940908) 업종이면서 귀속연도 ≠ 지급연도인 데이터 (2.12 참조)

### 2.5 리스트(테이블)

- 테이블 컴포넌트 사용 (공통 정책 1.3 적용)
- 기본 정렬: 최근 등록 순 (공통 정책 1.3 적용)
- 가장 좌측에 "행 선택" 체크박스 제공(멀티 선택) (공통 정책 1.3 적용)
- 헤더에 전체 선택/해제 체크박스 제공 (현재 페이지 전체 선택) (공통 정책 1.3 적용)
- 체크박스 영역 클릭 시 체크박스만 토글하고, 행 클릭(수정 팝업)은 발생하지 않는다.
- **합산 행 표시**: 예외 업종 합산 규칙(2.13)에 의해 동일 인물의 여러 건이 합산된 경우, 합산 행으로 표시한다. (상세 규칙은 2.13 참조)
- 컬럼 정의:

| 컬럼명 | 표시형식 | 정렬 | 비고 |
|--------|---------|------|------|
| 귀속연도 | "YYYY년" | 좌측 | |
| 귀속월 | "MM월" | 좌측 | |
| 성명(상호) | 텍스트 | 좌측 | |
| 주민(사업자)등록번호 | 텍스트 | 좌측 | 마스킹 없이 전체 표시 |
| 업종코드 | 텍스트 | 좌측 | |
| 지급액 | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 4.2, 4.3 적용. 합산 행인 경우 "N개 합산" 뱃지 표시 |
| 소득세 | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 4.2, 4.3 적용 |
| 지방소득세 | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 4.2, 4.3 적용 |
| 실지급액 | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 4.2, 4.3 적용 |
| 신고파일 최종 생성일 | 날짜 (YYYY-MM-DD HH:mm:ss) | 중앙 | 미생성 시 "-" 표시 |

### 2.6 리스트 개수 선택(페이지 사이즈)

- 공통 정책 1.4 적용
- **위치: 페이지네이션 영역 내부 (테이블 하단)**
- 셀렉박스 제공
- 옵션: 15개 / 30개 / 50개 / 100개
- 기본 선택: 30개
- 페이지 사이즈 변경 시 현재 보고 있던 데이터 위치를 유지한다. (공통 정책 1.4 적용)

### 2.7 엑셀 다운로드

- 공통 정책 1.5 적용
- 다운로드 범위 규칙:
  1. 검색하지 않았을 경우: 전체 리스트 엑셀 다운로드
  2. 검색했을 경우: 검색 결과 리스트 엑셀 다운로드
  3. 리스트 선택 후 버튼 클릭 시: 선택한 리스트만 엑셀 다운로드
- 엑셀 정렬은 화면과 동일
- 데이터 0건일 경우: "다운로드할 데이터가 없습니다" 안내 메시지 표시 (공통 정책 1.5 적용)
- 엑셀 파일 내 금액 컬럼: 숫자 형식(순수 숫자)으로 저장 (콤마, "원" 없이) (공통 정책 1.5 적용)
- **파일명 규칙**: `YYYY년 M월 사업소득` (예: "2026년 3월 사업소득.xlsx")

### 2.8 선택 삭제

- 공통 정책 1.6 적용
- 좌측 체크박스 선택 시 활성화
- 데이터 0건일 경우 삭제 버튼 비활성화 (공통 정책 1.6 적용)
- 버튼 클릭 시 confirm 노출
  - **합산된 데이터가 없을 때**: "총 N건의 리스트를 삭제하시겠습니까?"
  - **합산된 데이터가 있을 때**: "총 N건(X개)의 리스트를 삭제하시겠습니까?" (N=선택한 행 개수, X=실제 레코드 개수)
- **삭제 방식: 물리삭제** (DB에서 완전 삭제)
- 삭제 성공 시 토스트 1.5초 노출 — 문구: "삭제가 완료되었습니다." (공통 정책 1.6 적용)
- 삭제 후 리스트/카운트/상단 요약 정보를 최신 상태로 갱신한다.
- 삭제 실패 시 처리 정책 (공통 정책 1.6 적용):
  - 건별 트랜잭션 적용, 부분 성공 허용
  - 삭제 실패 리스트 팝업 제공: **성명(상호) + 실패 사유** 표시
  - 삭제 실패한 리스트 개수 및 실패 사유 제공

### 2.9 전체 삭제

- **선택 삭제 버튼 옆에 '전체 삭제' 버튼을 배치한다.**
- 데이터 0건일 경우 전체 삭제 버튼 비활성화 (공통 정책 1.6 적용)
- 버튼 클릭 시 confirm 노출
  - 문구: `총 N명의 사업소득을 전체 삭제하시겠습니까?`
  - N = 해당 월의 전체 사업소득 리스트 건수
- 확인 버튼 클릭 시 해당 월의 사업소득 리스트를 전체 삭제한다.
- **삭제 방식: 물리삭제** (DB에서 완전 삭제)
- 삭제 성공 시 토스트 1.5초 노출 — 문구: "삭제가 완료되었습니다." (공통 정책 1.6 적용)
- 전체 삭제 실패 시 처리 정책: 선택 삭제와 동일하게 부분 성공 허용
  - 건별 트랜잭션 적용, 부분 성공 허용
  - 삭제 실패 리스트 팝업 제공: **성명(상호) + 실패 사유** 표시
  - 삭제 실패한 리스트 개수 및 실패 사유 제공
- 전체 삭제 후 리스트/카운트/상단 요약 정보를 최신 상태로 갱신한다.

### 2.10 사업소득 추가

- **리스트 상단에 '사업소득 추가' 버튼을 배치한다.**
- 버튼 클릭 시 SPS_BI_03 '사업소득 추가' 팝업을 노출한다.

### 2.11 행 클릭

- **비합산 행** 클릭 시 SPS_BI_04 '사업소득 수정' 팝업을 단건 모드로 노출한다. (공통 정책 1.3 적용)
- **합산 행** 클릭 시 SPS_BI_04 '사업소득 수정' 팝업을 다건 모드(탭 기반)로 노출한다. (2.13 합산 규칙 참조)
- 수정 팝업 호출 시 해당 행의 레코드 배열을 전달한다. (공통 정책 5.2 적용)
  - 비합산 행: 1건 배열, 합산 행: 합산에 포함된 모든 레코드 배열.
- **수정 팝업의 필드 구성 및 상세 정책은 SPS_BI_04 정책서에서 별도 정의한다.**

### 2.12 귀속 기준 예외 규칙 (간이지급명세서)

- 기본 원칙: 해당 월의 리스트에는 **지급연월**이 해당 월인 데이터만 포함된다.
- 예외 적용 대상 업종:
  - 보험설계사 (940906)
  - 음료배달 (940907)
  - 방문판매원 (940908)
- 예외 적용 조건 ①: 위 업종코드에 해당하면서, **귀속연도와 지급연도가 서로 다른 경우** (귀속월은 고려하지 않고 연도 기준으로만 판정)
- 예외 적용 조건 ②(12월 지급 합류): 위 업종코드에 해당하면서, **귀속연도와 지급연도가 같고, 지급월이 12월이며**, 동일 귀속연월+주민(사업자)등록번호+업종코드로 **조건①에 해당하는 데이터가 존재**하는 경우
- 예외 처리: 조건①, 조건②에 해당하는 데이터는 지급연월 기준 리스트가 아닌 **귀속연도 12월** 리스트에 포함시킨다.
  - 예: 귀속연월 2025년 8월, 지급연월 2026년 2월, 업종코드 보험설계사(940906) → **2025년 12월** 리스트에 포함, 2026년 2월 리스트에는 미포함
- **리스트 표시 시 귀속월 컬럼에는 원래 귀속월을 그대로 표시한다.** (12월로 변경 표시하지 않음)
- 그 외 모든 경우: 지급연월 기준 유지
- 판정 흐름:
  1. 업종코드가 보험설계사(940906), 음료배달(940907), 방문판매원(940908)인지 확인
  2. 귀속연도와 지급연도 비교
  3. 서로 다를 경우(조건①) → 귀속연도 12월 리스트에 포함
  4. 같고 지급월이 12월인 경우(조건② 후보) → 동일 귀속연월+주민번호+업종코드로 조건①에 해당하는 데이터가 존재하면 귀속연도 12월 리스트에 합류
  5. 그 외 → 지급연월 기준 유지

### 2.13 예외 업종 합산 표시 규칙

예외 업종(보험설계사(940906), 음료배달(940907), 방문판매원(940908))에 해당하면서 귀속연도 ≠ 지급연도인 데이터는, 동일 인물이 여러 지급월에 걸쳐 등록할 수 있다. 이 경우 월별 리스트에서 **합산 행**으로 표시한다.

#### 2.13.1 합산 그룹핑 기준

- **귀속연월 + 주민(사업자)등록번호 + 업종코드**가 동일한 예외 업종 데이터를 하나의 그룹으로 합산한다.
- 12월 지급 합류 조건(2.12 조건②)에 해당하는 데이터도 동일 그룹에 포함하여 합산한다.
- 그룹 내 레코드가 2건 이상인 경우 **합산 행**으로 표시한다.
- 그룹 내 레코드가 1건인 경우 기존과 동일하게 개별 행으로 표시한다.
- 비예외 업종 데이터는 합산 대상이 아니며, 항상 개별 행으로 표시한다.

#### 2.13.2 합산 행 표시

- 합산 행의 귀속연월, 성명(상호), 주민(사업자)등록번호, 업종코드는 그룹 대표값(첫 번째 레코드 기준)을 표시한다.
- 합산 행의 **지급액, 소득세, 지방소득세, 실지급액**은 그룹 내 모든 레코드의 **합산값**을 표시한다.
- 합산 행의 지급액 옆에 **"N개 합산"** 뱃지를 표시한다. (N = 그룹 내 레코드 수)
- 합산 행은 일반 행과 **시각적으로 구분**한다. (배경색 구분)

#### 2.13.3 합산 행 체크박스 동작

- 합산 행의 체크박스를 선택하면, 그룹 내 **모든 레코드의 ID**가 선택된다.
- 합산 행의 체크박스를 해제하면, 그룹 내 **모든 레코드의 ID**가 해제된다.
- 선택 삭제 시 합산 행에 포함된 모든 개별 레코드가 삭제된다.

#### 2.13.4 합산 행 클릭

- 합산 행 클릭 시 SPS_BI_04 **'사업소득 수정'** 팝업을 다건 모드(탭 기반)로 노출한다.
- 그룹에 포함된 모든 레코드 데이터를 팝업에 전달한다.
- 수정 팝업의 상세 정책은 SPS_BI_04 정책서를 참조한다.

#### 2.13.5 상단 요약 정보와 합산 행의 관계

- **상단 요약 정보**(건수, 총 지급액, 총 소득세 등)는 합산 후 **행(row)** 기준으로 집계한다.
  - 예: 합산 행이 3개를 합산한 경우, 건수에는 1건으로 카운트된다. (총 3건(4개)일 경우 상단 요약정보에는 "3건"으로 표시)
- **리스트 카운트**(페이지네이션)는 합산 후 행 수 기준으로 표시한다.
  - 표시 형식: "총 N건 (M개)" — N = 합산 후 행 개수, M = 개별 레코드 개수

#### 2.13.6 엑셀 다운로드와 합산 행의 관계

- 엑셀 다운로드 시 합산 행이 아닌 **개별 레코드 단위**로 내보낸다.
- 엑셀 파일에는 지급연도/지급월 컬럼이 포함된다. (합산된 레코드의 지급월이 서로 다를 수 있으므로)

### 2.14 간이지급명세서 생성

- **위치**: 상단 title 영역 우측 (PageHeader의 actions 영역)
- **버튼명**: "간이지급명세서 생성"
- **버튼 클릭 시**: Popover가 노출된다.

#### 2.14.1 Popover 구성

- **형태**: 메뉴 아이템 리스트 형태 (제목 없음)
- **크기**: 256px (w-64)
- **옵션 구성**:

| 옵션명 | 동작 |
|--------|------|
| 조회된 전체 대상자(N건) | 클릭 즉시 Popover 닫히고 SPS_BI_08 팝업 노출 (N = 현재 조회된 리스트 행 개수) |
| 선택 대상자(N건) | 클릭 즉시 Popover 닫히고 SPS_BI_08 팝업 노출 (N = 선택된 체크박스 개수, 단, 선택된 대상자가 없을 경우 alert 표시) |

#### 2.14.2 조회된 전체 대상자

- **대상 범위**:
  - **검색하지 않은 경우**: 해당 월의 전체 리스트 (합산 행 포함)
  - **검색한 경우**: 검색 결과 리스트
- **전달 데이터**: 대상 리스트의 모든 개별 레코드 ID 배열
  - 합산 행인 경우: 그룹 내 모든 개별 레코드 ID를 포함
- **건수 표시**: "조회된 전체 대상자(N건)" (N = 현재 조회된 리스트 행 개수, 합산 후 행 개수 기준)

#### 2.14.3 선택 대상자

- **대상 범위**: 리스트 좌측 체크박스로 선택된 대상자
- **전달 데이터**: 선택된 개별 레코드 ID 배열 (`selectedIds`)
  - 합산 행 체크 시: 그룹 내 모든 개별 레코드 ID가 포함됨
- **건수 표시**: "선택 대상자(N건)" (N = 선택된 체크박스 개수, 행 개수 기준)
- **유효성 검증**:
  - 선택된 대상자가 없을 경우 (체크박스 선택 없음):
    - Alert 경고 메시지 표시: "선택된 대상자가 없습니다. 목록 내 좌측 대상자를 선택해주세요."
    - SPS_BI_08 팝업은 노출되지 않음

#### 2.14.4 SPS_BI_08 팝업 호출

- **호출 조건**: 유효성 검증 통과 시
- **전달 파라미터**:
  - `year`: 현재 조회 중인 연도
  - `month`: 현재 조회 중인 월
  - `targetIds`: 선택된 개별 레코드 ID 배열
  - `targetType`: 'all' (조회된 전체 대상자) 또는 'selected' (선택 대상자)
- **팝업 상세 정책**: SPS_BI_08 정책서 참조
