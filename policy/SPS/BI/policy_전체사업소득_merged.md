# 전체 사업소득 화면(SPS_BI_05) - 최종 정책서

---

## 1. 화면 개요

- **화면 ID**: SPS_BI_05
- **화면명**: 전체 사업소득
- **목적**: 사업소득에 등록된 전체 데이터를 조회·관리하는 화면. 월별 필터 없이 모든 사업소득 데이터를 한 화면에서 확인할 수 있다.
- **주요 기능**: 성명(상호) 검색, 전체 사업소득 리스트 조회, 선택 삭제, 사업소득 추가(팝업 호출), 엑셀 업로드, 엑셀 다운로드
- **진입 경로**: 사업소득 모듈의 '전체 사업소득' 메뉴 클릭 시 이동
- **데이터 연동**: 월별 사업소득(SPS_BI_01, SPS_BI_02) 메뉴와 서로 연동되어 있으며, 추가/수정/삭제 시 양쪽 모두 반영된다.
- **연관 화면**:
  - SPS_BI_01 (사업소득 합산) — 동일 데이터 소스 공유
  - SPS_BI_02 (사업소득 월별 리스트) — 데이터 연동
  - SPS_BI_06 (전체 사업소득 추가 팝업) — '사업소득 추가' 버튼 클릭 시 노출
  - SPS_BI_07 (전체 사업소득 수정 팝업) — 행 클릭 시 노출
- **연관 기능 정책**: [사업소득 엑셀 업로드 정책서](policy_사업소득엑셀업로드_merged.md) — 엑셀 업로드 기능 상세 정책 참조

## 2. 요청한 화면 기능

### 2.1 검색영역

- 구성: 인풋박스 + 검색 버튼
- 검색항목: **성명(상호)**
- placeholder 문구: `성명(상호)를 입력해주세요.` (공통 정책 1.1 — 검색항목 1개)
- Enter 입력 시 검색 실행 (공통 정책 1.1 권장사항 적용)
- 검색 방식: **부분 일치** (LIKE '%keyword%') — 입력된 검색어가 성명(상호)에 포함되어 있으면 조회
- 허용 문자: 한글·영문·숫자·공백·허용 특수문자(&, ', -, ., ·, (, )), 허용되지 않은 문자는 입력 불가
- 검색 인풋박스 빈 값/공백 처리, 최대 입력 길이: 공통 정책 1.1 적용
- 검색 시 하단 리스트 및 리스트 개수를 검색 결과 기준으로 갱신한다.
- 검색 실행 시 페이지네이션 1페이지로 초기화 (공통 정책 1.1 적용)
- 검색 실행 시 체크박스 선택 상태 모두 초기화 (공통 정책 1.1 적용)

### 2.2 리스트 개수(카운트)

- 공통 정책 1.2 적용
- 기본: 등록된 총 리스트 개수 제공
- 검색 상태: 검색 결과 개수 제공

### 2.3 리스트(테이블)

- 테이블 컴포넌트 사용 (공통 정책 1.3 적용)
- 기본 정렬: 최근 등록 순 (공통 정책 1.3 적용)
- 가장 좌측에 "행 선택" 체크박스 제공(멀티 선택) (공통 정책 1.3 적용)
- 헤더에 전체 선택/해제 체크박스 제공 (현재 페이지 전체 선택) (공통 정책 1.3 적용)
- 체크박스 영역 클릭 시 체크박스만 토글하고, 행 클릭(수정 팝업)은 발생하지 않는다.
- **데이터 범위**: 등록된 모든 사업소득 데이터를 표시한다. 귀속 기준 예외 규칙에 의한 필터링 없이, 각 데이터의 귀속연도/귀속월/지급연도/지급월을 그대로 표시한다.
- 컬럼 정의:

| 컬럼명 | 표시형식 | 정렬 | 비고 |
|--------|---------|------|------|
| 귀속연도 | "YYYY년" | 좌측 | |
| 귀속월 | "MM월" | 좌측 | |
| 지급연도 | "YYYY년" | 좌측 | |
| 지급월 | "MM월" | 좌측 | |
| 성명(상호) | 텍스트 | 좌측 | |
| 주민(사업자)등록번호 | 텍스트 | 좌측 | 마스킹 없이 전체 표시 |
| 업종코드 | 텍스트 | 좌측 | |
| 지급액 | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 4.2, 4.3 적용 |
| 소득세 | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 4.2, 4.3 적용 |
| 지방소득세 | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 4.2, 4.3 적용 |
| 실지급액 | 금액(천 단위 콤마 + "원") | 우측 | 공통 정책 5.1, 4.2, 4.3 적용 |

### 2.4 리스트 개수 선택(페이지 사이즈)

- 공통 정책 1.4 적용
- **위치: 페이지네이션 영역 내부 (테이블 하단)**
- 셀렉박스 제공
- 옵션: 15개 / 30개 / 50개 / 100개
- 기본 선택: 30개
- 페이지 사이즈 변경 시 현재 보고 있던 데이터 위치를 유지한다. (공통 정책 1.4 적용)

### 2.5 엑셀 다운로드

- 공통 정책 1.5 적용
- 다운로드 범위 규칙:
  1. 검색하지 않았을 경우: 전체 리스트 엑셀 다운로드
  2. 검색했을 경우: 검색 결과 리스트 엑셀 다운로드
  3. 리스트 선택 후 버튼 클릭 시: 선택한 리스트만 엑셀 다운로드
- 엑셀 정렬은 화면과 동일
- 데이터 0건일 경우: "다운로드할 데이터가 없습니다" 안내 메시지 표시 (공통 정책 1.5 적용)
- 엑셀 파일 내 금액 컬럼: 숫자 형식(순수 숫자)으로 저장 (콤마, "원" 없이) (공통 정책 1.5 적용)
- **파일명 규칙**: `전체 사업소득.xlsx`

### 2.6 선택 삭제

- 공통 정책 1.6 적용
- 좌측 체크박스 선택 시 활성화
- 데이터 0건일 경우 삭제 버튼 비활성화 (공통 정책 1.6 적용)
- 버튼 클릭 시 confirm 노출
  - 문구: "총 N개의 리스트를 삭제하시겠습니까?"
- **삭제 방식: 물리삭제** (DB에서 완전 삭제)
- 삭제 성공 시 토스트 1.5초 노출 — 문구: "삭제가 완료되었습니다." (공통 정책 1.6 적용)
- 삭제 후 리스트/카운트를 최신 상태로 갱신한다.
- **데이터 연동**: 삭제된 데이터는 SPS_BI_01(사업소득 합산) 및 SPS_BI_02(사업소득 월별 리스트)에서도 즉시 반영된다.
- 삭제 실패 시 처리 정책 (공통 정책 1.6 적용):
  - 건별 트랜잭션 적용, 부분 성공 허용
  - 삭제 실패 리스트 팝업 제공: **성명(상호) + 실패 사유** 표시
  - 삭제 실패한 리스트 개수 및 실패 사유 제공

### 2.7 엑셀 업로드

- 사업소득 합산 화면(SPS_BI_01)의 엑셀 업로드 기능과 **동일**하다.
- 상세 정책은 **[사업소득 엑셀 업로드 정책서(policy_사업소득엑셀업로드_merged.md)](policy_사업소득엑셀업로드_merged.md)** 를 참조한다.
- 엑셀 업로드 완료 후 전체 사업소득 리스트를 **자동 갱신**한다.
- **데이터 연동**: 업로드된 데이터는 SPS_BI_01(사업소득 합산) 및 SPS_BI_02(사업소득 월별 리스트)에서도 즉시 반영된다.

### 2.8 사업소득 추가

- **리스트 상단에 '사업소득 추가' 버튼을 배치한다.**
- 버튼 클릭 시 SPS_BI_06 '전체 사업소득 추가' 팝업을 노출한다.
- **데이터 연동**: 추가된 데이터는 SPS_BI_01(사업소득 합산) 및 SPS_BI_02(사업소득 월별 리스트)에서도 즉시 반영된다.

### 2.9 행 클릭

- 행 클릭 시 SPS_BI_07 '전체 사업소득 수정' 팝업을 노출한다. (공통 정책 1.3 적용)
- 수정 팝업 호출 시 전달하는 키값은 별도 DB PK(id)를 사용한다. (공통 정책 5.2 적용)
- **수정 팝업의 필드 구성 및 상세 정책은 SPS_BI_07 정책서에서 별도 정의한다.**
- **데이터 연동**: 수정된 데이터는 SPS_BI_01(사업소득 합산) 및 SPS_BI_02(사업소득 월별 리스트)에서도 즉시 반영된다.

### 2.10 데이터 표시 규칙

- 본 화면은 **모든 사업소득 데이터를 표시**한다. 연도/월별 필터링 없이 등록된 전체 데이터를 조회한다.
- 귀속 기준 예외 규칙에 의한 재귀속 필터링은 적용하지 않는다. (예외 규칙은 월별 리스트(SPS_BI_02) 및 합산(SPS_BI_01) 화면에만 적용)
- 각 데이터의 귀속연도, 귀속월, 지급연도, 지급월을 **저장된 원본 값 그대로** 표시한다.

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
|--------------|--------------|------------|------|
| 1.6 선택 삭제 confirm 문구 | "총 N개의 리스트를 삭제하시겠습니까?" | 그대로 적용 | 충돌 없음 |
| (추가) 데이터 연동 | 공통 정책에 없음 | **추가** — 데이터 추가/수정/삭제 시 SPS_BI_01, SPS_BI_02 화면과 양방향 연동 | 월별 사업소득 메뉴와 전체 사업소득 메뉴가 동일 데이터 소스를 공유하므로 한쪽에서 변경 시 다른 쪽에도 반영 |
| (추가) 엑셀 업로드 | 공통 정책에 없음 | **추가** — SPS_BI_01 엑셀 업로드 기능과 동일한 기능 제공 | 화면별 추가 기능 |

> 본 화면은 공통 정책과 대부분 일치하며, 주요 충돌 없이 공통 정책을 그대로 적용한다.
> 데이터 연동 및 엑셀 업로드 기능은 공통 정책에 없는 화면 전용 추가 기능이다.
