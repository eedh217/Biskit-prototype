# 사업소득 엑셀 업로드 기능(SPS_BI_01) - 최종 정책서

---

## 1. 기능 개요

- **화면 ID**: SPS_BI_01
- **화면명**: 사업소득 합산
- **기능명**: 사업소득 엑셀 업로드
- **목적**: 사업소득 합산 화면에서 엑셀 파일을 통해 사업소득 데이터를 대량으로 업로드하는 기능
- **공통 정책 참조**: excel-upload-policy (공통 엑셀 업로드 정책)
- **연관 정책**: policy_사업소득추가팝업_merged.md (세율/계산 로직 동일)

---

## 2. 요청한 기능 정의

### 2.1 엑셀 업로드 버튼 (excel-upload-policy 적용)

- 사업소득 합산 화면에 **엑셀 업로드** 버튼을 제공한다.
- 버튼 클릭 시 2가지 옵션을 제공한다:
  - **파일 올리기**
  - **엑셀 양식 받기**

---

### 2.2 파일 올리기 (excel-upload-policy 1-1 적용)

- 시스템 파일 선택 팝업 노출
- 엑셀 파일만 업로드 가능
  - 허용 확장자: `.xlsx`, `.xls`
  - 파일 선택 팝업에서 `.xlsx`/`.xls`만 필터링하여 허용되지 않는 확장자는 **선택 자체 불가**
- 파일은 **1개만** 업로드 가능
- 다중 선택 불가
- **파일 크기 제한**: 최대 **10MB**

---

### 2.3 엑셀 양식 받기 (excel-upload-policy 1-2 적용)

- 양식 파일 즉시 다운로드
- 양식 파일명: `간이지급명세서_사업소득_업로드_양식.xlsx`

---

### 2.4 엑셀 양식 구조

- **Row 1~9**: 헤더/안내 영역
- **Row 10**: 컬럼명
- **Row 11**: 컬럼코드
- **Row 12**: 샘플 데이터 (업로드 시 **자동 무시**, 데이터로 처리되지 않음)
- **Row 13부터**: 실제 데이터 입력 영역 (13행부터 데이터로 인식)

#### 컬럼 정의

| 컬럼코드 | 컬럼명 | 필수여부 | 비고 |
|----------|--------|---------|------|
| name | *성명(상호) | 필수 | 최대 50자 |
| attributionYear | *귀속연도 | 필수 | 숫자만 허용 |
| attributionMonth | *귀속월 | 필수 | 1~12 숫자 |
| paymentYear | *지급연도 | 필수 | 숫자만 허용 |
| paymentMonth | *지급월 | 필수 | 1~12 숫자 |
| iino | *주민(사업자)등록번호 | 필수 | 최대 숫자 13자리, 13자리일 경우 주민등록번호 형식 검증 |
| isForeign | *내외국인 구분 (내국인:N, 외국인:Y) | 필수 | N 또는 Y |
| industryCode | *업종코드 | 필수 | 유효한 업종코드만 허용 |
| paymentSum | *지급액 | 필수 | 양의 정수만 허용, 최대 12자리 |
| taxRate | 세율 | 조건부 필수 | 외국인(Y) + 직업운동가(940904)일 경우에만 필수 (3 또는 20) |

---

### 2.5 업로드 기본 검증 (excel-upload-policy 2 적용)

- 엑셀 컬럼 구조는 양식과 **완전히 일치해야 한다**
- 컬럼 불일치 시 → **전체 업로드 실패**

#### 전체 실패 조건 및 에러 메시지

| 조건 | 에러 메시지 |
|------|------------|
| 컬럼 구조 불일치 | "양식에 맞지 않는 엑셀파일입니다." |
| 양식 외 입력 데이터가 없음 | "데이터가 없습니다. 데이터 입력 후 파일을 업로드해주세요." |

### 2.6 Row 단위 유효성 검사

엑셀 구조가 정상일 경우, 각 row별로 아래 유효성 검증을 수행한다.
실패 row만 실패 처리하고, 성공 row는 정상 저장한다.

| 순번 | 검사 항목 | 조건 | 에러 메시지 |
|------|----------|------|------------|
| 1 | 필수값 미입력 | 세율을 제외한 컬럼이 비어있을 경우 | "세율을 제외한 모든 컬럼이 필수입력값입니다." |
| 2 | 귀속연도 숫자 검증 | 정수가 아닐 경우 (소수점 포함 시 에러) | "귀속연도는 숫자로 입력해주세요." |
| 3 | 귀속연도 범위 검증 | 2025년 미만이거나 현재연도를 초과할 경우 | "귀속연도는 2025년부터 YYYY년까지 입력 가능합니다." (YYYY = 현재연도) |
| 4 | 지급연도 숫자 검증 | 정수가 아닐 경우 (소수점 포함 시 에러) | "지급연도는 숫자로 입력해주세요." |
| 5 | 지급연도 범위 검증 | 2025년 미만이거나 (현재연도+1)년을 초과할 경우 | "지급연도는 2025년부터 YYYY년까지 입력 가능합니다." (YYYY = 현재연도+1) |
| 6 | 귀속월 범위 검증 | 1~12 이내 정수가 아닐 경우 | "귀속월은 1~12 내 숫자로 입력해주세요." |
| 7 | 지급월 범위 검증 | 1~12 이내 정수가 아닐 경우 | "지급월은 1~12 내 숫자로 입력해주세요." |
| 8 | 지급연월 vs 귀속연월 선후관계 | 지급연월이 귀속연월보다 이전 날짜일 경우 | "지급연월은 귀속연월 이후 날짜여야 합니다." |
| 9 | 내외국인 구분 값 검증 | N, Y 이외 문자일 경우 | "내외국인 구분은 N, Y로 입력해주세요." |
| 10 | 업종코드 유효성 검증 | 시스템 업종코드 마스터 테이블에 등록되지 않은 업종코드일 경우 | "유효하지 않은 업종코드입니다." |
| 11 | 지급액 숫자 검증 | 정수가 아닌 문자일 경우 (소수점 포함 시 에러) | "지급액은 숫자만 입력 가능합니다." |
| 12 | 지급액 양수 검증 | 지급액이 0 이하일 경우 | "지급액을 양수로 입력해주세요." |
| 13 | 지급액 자릿수 검증 | 지급액이 12자리를 초과할 경우 | "지급액은 최대 12자리까지 입력 가능합니다." |
| 14 | 성명(상호) 길이 검증 | name이 50자를 초과할 경우 | "성명(상호)는 최대 50자까지 입력 가능합니다." |
| 15 | 주민(사업자)등록번호 자릿수 검증 | iino가 숫자 13자리를 초과할 경우 | "주민(사업자)등록번호는 최대 13자리까지 입력 가능합니다." |
| 16 | 주민등록번호 형식 검증 | iino가 13자리일 경우 주민등록번호 형식이 유효하지 않을 경우 | "유효하지 않은 주민등록번호 형식입니다." |
| 17 | 외국인 직업운동가 세율 검증 | 외국인(Y) + 업종코드 940904이면서 세율이 3 또는 20이 아닌 경우 | "외국인 직업운동가일 경우, 세율을 3 또는 20으로 입력해주세요." |
| 18 | 중복 데이터 검사 | 귀속연월 + 지급연월 + 주민(사업자)등록번호 + 업종코드가 동일한 데이터가 이미 존재할 경우 (파일 내 중복 시 첫 번째 row 성공, 이후 row 실패) | "지급연월 기준 귀속연월, 주민(사업자)등록번호, 업종코드가 동일한 사업소득이 존재합니다." |

> **복수 에러 처리**: 한 row에서 여러 유효성 검사가 실패할 경우, 검증 순서(순번)에 따라 **첫 번째 에러만** 표시한다.

### 2.7 내외국인 구분 처리

- **N**: 내국인
- **Y**: 외국인

---

### 2.8 세율 처리 정책

#### 2.8.1 세율 결정 로직

| 조건 | 세율 | 세율 입력값 처리 |
|------|------|----------------|
| 기본 (일반적인 경우) | **3%** | 세율 입력값 무시 (기본값 적용) |
| 업종코드 = 봉사료수취자(940905) | **5%** | 세율 입력값 무시 (5% 고정) |
| 내외국인 = N(내국인) AND 업종코드 = 직업운동가(940904) | **3%** | 기본 세율 3% 적용, 세율 입력값 무시 |
| 내외국인 = Y(외국인) AND 업종코드 = 직업운동가(940904) | **3% 또는 20%** (사용자 입력) | 세율 필수 입력, 3 또는 20만 허용 |

#### 2.8.2 세율 입력값 무시 규칙

- 외국인(Y) + 직업운동가(940904) **이외**의 조건에서 세율을 입력한 경우:
  - 입력 데이터를 **무시**한다.
  - 기본 세율(3%)을 적용하며, 봉사료수취자(940905)일 경우 5%를 적용한다.

### 2.9 자동 계산 로직 (사업소득 추가 팝업과 동일)

각 row의 세율이 확정된 후, 서버에서 아래 순서로 자동 계산하여 저장한다.

#### 2.9.1 계산 순서

1. **세율 확정** (2.8 세율 결정 로직에 따라)
2. **소득세 계산**: 소득세 = 지급액 × 세율 (원단위 이하 절사)
3. **소액부징수 판정**: 소득세가 1,000원 미만인 경우 → 소득세 = **0원**
4. **지방소득세 계산**:
   - 소득세 = 0원인 경우 → 지방소득세 = **0원**
   - 소득세 > 0원인 경우 → 지방소득세 = 소득세 × 0.1 (원단위 이하 절사)
5. **실지급액 계산**: 실지급액 = 지급액 - 소득세 - 지방소득세

#### 2.9.2 계산 예시

**예시 1. 지급액 30,000원 / 세율 3%**

| 항목 | 계산 | 결과 |
|------|------|------|
| 소득세 | 30,000 × 0.03 | 900원 |
| 소액부징수 판정 | 900 < 1,000 | 소득세 = **0원** |
| 지방소득세 | 소득세 0원이므로 | **0원** |
| 실지급액 | 30,000 - 0 - 0 | **30,000원** |

**예시 2. 지급액 100,000원 / 세율 3%**

| 항목 | 계산 | 결과 |
|------|------|------|
| 소득세 | 100,000 × 0.03 | **3,000원** |
| 소액부징수 판정 | 3,000 >= 1,000 | 소득세 유지 |
| 지방소득세 | 3,000 × 0.1 | **300원** |
| 실지급액 | 100,000 - 3,000 - 300 | **96,700원** |

### 2.10 데이터 저장 정책

- **저장 모드**: 신규 등록
- 성공 row는 **즉시** 신규 저장 (부분 실패 시에도 성공 row는 즉시 저장)
- **지급연도/지급월 기준 저장**: 엑셀 내 지급연도, 지급월 값에 따라 해당 연도/월의 사업소득 데이터로 저장한다.
  - 예: 엑셀의 지급연도/지급월이 2026년 2월일 경우 → 사업소득 합산 화면의 2026년 2월 데이터로 저장
- **대표 식별값**: 주민(사업자)등록번호

### 2.10.1 귀속 기준 예외 규칙 (간이지급명세서 표시 기준)

- 저장된 데이터가 월별 리스트 및 합산 화면에 **표시되는 월**은 귀속 기준 예외 규칙에 따라 달라질 수 있다.
- 예외 적용 대상 업종: 보험설계사(940906), 음료배달(940907), 방문판매원(940908)
- 예외 적용 조건: 위 업종코드에 해당하면서 **귀속연도 ≠ 지급연도**인 경우 (귀속월은 고려하지 않고 연도 기준으로만 판정)
- 예외 처리: 해당 데이터는 지급연월 기준 리스트가 아닌 **귀속연도 12월** 리스트에 표시된다.
  - 예: 엑셀 데이터의 귀속연월 2025/08, 지급연월 2026/02, 보험설계사(940906) → 저장은 지급연월(2026/02) 기준, 표시는 **2025년 12월** 리스트
- 그 외 모든 경우: 지급연월 기준으로 표시

### 2.11 업로드 결과 화면 (excel-upload-policy 적용)

업로드 완료 후 반드시 다음을 표시한다:

- **총 N건**
- **성공 건수**
- **실패 건수**
- **실패 row 목록**

#### 실패 row 표시 규칙

각 실패 row는 다음을 포함한다:

- **대표 식별값**: 주민(사업자)등록번호
- **실패 사유**: 해당 row의 유효성 검사 실패 메시지

---

### 2.12 실패 데이터 다운로드 (excel-upload-policy 적용)

부분 실패가 발생한 경우 반드시 다음 기능을 제공한다:

- **버튼명**: "실패 데이터 다운로드"
- 실패 row만 포함된 엑셀 파일 생성
- **파일명**: `사업소득업로드_실패목록_YYYYMMDD.xlsx` (업로드 일시 기준)
- **파일 구성**:
  - 기존 업로드 컬럼 유지
  - 마지막 컬럼에 **"실패사유"** 컬럼 추가
  - 각 실패 row에 실패 사유 입력
- **목적**: 사용자가 수정 후 재업로드 가능하도록 지원

### 2.13 업로드 완료 처리

#### 전체 성공 시

- 토스트 팝업 **1.5초** 노출
- 문구: **"엑셀 업로드를 완료했습니다."**
- 사업소득 합산 화면 데이터 새로고침
- 해당 지급연월의 데이터 반영

#### 부분 실패 시

- 토스트 **미노출**
- 성공/실패 집계 표시
- 실패 목록 표시
- **실패 데이터 다운로드** 버튼 노출
- **확인** 버튼 제공 (클릭 시 합산 화면으로 이동, 성공 건 반영된 상태로 새로고침)

#### 전체 실패 시 (row 단위)

- 부분 실패와 **동일한 UI** 표시 (성공 0건, 실패 N건)
- 실패 목록 표시
- **실패 데이터 다운로드** 버튼 노출
- **확인** 버튼 제공

---

### 2.14 업로드 진행 중 이탈 처리

- 업로드가 진행 중일 때 사용자가 페이지를 이탈(뒤로가기, 새로고침, 다른 메뉴 클릭)하려 할 경우 **확인 팝업**을 표시한다.
  - 팝업 문구: "업로드가 진행 중입니다. 페이지를 떠나시겠습니까?"
  - **확인** 버튼 클릭 시: 업로드 취소, 페이지 이탈
  - **취소** 버튼 클릭 시: 팝업 닫기, 업로드 계속 진행

---

### 2.15 동시 업로드 처리

- 동일 사용자 또는 다른 사용자가 동시에 엑셀 업로드를 수행할 경우 **큐잉 처리**한다.
- 먼저 요청된 업로드가 완료된 후 다음 업로드가 처리된다.

### 2.16 제한 정책 (excel-upload-policy 적용)

- 여러 파일 업로드 불가
- 자동 컬럼 매핑 금지
- 컬럼 순서 무시 로직 금지
- 임의 컬럼 허용 금지

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 기능 적용 | 사유 |
|--------------|--------------|------------|------|
| excel-upload-policy 대표 식별값 | 화면에서 정의 | **주민(사업자)등록번호** | 사업소득의 고유 식별값으로 주민(사업자)등록번호 사용 |
| excel-upload-policy 실패 파일명 | 파일명 예시: 급여업로드_실패목록_20260204 | **변경** → 사업소득업로드_실패목록_YYYYMMDD | 화면명에 맞게 파일명 변경 |
| excel-upload-policy 저장 모드 | 화면에서 정의 | **신규 등록** | 엑셀 업로드 데이터는 항상 신규 등록 |
| 공통 정책 5.x 추가 팝업 | 추가 팝업 공통 정책 | **미적용** | 본 기능은 엑셀 업로드 기능이므로 추가 팝업 정책 해당 없음 |
| 공통 정책 5.x 수정 팝업 | 수정 팝업 공통 정책 | **미적용** | 본 기능은 엑셀 업로드 기능이므로 수정 팝업 정책 해당 없음 |

> 본 기능은 excel-upload-policy(공통 엑셀 업로드 정책)를 기본으로 적용하며, 화면별 세율/계산 로직은 사업소득 추가 팝업(SPS_BI_03)과 동일하게 적용한다.
