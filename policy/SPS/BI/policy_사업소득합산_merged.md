# 사업소득 합산 화면(SPS_BI_01) - 최종 정책서

---

## 1. 화면 개요

- **화면 ID**: SPS_BI_01
- **화면명**: 사업소득 합산
- **목적**: 선택한 연도의 1월~12월 사업소득 데이터를 월별로 합산하여 조회하는 화면
- **주요 기능**: 연도 선택, 월별 합산 데이터 조회, 신고파일 다운로드, 월별 상세 화면 이동, 사업소득 엑셀 업로드
- **연관 화면**: 사업소득 월별 리스트(2번 화면) — 행 클릭 시 이동
- **연관 기능 정책**: [사업소득 엑셀 업로드 정책서](policy_사업소득엑셀업로드_merged.md) — 엑셀 업로드 기능 상세 정책 참조

---

## 2. 요청한 화면 기능

### 2.1 연도 선택 셀렉박스

- 화면 상단에 연도 셀렉박스를 제공한다.
- 선택 가능 범위: **2026년 ~ (현재 연도 + 1)년**
  - 예: 현재 2026년 → 2026년, 2027년
  - 예: 현재 2028년 → 2026년, 2027년, 2028년, 2029년
- 기본 선택값: **현재 연도**
- 연도 선택 시 하단 테이블 데이터를 해당 연도 기준으로 갱신한다.
- 항상 기본값(현재 연도)이 선택되어 있으므로 placeholder("선택")는 적용하지 않는다.

### 2.2 월별 합산 테이블

- 행: 1월 ~ 12월 (기본 12행). 12월 예외 데이터 존재 시 12월 하단에 추가 2행 생성 (2.8 참조)
- 기본 정렬: 1월 → 12월 순 (월 오름차순 고정)
- 컬럼 정의:

| 컬럼명 | 표시형식 | 정렬 | 비고 |
|--------|---------|------|------|
| 월 | "N월" | 좌측 | 1월~12월 고정 |
| 건수(소득자건수) | 숫자 | 우측 | 해당 월 소득자 행 개수 (합산 후 row 개수). 월별 리스트 화면(SPS_BI_02)의 상단 요약 정보 건수와 동일 |
| 총 지급액 | 금액(천 단위 콤마 + "원") | 우측 | 해당 월 지급액 합계 |
| 총 소득세 | 금액(천 단위 콤마 + "원") | 우측 | 해당 월 소득세 합계 |
| 총 지방소득세 | 금액(천 단위 콤마 + "원") | 우측 | 해당 월 지방소득세 합계 |
| 총 실지급액 | 금액(천 단위 콤마 + "원") | 우측 | 해당 월 실지급액 합계 |
| 신고파일 최종생성일 | 날짜(YYYY-MM-DD HH:MM:SS) | 중앙 | 가장 최근 신고파일 생성 날짜와 시간. 미생성 시 "-" 표시 |
| 다운로드 | 버튼 | 중앙 | 신고파일 다운로드. 미생성 시 버튼 공간 유지 후 숨김(visibility:hidden) |

- **데이터 출처**: 2번 화면(해당 월 사업소득 리스트)의 개인별 지급액/소득세/지방소득세/실지급액의 합계
- **금액 표시**: 공통 정책 5.1 적용 — 천 단위 콤마, 공통 정책 5.3 적용 — "원" 단위 표기
  - 예: 1,000,000원
- **금액 정렬**: 공통 정책 5.2 적용 — 우측 정렬
- **음수 금액**: 음수 금액은 발생하지 않는다.
- **데이터 없는 월**: 건수 0, 금액 컬럼은 0원으로 표시

### 2.3 행 클릭 (월별 상세 화면 이동)

- 행 클릭 시 해당 월의 사업소득 리스트 화면(2번 화면)으로 이동한다.
- **클릭 영역**: 다운로드 버튼 영역을 **제외한** 행 전체
- **예외**: 12월 하단 추가 행("YYYY년 지급", "YYYY년 이후 지급")은 클릭할 수 없다. 행 클릭 이벤트를 연결하지 않는다. (2.8 참조)

### 2.4 다운로드

- 각 월별 행에 다운로드 버튼을 제공한다.
- 신고파일이 생성된 경우, 가장 최근에 생성한 신고파일을 다운로드한다.
- **신고파일 미생성 시**: 다운로드 버튼을 공간 유지 후 숨김(visibility:hidden) 처리한다. 버튼 공간은 그대로 차지한다.
- **12월 하단 추가 행**: 다운로드 버튼을 제공하지 않는다. (2.8 참조)
- 다운로드 파일명: `YYYY년 M월 간이지급명세서.xlsx`
  - 예: `2026년 1월 간이지급명세서.xlsx`
- **신고파일 데이터 범위**: 신고파일 생성 시에도 귀속 기준 예외 규칙(2.7)이 동일하게 적용된다. 해당 월의 신고파일에는 예외 규칙에 의해 재귀속된 데이터가 포함된다.
  - 예: 2025년 12월 신고파일에는 예외 업종(보험설계사/음료배달/방문판매원)으로서 귀속연도 2025년, 지급연도 2026년인 데이터가 포함된다.

### 2.5 엑셀 업로드

- 사업소득 합산 화면에 **엑셀 업로드** 버튼을 제공한다.
- 엑셀 파일을 통해 사업소득 데이터를 대량으로 업로드할 수 있다.
- 엑셀 업로드 완료 후 합산 테이블 데이터를 **자동 갱신**한다.
- 엑셀 업로드 기능의 상세 정책은 **[사업소득 엑셀 업로드 정책서(policy_사업소득엑셀업로드_merged.md)](policy_사업소득엑셀업로드_merged.md)** 를 참조한다.

### 2.6 신고파일 최종생성일

- 2번 화면에서 신고파일 생성 기능을 통해 가장 최근에 생성한 날짜와 시간을 표시한다.
- **표시 형식**: YYYY-MM-DD HH:MM:SS
  - 예: 2026-03-15 14:30:25
- **신고파일 미생성 시**: "-" 표시
- **12월 하단 추가 행**: 신고파일 최종생성일을 표시하지 않는다. (2.8 참조)

### 2.7 귀속 기준 예외 규칙 (간이지급명세서)

- 기본 원칙: 사업소득 데이터는 **지급연월** 기준으로 해당 월의 합산 데이터에 포함된다.
- 예외 적용 대상 업종:
  - 보험설계사 (940906)
  - 음료배달 (940907)
  - 방문판매원 (940908)
- 예외 적용 조건: 위 업종코드에 해당하면서, **귀속연도와 지급연도가 서로 다른 경우** (귀속월은 고려하지 않고 연도 기준으로만 판정)
- 예외 처리: 해당 데이터는 지급연월이 아닌 **귀속연도 12월** 합산 데이터에 포함시킨다.
  - 예: 귀속연월 2025년 8월, 지급연월 2026년 2월, 업종코드 보험설계사(940906) → **2025년 12월** 합산에 포함, 2026년 2월 합산에는 미포함
- 그 외 모든 경우 (예외 업종이 아닌 일반 업종에서 귀속연도와 지급연도가 다른 경우 포함): **지급월** 기준으로 합산
- 판정 흐름:
  1. 업종코드가 보험설계사(940906), 음료배달(940907), 방문판매원(940908)인지 확인
  2. 귀속연도와 지급연도 비교
  3. 서로 다를 경우 → 귀속연도 12월로 재귀속
  4. 그 외 → 지급연월 기준 유지

### 2.8 12월 예외 데이터 표시 규칙

#### 2.8.1 적용 조건

다음 조건을 **모두** 만족하는 데이터가 **1건 이상** 존재하는 경우에만, 12월 하단에 추가 행을 생성한다.

- 업종코드가 다음 중 하나: 보험설계사(940906), 음료배달(940907), 방문판매원(940908)
- 귀속연도와 지급연도가 서로 다른 경우
- 해당 데이터가 현재 선택한 연도의 합산 대상에 포함되는 경우

예외 데이터가 존재하지 않으면 기존 1월~12월 12행만 표시한다.

#### 2.8.2 12월 기본 행

- 12월 기본 행은 기존과 동일하게 **예외 데이터를 포함한 전체 12월 합산**을 표시한다.
- 추가 2행은 12월 기본 행의 **소계(내역 분류)** 역할이다.
- 클릭, 신고파일 최종생성일, 다운로드 버튼 등 기존 기능을 그대로 유지한다.

#### 2.8.3 추가 행 구성

12월 기본 행 하단에 다음 두 개의 행을 추가한다. (YYYY = 귀속연도)

| 행 라벨 | 데이터 합산 기준 |
|---------|----------------|
| "YYYY년 지급" | 업종 무관, 지급연월이 YYYY년 12월인 전체 데이터의 합계. 단, 예외 업종(보험설계사(940906)/음료배달(940907)/방문판매원(940908))은 귀속연도와 지급연도가 다른 경우 제외한다. |
| "YYYY년 이후 지급" | 귀속연도 = YYYY이고 지급연도가 YYYY년보다 이후인 데이터의 합계 |

- 예: 귀속연도 2025년 → "2025년 지급", "2025년 이후 지급"
- "YYYY년 지급" 예시:
  - 일반 업종(940903), 지급연월 2025년 12월 → 합산 대상 (업종 무관)
  - 보험설계사(940906), 귀속연도 2025년, 지급연월 2025년 12월 → 합산 대상 (귀속연도 = 지급연도)
  - 보험설계사(940906), 귀속연도 2024년, 지급연월 2025년 12월 → 제외 (귀속연도 ≠ 지급연도)
- 추가 행의 컬럼은 기본 행과 동일하며 **건수(소득자건수)를 포함**한다. 단, **신고파일 최종생성일**과 **다운로드 버튼**은 표시하지 않는다.

#### 2.8.4 추가 행 UI 동작

- 행 클릭 이벤트를 연결하지 않는다. (상세 화면 이동 불가)
- 커서 포인터 등 클릭 가능 UI 피드백을 제공하지 않는다.

#### 2.8.5 추가 행 시각적 구분

- 추가 2행은 12월 기본 행 및 일반 월별 행과 **시각적으로 구분**한다.
- 구분 방법: 다음 세 가지를 모두 적용한다.
  - 배경색 변경
  - 들여쓰기
  - 폰트 크기 살짝 작게

#### 2.8.6 표시 조건 정리

| 상황 | 표시 |
|------|------|
| 예외 데이터 없음 | 기존 1월~12월 12행만 표시 |
| 예외 데이터 있음 | 12월 하단에 "YYYY년 지급", "YYYY년 이후 지급" 2행 추가 (총 14행) |
| 예외 데이터가 실제 존재하지 않는 경우 | 추가 행을 생성하지 않음 (데이터 0건이면 미생성) |

#### 2.8.7 처리 흐름

1. 현재 선택 연도 기준 합산 데이터 조회
2. 예외 업종 + 귀속연도 ≠ 지급연도 데이터 존재 여부 확인
3. 존재할 경우 12월 하단에 2개 행 추가
4. "YYYY년 지급" / "YYYY년 이후 지급" 데이터 분리 집계
5. 추가 행은 클릭 비활성 처리
6. 추가 행에는 신고파일 정보 미노출

### 2.9 접근 권한

- 본 화면은 **모든 로그인 사용자**가 접근할 수 있다.
- 별도의 역할/권한 제한을 적용하지 않는다.

### 2.10 로딩 상태 표시

- 연도 변경 시 또는 초기 로딩 시 데이터를 불러오는 동안 **로딩 인디케이터**(스피너)를 표시한다.

## 3. 충돌 지점 정리

| 공통 정책 항목 | 공통 정책 내용 | 본 화면 적용 | 사유 |
|--------------|--------------|------------|------|
| 1.3 행 클릭 → 수정 팝업 | 행 클릭 시 수정 팝업 노출 | **변경** → 월별 상세 화면 이동 | 합산 화면이므로 수정 팝업이 아닌 해당 월 리스트 화면으로 이동 |
