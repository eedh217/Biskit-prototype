# 사업소득 간이지급명세서 생성 팝업(SPS_BI_08) - 최종 정책서

---

## 1. 화면 개요

- **화면 ID**: SPS_BI_08
- **화면명**: 사업소득 간이지급명세서 생성 팝업
- **목적**: 사업소득 월별 리스트 화면(SPS_BI_02)에서 선택한 대상자들의 간이지급명세서를 생성하고 다운로드하기 위한 팝업
- **진입 경로**: SPS_BI_02 화면의 '간이지급명세서 생성' 버튼 클릭 → Popover에서 대상 선택 후 노출
- **전달 파라미터**:
  - `year`: 현재 조회 중인 연도
  - `month`: 현재 조회 중인 월
  - `targetIds`: 선택된 개별 레코드 ID 배열
  - `targetType`: 'all' (조회된 전체 대상자) 또는 'selected' (선택 대상자)
- **접근 권한**: 별도 권한 제한 없음 (모든 사용자 접근 가능)
- **연관 화면**:
  - SPS_BI_02 (사업소득 월별 리스트) - 부모 화면
  - SPS_BI_01 (사업소득 합산) - 신고파일 최종생성일 업데이트 대상

## 2. 요청한 화면 기능

### 2.1 닫기/이탈 방지 (공통 정책 5.1 적용)

- 팝업을 벗어나려는 모든 행동(팝업 외 영역 클릭, ESC 키 입력, 닫기 버튼 클릭, 취소 버튼 클릭, 브라우저 뒤로가기 등) 발생 시 confirm 노출
- 단, 다음의 경우에는 confirm 없이 팝업을 바로 닫는다:
  1. 사용자가 아무런 입력도 하지 않은 초기 상태
  2. 기존 입력값(LocalStorage에 저장된 값)에서 변경된 내용이 없을 경우
- **초기 상태 판정 기준**: 모든 입력 항목의 최종값이 초기값(LocalStorage에서 불러온 값)과 동일하면 초기상태로 판정한다.
- confirm 문구: **"간이지급명세서 생성을 취소하시겠습니까?"**
  - 확인 버튼 클릭 시: 팝업 모두 닫힘 (간이지급명세서 생성 팝업 + 대상 선택 Popover 모두 닫힘)
  - 취소 버튼 클릭 시: 간이지급명세서 생성 팝업 유지

### 2.2 상단 요약 정보

해당 월의 선택된 대상자에 대한 사업소득 간이지급명세서 요약 정보를 상단에 표시한다.

- **계산 대상**: 전달받은 `targetIds`에 해당하는 개별 레코드들
  - 합산 행인 경우: 그룹 내 모든 개별 레코드의 값을 다시 합산
- **표시 항목**:

| 항목명 | 표시형식 | 계산 방식 | 비고 |
|--------|---------|----------|------|
| 건수(소득자 건수) | 숫자 + "건" | 선택된 행 개수 (합산 행도 1건으로 카운트) | 예: 합산 행 1개 + 개별 행 3개 = 4건 |
| 총 지급액 | 금액(천 단위 콤마 + "원") | 선택된 대상자들의 지급액 합계 | 공통 정책 5.1, 4.3 적용 |
| 총 소득세 | 금액(천 단위 콤마 + "원") | 선택된 대상자들의 소득세 합계 | 공통 정책 5.1, 4.3 적용 |
| 총 지방소득세 | 금액(천 단위 콤마 + "원") | 선택된 대상자들의 지방소득세 합계 | 공통 정책 5.1, 4.3 적용 |
| 총 실지급액 | 금액(천 단위 콤마 + "원") | 선택된 대상자들의 실지급액 합계 | 공통 정책 5.1, 4.3 적용 |

- 금액 표시: 공통 정책 5.1(천 단위 콤마), 4.3("원" 단위 표기) 적용
  - 예: 1,000,000원

### 2.3 입력 항목 정의

다음 항목은 **모두 필수 입력값**이다.

| 항목명 | 입력 타입 | 제약 조건 | 기본값 | 비고 |
|--------|----------|----------|--------|------|
| 상호(법인명) | 단일 인풋박스 | 최대 30자, 허용 문자: 한글·영문·숫자·공백·허용 특수문자(&, ', -, ., ·, (, )), 공백만 입력 시 trim 후 빈값 판정(필수 에러) | LocalStorage에서 불러온 값 | 허용되지 않은 문자는 입력 불가 |
| 대표자 성명 | 단일 인풋박스 | 최대 30자, 한글·영문·숫자·공백만 입력 가능, 특수문자 입력 불가, 공백만 입력 시 trim 후 빈값 판정(필수 에러) | LocalStorage에서 불러온 값 | 허용되지 않은 문자는 입력 불가 |
| 사업자등록번호 | 단일 인풋박스 | 숫자만 입력 가능, 정확히 10자리만 허용 | LocalStorage에서 불러온 값 | 숫자 외 문자 입력 차단 |
| 법인등록번호 | 단일 인풋박스 | 숫자만 입력 가능, 정확히 13자리만 허용 | LocalStorage에서 불러온 값 | 숫자 외 문자 입력 차단 |
| 담당자 성명 | 단일 인풋박스 | 최대 30자, 한글·영문·숫자·공백만 입력 가능, 특수문자 입력 불가, 공백만 입력 시 trim 후 빈값 판정(필수 에러) | LocalStorage에서 불러온 값 | 허용되지 않은 문자는 입력 불가 |
| 담당자 부서 | 단일 인풋박스 | 최대 30자, 한글·영문·숫자·공백만 입력 가능, 특수문자 입력 불가, 공백만 입력 시 trim 후 빈값 판정(필수 에러) | LocalStorage에서 불러온 값 | 허용되지 않은 문자는 입력 불가 |
| 담당자 전화번호 | 단일 인풋박스 | 최대 15자, 숫자·하이픈(-)·괄호( ( ) )만 입력 가능 | LocalStorage에서 불러온 값 | 허용되지 않은 문자는 입력 불가 |
| 홈택스ID | 단일 인풋박스 | 최대 20자, 영문·숫자만 입력 가능, 공백 입력 불가 (입력 중 공백 차단) | LocalStorage에서 불러온 값 | 허용되지 않은 문자는 입력 불가 |
| 관할세무서 코드 | 단일 인풋박스 | 최대 3자리 | LocalStorage에서 불러온 값 | 허용 문자 제한 없음 |
| 생성목적 | 단일 인풋박스 | 최대 50자 | LocalStorage에서 불러온 값 | 허용 문자 제한 없음, 특수문자 허용 |

#### 입력 제한 정리

- **숫자만 입력 가능**: 사업자등록번호, 법인등록번호 → 입력 중 숫자 외 문자 입력 차단
- **한글·영문·숫자·공백만**: 대표자 성명, 담당자 성명, 담당자 부서 → 특수문자 입력 차단
- **허용 특수문자 있음**: 상호(법인명) - &, ', -, ., ·, (, )
- **숫자·하이픈·괄호만**: 담당자 전화번호 - 0-9, -, (, )
- **영문·숫자만**: 홈택스ID - a-z, A-Z, 0-9 (공백 차단)
- **제한 없음**: 관할세무서 코드, 생성목적

### 2.4 유효성 검사 (공통 정책 5.3 적용)

- 검증 타이밍: **blur / submit**
- 에러 표시: 컴포넌트 하단에 에러 문구 노출
- 재입력 후 검증 통과 시 에러 문구 제거
- 포커스 아웃 시 정상 입력으로 판단되면 에러 문구 제거

#### 2.4.1 필수값 검증 (blur 시)

- 모든 필수 입력 항목이 미입력 상태에서 blur 시, 해당 필드 하단에 에러 문구 노출
- **공백만 입력된 경우 trim 후 빈값으로 판정**하여 동일하게 필수 에러 처리
- 에러 문구: **"필수 입력 항목입니다."**

#### 2.4.2 사업자등록번호 자릿수 검사 (blur 시)

- 사업자등록번호는 **정확히 10자리**만 유효하다.
- 10자리가 아닌 경우 blur 시 에러 처리
- 에러 문구: **"사업자등록번호는 10자리만 입력 가능합니다."**

#### 2.4.3 법인등록번호 자릿수 검사 (blur 시)

- 법인등록번호는 **정확히 13자리**만 유효하다.
- 13자리가 아닌 경우 blur 시 에러 처리
- 에러 문구: **"법인등록번호는 13자리만 입력 가능합니다."**

### 2.5 생성 완료 및 다운로드

#### 2.5.1 생성 버튼 클릭 시 동작

1. **유효성 검증**: 모든 필수 입력 항목 검증
2. **입력값 저장**: LocalStorage에 입력값 저장 (키: `biskit_sps_bi_creation_form`)
3. **간이지급명세서 생성**:
   - 신고파일 최종생성일 업데이트 (2.7 참조)
4. **파일 다운로드**: PDF 파일 자동 다운로드 시작
   - **파일명**: `YYYY년 MM월 사업소득 간이지급명세서.pdf` (YYYY=연도, MM=월)
   - **파일 형식**: PDF (현재는 테스트이므로 빈 PDF 파일)
5. **Alert 노출**: 다운로드 시작과 동시에 Alert 노출
   - 문구: **"YYYY년 MM월 사업소득 간이지급명세서 생성이 완료되었습니다."** (YYYY=연도, MM=월)
6. **팝업 닫기**: Alert 확인 버튼 클릭 시 팝업 닫기 (간이지급명세서 생성 팝업 + 대상 선택 Popover 모두 닫힘)
7. **부모 화면 갱신**: SPS_BI_02 리스트/카운트/상단 요약 정보를 최신 상태로 갱신

#### 2.5.2 취소 버튼 클릭 시 동작

- confirm 로직 적용 (2.1 참조)
- 변경 사항이 있으면 confirm 노출
- 변경 사항이 없으면 바로 팝업 닫기

### 2.6 입력값 저장

- **저장 방식**: LocalStorage
- **저장 키**: `biskit_sps_bi_creation_form`
- **저장 시점**: 생성 버튼 클릭 시
- **저장 내용**: 모든 입력 항목의 값
- **불러오기**: 팝업 노출 시 LocalStorage에서 이전 입력값을 불러와 자동 입력
- **초기값 없는 경우**: 모든 필드를 빈 값으로 표시

### 2.7 신고파일 최종생성일 업데이트

간이지급명세서 생성 완료 시 다음 화면의 신고파일 최종생성일을 업데이트한다.

#### 2.7.1 사업소득 합산 화면(SPS_BI_01) 업데이트

- **업데이트 대상**: 해당 월(year + month)의 신고파일 최종생성일 필드
- **업데이트 값**: 현재 시각 (YYYY-MM-DD HH:mm:ss)
- **다운로드 버튼 추가**:
  - **위치**: 신고파일 최종생성일 컬럼 (다운로드 column)
  - **형태**: 아이콘 버튼
  - **노출 조건**: 신고파일 최종생성일이 있을 때만 노출
  - **클릭 시 동작**: 간이지급명세서 PDF 파일 다운로드
    - 파일명: `YYYY년 MM월 사업소득 간이지급명세서.pdf`

#### 2.7.2 사업소득 월별 리스트 화면(SPS_BI_02) 업데이트

- **업데이트 대상**: 선택된 각 대상자들(`targetIds`)의 신고파일 최종생성일 필드
- **업데이트 값**: 현재 시각 (YYYY-MM-DD HH:mm:ss)

### 2.8 저장 실패 처리

- 저장(submit) 실패 시 (비즈니스 로직 실패 등) **팝업을 유지**하고 에러를 표시한다.
- 유효성 검증 실패: 해당 필드 하단에 에러 문구 노출 후 팝업 유지
- 서버 에러: alert 노출 후 팝업 유지
  - 문구: **"서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."**

### 2.9 전체 동작 흐름

```
1. 사용자가 SPS_BI_02 화면에서 '간이지급명세서 생성' 버튼 클릭
2. Popover에서 '조회된 전체 대상자' 또는 '선택 대상자' 선택
3. SPS_BI_08 팝업 노출 (year, month, targetIds, targetType 전달)
4. LocalStorage에서 이전 입력값을 불러와 자동 입력
5. 상단 요약 정보를 선택된 대상자 기준으로 계산하여 표시
6. 사용자가 간이지급명세서 정보를 입력/수정
7. blur 이벤트마다 개별 유효성 검사를 수행
8. '생성' 버튼 클릭 시:
   a. 전체 유효성 검증
   b. LocalStorage에 입력값 저장
   c. 간이지급명세서 생성 (신고파일 최종생성일 업데이트)
   d. PDF 파일 다운로드 시작
   e. Alert 노출 ("YYYY년 MM월 사업소득 간이지급명세서 생성이 완료되었습니다.")
   f. Alert 확인 시 팝업 닫기
   g. 부모 화면(SPS_BI_02) 갱신
9. '취소' 버튼 또는 닫기 동작 시:
   a. 변경 사항 확인
   b. 변경 사항이 있으면 confirm 노출
   c. confirm 확인 시 팝업 닫기
10. 생성 실패 시 팝업을 유지하고 에러를 표시
```
