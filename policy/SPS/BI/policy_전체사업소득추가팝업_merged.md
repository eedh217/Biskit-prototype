# 전체 사업소득 추가 팝업(SPS_BI_06) - 최종 정책서

---

## 1. 화면 개요

- **화면 ID**: SPS_BI_06
- **화면명**: 전체 사업소득 추가 팝업
- **목적**: 전체 사업소득 화면(SPS_BI_05)에서 신규 사업소득 데이터를 등록하기 위한 입력 팝업
- **기본 기능**: SPS_BI_03(사업소득 추가 팝업)과 동일하며, **지급연도**와 **지급월** 입력 필드가 추가된다.
- **종속 조건**: SPS_BI_03과 달리 특정 지급연월에 종속되지 않으며, 사용자가 지급연도와 지급월을 직접 입력한다.
- **귀속 기준 예외 규칙**: SPS_BI_03과 동일. 업종코드가 보험설계사(940906), 음료배달(940907), 방문판매원(940908) 중 하나이면서 **귀속연도 ≠ 지급연도**인 경우, 등록된 데이터는 월별 리스트(SPS_BI_02)에서 **귀속연도 12월** 리스트에 표시된다. (귀속월은 고려하지 않고 연도 기준으로만 판정)
  - **예외 규칙 적용 시 안내**: submit 시 저장 전에 confirm을 노출한다.
    - confirm 문구: **"해당 데이터는 YYYY년 12월 사업소득에 표시됩니다."** (YYYY = 귀속연도)
    - 확인 클릭 시: 저장 진행 → 토스트("사업소득 추가를 완료했습니다.") 1.5초 노출
    - 취소 클릭 시: 저장 중단, 추가 팝업 유지
  - 예외 규칙 미해당 시: 기존 동작 유지 (바로 저장 후 토스트만 노출)
- **진입 경로**: SPS_BI_05 화면의 '사업소득 추가' 버튼 클릭 시 노출
- **접근 권한**: 별도 권한 제한 없음 (모든 사용자 접근 가능)
- **데이터 연동**: 추가된 데이터는 SPS_BI_01(사업소득 합산) 및 SPS_BI_02(사업소득 월별 리스트)에서도 즉시 반영된다.
- **연관 화면**:
  - SPS_BI_05 (전체 사업소득) — 부모 화면
  - SPS_BI_03 (사업소득 추가 팝업) — 기본 기능 동일 (지급연월 고정 vs 입력)

## 2. 요청한 화면 기능

### 2.1 닫기/이탈 방지 (공통 정책 5.1 적용)

- SPS_BI_03과 동일
- 팝업을 벗어나려는 모든 행동(팝업 외 영역 클릭, ESC 키 입력, 닫기 버튼 클릭, 브라우저 뒤로가기 등) 발생 시 confirm 노출
- 단, 사용자가 아무런 입력도 하지 않은 초기 상태에서는 confirm 없이 팝업을 바로 닫는다.
- **초기 상태 판정 기준**: 모든 입력 항목의 최종값이 초기값과 동일하면 초기상태로 판정한다. (예: 내외국인을 외국인으로 변경 후 다시 내국인으로 되돌린 경우 → 초기상태로 판정, confirm 없이 닫힘)
- confirm 문구: **"사업소득 추가를 취소하시겠습니까?"**

### 2.2 추가 완료 토스트 (공통 정책 5.2 적용)

- 추가 성공 시 토스트 1.5초 노출
- 문구: **"사업소득 추가를 완료했습니다."**
- 추가 완료 후 **팝업을 자동으로 닫고**, SPS_BI_05 리스트/카운트를 최신 상태로 갱신한다.

### 2.4 자동 계산 영역

입력 영역 하단에 다음 항목을 제공한다. 모두 **readonly**로 제공되며, 사용자가 직접 수정할 수 없다.

| 항목명 | 표시형식 | 상태 | 비고 |
|--------|---------|------|------|
| 세율(%) | 퍼센트 | readonly (조건부 입력 가능) | 기본 3%, 조건에 따라 변동 |
| 소득세 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 4.3 적용 |
| 지방소득세 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 4.3 적용 |
| 실지급액 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 4.3 적용 |

### 2.5 자동 계산 동작 규칙

#### 2.5.1 계산 트리거

- 모든 필수 입력값이 입력된 상태에서, 마지막 입력 필드 입력 시 **실시간으로** 자동 계산을 수행한다.

#### 2.5.2 세율(%) 결정 로직

| 조건 | 세율 | 필드 상태 |
|------|------|----------|
| 기본 (일반적인 경우) | **3%** | readonly |
| 업종코드 = 봉사료수취자(940905) | **5%** | readonly (외국인이어도 5% 고정) |
| 내외국인 = 외국인 AND 업종코드 = 직업운동가(940904) | **3% 또는 20%** (사용자 선택) | **입력 가능** (라디오 버튼으로 전환) |

- 내외국인이 외국인이고 업종코드가 직업운동가(940904)인 경우:
  - 세율(%) 영역이 **라디오 버튼 형태**로 변경된다.
  - 선택 값: **3%**, **20%**
  - 기본 선택값: **3%**
  - 사용자가 선택한 세율을 적용한다.
- 해당 조건을 벗어나는 경우, 세율(%) 필드는 다시 readonly 상태로 복귀한다.

#### 2.5.3 계산 순서

1. **세율 확정** (위 세율 결정 로직에 따라)
2. **소득세 계산**: 소득세 = 지급액 x 세율 (원단위 이하 절사)
3. **소액부징수 판정**: 소득세가 1,000원 미만인 경우 → 소득세 = 0원
4. **지방소득세 계산**:
   - 소득세 = 0원인 경우 → 지방소득세 = 0원
   - 소득세 > 0원인 경우 → 지방소득세 = 소득세 x 0.1 (원단위 이하 절사)
5. **실지급액 계산**: 실지급액 = 지급액 - 소득세 - 지방소득세

> ※ "원단위 이하 절사"는 Math.floor(소수점 이하 버림)를 적용한다.

#### 2.5.4 계산 예시

**예시 1. 지급액 30,000원 / 세율 3%**

| 항목 | 계산 | 결과 |
|------|------|------|
| 소득세 | 30,000 x 0.03 | 900원 |
| 소액부징수 판정 | 900 < 1,000 | 소득세 = **0원** |
| 지방소득세 | 소득세 0원이므로 | **0원** |
| 실지급액 | 30,000 - 0 - 0 | **30,000원** |

**예시 2. 지급액 100,000원 / 세율 3%**

| 항목 | 계산 | 결과 |
|------|------|------|
| 소득세 | 100,000 x 0.03 | **3,000원** |
| 소액부징수 판정 | 3,000 >= 1,000 | 소득세 유지 |
| 지방소득세 | 3,000 x 0.1 | **300원** |
| 실지급액 | 100,000 - 3,000 - 300 | **96,700원** |

### 2.6 유효성 검사 (공통 정책 5.3 적용)

- 검증 타이밍: **blur / submit**
- 에러 표시: 컴포넌트 하단에 에러 문구 노출
- 재입력 후 검증 통과 시 에러 문구 제거
- 포커스 아웃 시 정상 입력으로 판단되면 에러 문구 제거

#### 2.6.1 필수값 검증 (blur 시)

- 모든 필수 입력 항목(**지급연도, 지급월** 포함)이 미입력 상태에서 blur 시, 해당 필드 하단에 에러 문구 노출
- **공백만 입력된 경우 trim 후 빈값으로 판정**하여 동일하게 필수 에러 처리
- 에러 문구: **"필수 입력 항목입니다."**

#### 2.6.1-1 지급액 0원 검증 (blur 시)

- 지급액이 0원인 경우 blur 시 에러 처리
- 에러 문구: **"지급액은 1원 이상 입력해야 합니다."**

#### 2.6.2 귀속연월 유효성 검사 (blur 시)

- **지급연도와 지급월이 모두 선택**되고, **귀속연도와 귀속월이 모두 선택**된 상태에서 blur 시에만 검사한다. (하나라도 미선택 시 검사하지 않음)
- 귀속연월(귀속연도 + 귀속월)이 지급연월(지급연도 + 지급월)보다 **이후** 날짜인 경우 에러 (동일 연월은 허용, 귀속연월 ≤ 지급연월)
- 에러 표시 위치: **귀속연도+귀속월 영역 전체 하단**
- 에러 문구: **"귀속연월은 지급연월보다 이전 날짜여야합니다."**

#### 2.6.3 주민(사업자)등록번호 자릿수 검사 (blur 시)

- 주민(사업자)등록번호는 **10자리 또는 13자리**만 유효하다.
- 10자리, 13자리가 아닌 경우 blur 시 에러 처리
- 에러 문구: **"주민(사업자)등록번호는 10자리 또는 13자리만 입력 가능합니다."**

#### 2.6.4 주민(사업자)등록번호 체크디짓 검사 (blur 시)

- 주민등록번호(13자리) 및 사업자등록번호(10자리)에 대해 **체크디짓 알고리즘 검증**을 수행한다.
- 체크디짓 검증 실패 시 에러 처리
- 에러 문구: **"유효하지 않은 주민(사업자)등록번호입니다."**

#### 2.6.5 병의원 업종 예외 검사 (blur 시)

- 업종코드가 **병의원(851101)**인 경우, 주민(사업자)등록번호는 **10자리 이하만** 허용
- 10자리를 초과하는 경우 blur 시 에러 처리
- 에러 문구 표시 위치: **업종코드 또는 주민(사업자)등록번호 중 나중에 입력한 항목 하단**에 표시
- 에러 문구: **"병의원인 경우, 사업자등록번호만 입력하실 수 있습니다."**

#### 2.6.6 중복 데이터 검사 (submit 시)

- submit 시점에 **서버에서 최종 중복 체크**를 수행한다.
- **지급연월 + 귀속연월 + 주민(사업자)등록번호 + 업종코드**가 모두 동일한 데이터가 이미 존재하는 경우
- submit 시 등록을 차단하고 **alert 팝업**으로 에러를 표시한다.
- alert 문구: **"지급연월, 귀속연월, 주민(사업자)등록번호, 업종코드가 동일한 사업소득이 존재합니다."**
- **참고**: 동일 인물(주민등록번호)·귀속연월·업종코드라도 **지급연월이 다르면** 별도 건으로 등록할 수 있다. (예외 업종의 여러 달 지급 등)

### 2.7 안내 문구

팝업 하단에 다음 안내 문구를 **고정 표시**한다.

> ※ 지급총액 입력 시 업종코드에 따라 세율이 자동 적용되어 소득세, 지방소득세가 계산됩니다.
> ※ 소액부징수(소득세액이 1천원 미만)인 경우, 소득세가 면제됩니다.
> ※ 직업운동가(940904) 중 프로스포츠 구단과의 계약기간이 3년 이하인 외국인 직업 운동가일 경우, 세율 20%
> ※ 봉사료 수취자(940905) 중 「소득세법 시행령」제184조의2에 해당하는 봉사료 수입금액의 경우, 세율 5%

- 위 문구는 정보 제공 목적이다.

### 2.8 저장 실패 처리

- 저장(submit) 실패 시 (중복 검사 실패, 비즈니스 로직 실패 등) **팝업을 유지**하고 에러를 표시한다.
- 중복 검사 실패: alert 노출 후 팝업 유지
- 기타 비즈니스 로직 실패: 해당 필드 하단에 에러 문구 노출 후 팝업 유지
- 서버 에러: 공통 정책 5.1 alert 노출 후 팝업 유지

### 2.9 전체 동작 흐름

```
1. 사용자가 지급연도, 지급월을 포함한 모든 필수값을 입력한다.
2. 마지막 필드에서 focus out 시 세율 및 세액을 자동 계산한다.
3. blur 이벤트마다 개별 유효성 검사를 수행한다.
4. submit 시 입력한 지급연월 기준으로 중복 검사를 수행한다.
5. 귀속 기준 예외 해당 시 confirm 노출 (취소 시 저장 중단).
6. 모든 조건을 통과하면 데이터를 등록한다.
7. 등록 성공 시 토스트 노출 후 팝업을 자동으로 닫고 부모 화면(SPS_BI_05)을 갱신한다.
8. 등록 실패 시 팝업을 유지하고 에러를 표시한다.
```

### 2.10 서버 에러 처리 (공통 정책 5.1 적용)

- 서버 에러 발생 시 alert 팝업 노출
- 문구: **"서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."**
