# 사업소득 수정 팝업(SPS_BI_04) - 최종 정책서

---

## 1. 화면 개요

- **화면 ID**: SPS_BI_04
- **화면명**: 사업소득 수정 팝업
- **목적**: 사업소득 월별 리스트 화면(SPS_BI_02)에서 등록된 사업소득 데이터를 수정 또는 삭제하기 위한 팝업. 단건(비합산 행) 또는 다건(합산 행) 진입 모두 동일한 팝업을 사용한다.
- **단건/다건 모드**: 진입 시 전달받은 레코드 수에 따라 동작이 분기된다.
  - **단건 (1건)**: 탭 UI 숨김, 기존 단건 수정 모드
  - **다건 (2건 이상)**: 탭 UI 표시, 탭별 편집/삭제. 탭 삭제로 1건 남으면 자동으로 단건 모드로 전환.
- **종속 조건**: 해당 팝업은 특정 지급연월(YYYY년 MM월)에 종속되며, 수정되는 데이터는 현재 리스트 화면의 지급연월에 포함된다.
- **귀속 기준 예외 규칙**: 단, 업종코드가 보험설계사(940906), 음료배달(940907), 방문판매원(940908) 중 하나이면서 **귀속연도 ≠ 지급연도**인 경우, 수정된 데이터는 현재 지급연월 리스트가 아닌 **귀속연도 12월** 리스트에 표시된다. (귀속월은 고려하지 않고 연도 기준으로만 판정)
  - 귀속연도 = 지급연도인 경우에는 예외 규칙이 적용되지 않으며, 일반적인 수정 흐름을 따른다.
  - 귀속연도 또는 업종코드를 수정할 경우 예외 규칙의 적용 여부가 변경될 수 있다.
  - 예: 지급연월 2026/02, 귀속연월 2025/08, 보험설계사(940906) → 2025년 12월 리스트에 표시, 2026년 2월 리스트에는 미포함
  - **예외 규칙 적용 시 안내**:
    - **단건**: submit 시 저장 전에 confirm을 노출한다.
      - confirm 문구: **"해당 데이터는 YYYY년 12월 사업소득에 표시됩니다."** (YYYY = 귀속연도)
      - 확인 클릭 시: 저장 진행 → 토스트("사업소득 수정을 완료했습니다.") 1.5초 노출
      - 취소 클릭 시: 저장 중단, 수정 팝업 유지
    - **다건**: confirm 없이 바로 저장 후 토스트만 노출한다.
  - 예외 규칙 미해당 시: 기존 동작 유지 (바로 저장 후 토스트만 노출)
- **진입 경로**: SPS_BI_02 화면의 리스트 행 클릭 시 노출 (단건 행 또는 합산 행 모두)
- **연관 화면**:
  - SPS_BI_02 (사업소득 월별 리스트) - 부모 화면
  - SPS_BI_03 (사업소득 추가 팝업) - 기본 기능 동일

## 2. 요청한 화면 기능

### 2.1 수정 팝업 호출 키 (공통 정책 5.2 적용)

- 수정 팝업 호출 시 전달하는 키값은 별도 DB PK(id)를 사용한다.

### 2.2 닫기/이탈 방지 (공통 정책 4.3 적용)

- 팝업을 벗어나려는 모든 행동(팝업 외 영역 클릭, ESC 키 입력, 닫기 버튼 클릭, 브라우저 뒤로가기 등) 발생 시 confirm 노출
- 단, 기존 입력값에서 변경된 내용이 없는 경우에는 confirm 없이 팝업을 바로 닫는다.
- confirm 문구: **"사업소득 수정을 취소하시겠습니까?"**

### 2.3 수정 완료 토스트 (공통 정책 5.4 적용)

- 수정 성공 시 토스트 1.5초 노출
- 문구: **"사업소득 수정을 완료했습니다."**
- 수정 완료 후 SPS_BI_02 리스트/카운트/상단 요약 정보를 최신 상태로 갱신한다.
- 수정 완료 후 SPS_BI_02의 기존 스크롤 위치를 유지한다.

### 2.5 자동 계산 영역

입력 영역 하단에 다음 항목을 제공한다. 모두 **readonly**로 제공되며, 사용자가 직접 수정할 수 없다.

| 항목명 | 표시형식 | 상태 | 비고 |
|--------|---------|------|------|
| 세율(%) | 퍼센트 | readonly (조건부 입력 가능) | 기본 3%, 조건에 따라 변동 |
| 소득세 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 4.3 적용 |
| 지방소득세 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 4.3 적용 |
| 실지급액 | 금액(천 단위 콤마 + "원") | readonly | 공통 정책 5.1, 4.3 적용 |

### 2.6 자동 계산 동작 규칙

#### 2.6.1 계산 트리거

- 팝업 진입 시 기존 등록값을 기반으로 자동 계산 결과를 표시한다.
- 지급액 또는 세율에 영향을 주는 필드(내외국인 구분, 업종코드, 지급액) 변경 시 **실시간으로** 자동 재계산을 수행한다.
- 특히 **내외국인 구분** 변경 시 세율 조건부 전환이 즉시 트리거된다.

#### 2.6.2 세율(%) 결정 로직

| 조건 | 세율 | 필드 상태 |
|------|------|----------|
| 기본 (일반적인 경우) | **3%** | readonly |
| 업종코드 = 봉사료수취자(940905) | **5%** | readonly |
| 내외국인 = 외국인 AND 업종코드 = 직업운동가(940904) | **3% 또는 20%** (사용자 선택) | **입력 가능** (라디오 버튼으로 전환) |

- 내외국인이 외국인이고 업종코드가 직업운동가(940904)인 경우:
  - 세율(%) 영역이 **라디오 버튼 형태**로 변경된다.
  - 선택 값: **3%**, **20%**
  - 기본 선택값: 기존 등록값 바인딩 (신규 계산 시 **3%**)
  - 사용자가 선택한 세율을 적용한다.
- 해당 조건을 벗어나는 경우, 세율(%) 필드는 다시 readonly 상태로 복귀한다.
- 업종코드 변경으로 세율 라디오가 사라졌다가 다시 나타나는 경우(예: 940904 → 다른 코드 → 940904), **이전에 선택한 세율 값을 유지**한다.
- 세율 조건부 전환은 **업종코드 변경** 또는 **내외국인 구분 변경**에 의해 트리거된다.
  - 예: 내국인 → 외국인 변경 + 업종코드 직업운동가(940904) → 세율 라디오 나타남 (기본 3%)
  - 예: 외국인 → 내국인 변경 + 업종코드 직업운동가(940904) → 세율 라디오 사라짐, 3% 고정

#### 2.6.3 계산 순서

1. **세율 확정** (위 세율 결정 로직에 따라)
2. **소득세 계산**: 소득세 = 지급액 x 세율 (원단위 이하 절사)
3. **소액부징수 판정**: 소득세가 1,000원 미만인 경우 → 소득세 = 0원
4. **지방소득세 계산**:
   - 소득세 = 0원인 경우 → 지방소득세 = 0원
   - 소득세 > 0원인 경우 → 지방소득세 = 소득세 x 0.1 (원단위 이하 절사)
5. **실지급액 계산**: 실지급액 = 지급액 - 소득세 - 지방소득세

#### 2.6.4 계산 예시

**예시 1. 지급액 0원 / 세율 3%**

| 항목 | 계산 | 결과 |
|------|------|------|
| 소득세 | 0 x 0.03 | 0원 |
| 소액부징수 판정 | 0 < 1,000 | 소득세 = **0원** |
| 지방소득세 | 소득세 0원이므로 | **0원** |
| 실지급액 | 0 - 0 - 0 | **0원** |

**예시 2. 지급액 100,000원 / 세율 3%**

| 항목 | 계산 | 결과 |
|------|------|------|
| 소득세 | 100,000 x 0.03 | **3,000원** |
| 소액부징수 판정 | 3,000 >= 1,000 | 소득세 유지 |
| 지방소득세 | 3,000 x 0.1 | **300원** |
| 실지급액 | 100,000 - 3,000 - 300 | **96,700원** |

### 2.7 유효성 검사 (공통 정책 5.3 적용)

- 검증 타이밍: **blur / submit**
- 에러 표시: 컴포넌트 하단에 에러 문구 노출
- 재입력 후 검증 통과 시 에러 문구 제거
- 포커스 아웃 시 정상 입력으로 판단되면 에러 문구 제거
- 여러 필드에서 동시에 유효성 에러가 발생한 경우, **모든 에러를 동시에 표시**한다.
- 유효성 에러가 1건 이상 존재하는 경우, **수정 버튼을 비활성화**한다. 모든 에러가 해소되면 수정 버튼이 다시 활성화된다.

#### 2.7.1 필수값 검증 (blur 시)

- 모든 필수 입력 항목(귀속연도, 귀속월, 성명(상호), 내외국인 구분, 주민(사업자)등록번호, 업종코드, 지급액)이 미입력 상태에서 blur 시, 해당 필드 하단에 에러 문구 노출
- **공백만 입력된 경우 trim 후 빈값으로 판정**하여 동일하게 필수 에러 처리
- 에러 문구: **"필수 입력 항목입니다."**

#### 2.7.2 귀속연월 유효성 검사 (blur 시)

- 귀속연월(귀속연도 + 귀속월)이 지급연월보다 **이후** 날짜인 경우 에러 (동일 연월은 허용, 귀속연월 ≤ 지급연월)
- 에러 문구: **"귀속연월은 지급연월보다 이전 날짜여야합니다."**

#### 2.7.3 주민(사업자)등록번호 자릿수 검사 (blur 시)

- 주민(사업자)등록번호는 **10자리 또는 13자리**만 유효하다.
- 10자리, 13자리가 아닌 경우 blur 시 에러 처리
- 에러 문구: **"주민(사업자)등록번호는 10자리 또는 13자리만 입력 가능합니다."**

#### 2.7.4 주민(사업자)등록번호 체크디짓 검사 (blur 시)

- 주민등록번호(13자리) 및 사업자등록번호(10자리)에 대해 **체크디짓 알고리즘 검증**을 수행한다.
- 체크디짓 검증 실패 시 에러 처리
- 에러 문구: **"유효하지 않은 주민(사업자)등록번호입니다."**

#### 2.7.5 병의원 업종 예외 검사 (blur 시)

- 업종코드가 **병의원(851101)**인 경우, 주민(사업자)등록번호는 **10자리 이하만** 허용
- 10자리를 초과하는 경우 blur 시 에러 처리
- 에러 문구 표시 위치: **업종코드 또는 주민(사업자)등록번호 중 나중에 입력한 항목 하단**에 표시
- 에러 문구: **"병의원인 경우, 사업자등록번호만 입력하실 수 있습니다."**

#### 2.7.6 중복 데이터 검사 (submit 시)

- **지급연월 + 귀속연월 + 주민(사업자)등록번호 + 업종코드**가 모두 동일한 데이터가 이미 존재하는 경우 (자기 자신 제외)
- submit 시 수정을 차단한다.
- alert 문구: **"지급연월, 귀속연월, 주민(사업자)등록번호, 업종코드가 동일한 사업소득이 존재합니다."**
- **참고**: 동일 인물(주민등록번호)·귀속연월·업종코드라도 **지급연월이 다르면** 별도 건으로 존재할 수 있다. (예외 업종의 여러 달 지급 등)

### 2.8 삭제 (단건/다건 분기)

#### 2.8.1 단건 모드 삭제

- 수정 팝업 내 **"삭제"** 버튼을 제공한다.
- 삭제 버튼 클릭 시 confirm 팝업 노출
  - 데이터 수정 상태(변경사항 존재)와 무관하게, 수정 취소 confirm 없이 바로 삭제 confirm을 표시한다.
  - 문구: **"사업소득을 삭제하시겠습니까? 삭제한 정보는 복구할 수 없습니다."**
- confirm 확인 버튼 클릭 시
  1. **삭제 가능 여부 조건: 없음 (항상 삭제 가능)**
  2. 삭제 처리 후 팝업을 닫는다.
  3. SPS_BI_02 리스트/카운트/상단 요약 정보를 최신 상태로 갱신한다.
  4. 토스트: **"사업소득 삭제를 완료했습니다."**

#### 2.8.2 다건 모드 삭제 ("이 건 삭제")

- 각 탭에 **"이 건 삭제"** 버튼을 제공한다.
- 버튼 클릭 시 confirm 팝업 노출
  - 문구: **"YYYY년 M월 지급 건을 삭제하시겠습니까? 삭제한 정보는 복구할 수 없습니다."**
- confirm 확인 클릭 시:
  1. 해당 레코드를 **즉시 삭제**
  2. 팝업 내 토스트 1.5초 노출 — **"사업소득 삭제를 완료했습니다."**
  3. 해당 탭을 팝업에서 **제거**
  4. 남은 탭 중 가장 가까운 탭으로 자동 전환
  5. 남은 건이 1건이면 **단건 모드로 동적 전환** (탭 UI 사라짐, 삭제 버튼 텍스트 "삭제"로 변경)
  6. **SPS_BI_02 부모 화면의 리스트/카운트/상단 요약 정보를 즉시 갱신**한다. (팝업은 유지)
- **마지막 건 삭제**: 그룹 내 마지막 레코드를 삭제할 경우, 삭제 처리 후 **팝업을 자동으로 닫는다**.
  - SPS_BI_02 리스트/카운트/상단 요약 정보를 최신 상태로 갱신한다.

### 2.9 탭 구조 및 탭 전환 (다건 모드)

- 다건(합산 행) 진입 시 팝업 상단(타이틀 하단)에 **탭**을 표시한다.
- 각 탭은 합산 그룹 내 개별 레코드를 나타낸다.
- 탭 라벨: **"YYYY.MM 지급"** (해당 레코드의 지급연도.지급월)
- 탭 정렬: 지급연월 오름차순
- 유효성 에러가 있는 탭에는 **"!" 표시**를 추가하여 시각적으로 구분한다.
- 단건(1건) 진입 시 탭 UI는 숨긴다.

#### 탭 전환 로직
- **탭 클릭 시** 다음 순서로 동작한다:
  1. **현재 탭에 유효성 에러가 있는 경우**:
     - 탭 전환을 중단한다.
     - 첫 번째 에러 필드로 포커스를 이동한다.
     - 필드 우선순위: 귀속연도 → 귀속월 → 성명 → 주민번호 → 업종코드 → 지급액
  2. **현재 탭에 에러가 없고, 수정사항이 있는 경우**:
     - confirm 다이얼로그를 노출한다.
     - confirm 문구: **"수정한 내용을 저장하시겠습니까?"**
     - **"예" 클릭**: 현재 탭의 수정사항을 저장한 후 대상 탭으로 전환한다. 부모 화면(SPS_BI_02)을 즉시 갱신한다.
     - **"아니오" 클릭**: 현재 탭의 수정사항을 **원래 상태로 복원**한 후 대상 탭으로 전환한다.
  3. **현재 탭에 에러가 없고, 수정사항도 없는 경우**:
     - confirm 없이 즉시 대상 탭으로 전환한다.

### 2.10 수정 버튼 동작 (현재 탭만 저장)

- **"수정" 버튼** 클릭 시 **현재 선택된 탭(active tab)만** 유효성 검사를 수행한다.
- 검증 실패 시 에러를 표시하고 저장을 중단한다. (수정 버튼은 에러가 있으면 비활성화 상태)
- 검증 통과 시:
  - **단건 모드**: 예외 규칙(귀속연도 ≠ 지급연도인 예외 업종) 해당 시 confirm 노출 → 확인 시 저장 진행
  - **다건 모드**: 예외 규칙 해당 시에도 **confirm 없이 바로 저장** 진행
- 저장 성공 시:
  - 팝업을 닫는다.
  - 부모 화면(SPS_BI_02) 리스트/카운트/상단 요약 정보를 최신 상태로 갱신한다.
  - 토스트: **"사업소득 수정을 완료했습니다."** (1.5초)
- 저장 중 에러 발생 시 팝업을 유지하고 에러를 표시한다.

### 2.11 그룹 분리 (다건 모드)

- 귀속연월/업종코드/주민번호 수정 시 그룹핑 키(귀속연월 + 주민등록번호 + 업종코드) 변경으로 기존 그룹에서 자연 분리.
- 분리된 레코드는 월별 리스트에서 개별 행 또는 새로운 합산 그룹에 포함된다.
- 별도의 안내나 confirm 없이, 저장 후 리스트 갱신 시 자동으로 반영된다.

### 2.12 안내 문구

팝업 하단에 다음 안내 문구를 **고정 표시**한다.

> ※ 지급총액 입력 시 업종코드에 따라 세율이 자동 적용되어 소득세, 지방소득세가 계산됩니다.
> ※ 소액부징수(소득세액이 1천원 미만)인 경우, 소득세가 면제됩니다.
> ※ 직업운동가(940904) 중 프로스포츠 구단과의 계약기간이 3년 이하인 외국인 직업 운동가일 경우, 세율 20%
> ※ 봉사료 수취자(940905) 중 「소득세법 시행령」제184조의2에 해당하는 봉사료 수입금액의 경우, 세율 5%

- 위 문구는 정보 제공 목적이다.

### 2.13 저장 실패 처리

- 저장(submit) 실패 시 (중복 검사 실패, 비즈니스 로직 실패 등) **팝업을 유지**하고 에러를 표시한다.
- 중복 검사 실패: alert 노출 후 팝업 유지
- 기타 비즈니스 로직 실패: 해당 필드 하단에 에러 문구 노출 후 팝업 유지
- 서버 에러(5xx): 공통 정책 5.1 alert 노출 후 팝업 유지
- 클라이언트 에러(4xx): 서버 응답 메시지 기반 에러 문구 노출 후 팝업 유지
- 네트워크 오류: 공통 정책 5.2 alert 노출 후 팝업 유지

### 2.14 전체 동작 흐름

```
1. SPS_BI_02 리스트 행 클릭 시 해당 행의 레코드 배열을 전달하며 수정 팝업을 호출한다.
   - 비합산 행: 1건 배열 → 단건 모드 (탭 숨김)
   - 합산 행: 2건 이상 배열 → 다건 모드 (탭 표시)
2. 팝업 진입 시 기존 등록값이 각 필드에 바인딩된다. (다건 시 탭별로 독립 바인딩)
3. 팝업 진입 시 기존값 기반으로 자동 계산 결과(세율/소득세/지방소득세/실지급액)를 표시한다.
4. 모든 필드(귀속연도, 귀속월, 성명(상호), 내외국인 구분, 주민(사업자)등록번호, 업종코드, 지급액)를 변경할 수 있다.
5. 필드 변경 후 focus out 시 세율 및 세액을 자동 재계산한다. 내외국인 변경 시 세율 조건부 전환도 트리거된다.
6. blur 이벤트마다 개별 유효성 검사를 수행한다. 에러 시 수정 버튼 비활성화 (현재 탭의 에러만 체크).
7. 다건 모드에서 탭 전환 시:
   - 현재 탭에 에러 있음 → 첫 번째 에러 필드로 포커스, 탭 전환 중단
   - 현재 탭에 수정사항 있음 → confirm 노출 ("수정한 내용을 저장하시겠습니까?")
     - "예": 현재 탭만 저장 후 탭 전환, 부모 화면 갱신
     - "아니오": 현재 탭 원래 상태로 복원 후 탭 전환
   - 에러도 없고 수정사항도 없음 → 즉시 탭 전환
8. 다건 모드에서 "이 건 삭제"로 탭을 삭제할 수 있다. 1건 남으면 단건 모드로 전환.
9. "수정" 버튼 클릭 시 현재 탭만 검증 및 저장한다.
   - 단건 모드: 예외 규칙 해당 시 confirm 노출 (취소 시 저장 중단)
   - 다건 모드: 예외 규칙 해당 시에도 confirm 없이 바로 저장
10. 수정 성공 시 토스트("사업소득 수정을 완료했습니다.") 노출 후 팝업을 닫고 부모 화면을 갱신한다.
11. 수정 실패 시 팝업을 유지하고 에러를 표시한다.
12. 단건 삭제 버튼 클릭 시 삭제 confirm → 삭제 처리 → 팝업 닫힘 → 부모 화면 갱신.
```

### 2.15 서버 에러 처리 (공통 정책 5.1 적용)

- **서버 에러(5xx)** 발생 시 alert 팝업 노출
  - 문구: **"서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."**
  - 수정 팝업은 유지한다.
- **클라이언트 에러(4xx)** 발생 시 서버 응답 메시지 기반으로 해당 필드 하단에 에러 문구를 노출한다.
  - 수정 팝업은 유지한다.
- **네트워크 오류**(타임아웃, 연결 끊김 등) 발생 시 alert 팝업 노출
  - 문구: **"네트워크 연결을 확인해 주세요."**
  - 수정 팝업은 유지한다.
